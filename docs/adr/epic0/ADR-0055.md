# ADR-0055: AI Core Architecture

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

EREN's AI capabilities require a dedicated AI Core that provides cognitive functions (reasoning, trust, explanation) to all bounded contexts.

---

## Decision

**AI Core is a separate bounded context that provides cognitive capabilities as services to other contexts.**

AI Core does NOT own patient data. It receives queries and returns recommendations. All data stays in the source context.

---

## AI Core Capabilities

| Capability | Description |
|-----------|------------|
| Perceive | Receive and parse device data, alarms, clinical events |
| Remember | Store and retrieve conversation and patient context |
| Know | Retrieve relevant knowledge from the Knowledge base |
| Trust | Assess confidence and quality of reasoning |
| Reason | Generate reasoning chains from evidence |
| Decide | Produce recommendations based on reasoning |
| Explain | Generate human-readable explanations |
| Learn | Improve from feedback (EPIC 9) |
| Reflect | Self-evaluate reasoning quality |

See `EREN_COGNITIVE_MODEL.md` (Epic 0) for the full cognitive network.
