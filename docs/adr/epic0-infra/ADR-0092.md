# ADR-0092: API Gateway Strategy

**Status:** Proposed

**Date:** 2026-07-16

**Deciders:** Infrastructure Team

---

## Context

EREN needs an API gateway to handle cross-cutting concerns: TLS termination, rate limiting, authentication verification, and request routing.

---

## Decision

**NGINX Ingress Controller + Kong API Gateway for API management.**

- **NGINX Ingress:** TLS termination, rate limiting, basic routing (infrastructure layer)
- **Kong API Gateway:** Advanced API management, OAuth2 verification, plugin system (application layer)

---

## API Gateway Responsibilities

| Responsibility | Tool |
|---------------|------|
| TLS termination | NGINX Ingress |
| Rate limiting | NGINX Ingress + Kong |
| Auth verification | Kong |
| Request routing | NGINX Ingress |
| API versioning | Kong |
| Request/response logging | Kong |
| CORS | NGINX Ingress |

---

## Gateway vs Direct Access

```
External Client
    │
    ▼
CloudFlare (CDN + WAF)
    │
    ▼
ALB (Load Balancer)
    │
    ▼
NGINX Ingress Controller
    │
    ├──► Kong API Gateway ──► EREN API
    │
    └──► EREN Frontend (static)

Internal Services
    │
    ▼
Kong API Gateway
    │
    ├──► EREN API
    ├──► EREN Worker
    └──► EREN Cron
```
