# ADR-0102: SQLAlchemy 2.0 ORM

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

EREN needs an ORM that supports async database operations, works with PostgreSQL, integrates with Alembic for migrations, and enforces the Repository pattern defined in Epic 0.

## Decision

Use **SQLAlchemy 2.0+** with **asyncpg** as the database driver.

## Rationale

- **Async**: SQLAlchemy 2.0 async engine works with `asyncpg` for non-blocking I/O
- **Mature**: 15+ years of production usage, excellent documentation
- **Alembic integration**: Native support for Alembic migrations
- **Repository pattern**: Clean separation between ORM models and domain entities
- **Type safety**: SQLAlchemy 2.0 has improved typing support
- **Migration compatibility**: Works with `psycopg2` (sync) for Alembic

## Consequences

### Positive

- Full async/await support for API handlers
- Connection pooling with `asyncpg`
- Declarative and async ORM patterns
- Alembic autogenerate support

### Negative

- Two drivers needed: `asyncpg` (app) + `psycopg2` (Alembic)
- SQLAlchemy async requires careful session management

## Implementation

```python
# apps/api/app/core/database.py
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine
from sqlalchemy.orm import sessionmaker

engine = create_async_engine(
    settings.database_url,
    pool_size=20,
    max_overflow=10,
    pool_pre_ping=True,
)

async_session = sessionmaker(
    engine,
    class_=AsyncSession,
    expire_on_commit=False,
)
```
