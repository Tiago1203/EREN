# ADR-0100: FastAPI as API Framework

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

EREN needs a modern, high-performance Python web framework for its API layer. The framework must support async/await natively, have excellent OpenAPI/Swagger documentation, handle dependency injection, and integrate cleanly with SQLAlchemy, Pydantic, and OpenTelemetry.

## Decision

Use **FastAPI 0.115+** as the API framework for `apps/api`.

## Rationale

- **Async-first**: Native `async def` support matches SQLAlchemy 2.0 async drivers
- **OpenAPI**: Automatic Swagger UI and ReDoc generation out of the box
- **Type safety**: Full Pydantic v2 integration with validation
- **Performance**: Comparable to Node.js and Go on benchmarks
- **Ecosystem**: Large community, mature libraries (Uvicorn, Starlette)
- **OpenTelemetry**: Official instrumentation packages available

## Alternatives Considered

| Framework | Rejected Because |
|-----------|-----------------|
| Flask | No native async support, requires manual request parsing |
| Django Ninja | Too new, smaller ecosystem |
| Starlette | Too low-level (no dependency injection, no automatic OpenAPI) |

## Consequences

### Positive

- Automatic API documentation at `/docs` and `/redoc`
- FastAPI's dependency injection replaces manual DI container for HTTP layer
- Automatic Pydantic validation reduces boilerplate
- Background tasks via `BackgroundTasks`

### Negative

- FastAPI is tightly coupled to Pydantic for request/response validation
- Some flexibility traded for convention over configuration

## Implementation

```python
from fastapi import FastAPI, Depends

def create_app() -> FastAPI:
    app = FastAPI(title="EREN API", version="0.1.0")
    app.include_router(api_router, prefix=settings.api_v1_prefix)
    return app
```
