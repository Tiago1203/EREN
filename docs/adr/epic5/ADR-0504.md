# ADR-0504: Troubleshooting Engine

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Engineers need step-by-step guidance for troubleshooting medical devices. The Troubleshooting Engine provides interactive decision trees.

## Decision

We will implement an **interactive troubleshooting system**:

### Decision Tree Structure
```
START
  │
  ├─→ Is device powered? ──No──→ Check power supply
  │
  ├─→ Error code present? ──Yes──→ Check error code table
  │
  └─→ Physical inspection ──→ Check connections
```

### Safety Checks
- Power isolation
- Radiation safety
- Biohazard protocols

### Verification Steps
- Functional testing
- Calibration verification
- Documentation requirements

## Consequences

### Positive
- Consistent troubleshooting approach
- Reduced errors
- Training support

### Negative
- May not cover all scenarios
- Requires continuous update
