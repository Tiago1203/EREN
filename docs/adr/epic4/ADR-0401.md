# ADR-0401: LLM Provider Abstraction

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EREN necesita flexibility para usar diferentes LLM providers:
- **OpenAI** — GPT-4o, GPT-4 Turbo
- **Anthropic** — Claude 3.5 Sonnet, Claude 3 Opus
- **Ollama** — Modelos locales (Llama 3, Mistral)
- **Azure OpenAI** — Para compliance

Epic 0 ADR-0057 estableció LLM Provider Abstraction como requirement. Este ADR define la implementación.

---

## Decision

**Usar el patrón Adapter con un contrato común:**

```python
# core/cognitive/reasoning/domain/contracts/llm.py
class LLMContract(Protocol):
    """Contrato común para todos los LLM providers."""
    
    async def complete(
        self,
        prompt: str,
        system_prompt: str | None = None,
        temperature: float = 0.7,
        max_tokens: int = 4096,
        stop_sequences: list[str] | None = None,
    ) -> LLMResponse:
        """Genera una respuesta de texto."""
        ...
    
    async def complete_json(
        self,
        prompt: str,
        schema: dict,
        system_prompt: str | None = None,
        temperature: float = 0.3,
    ) -> LLMResponse:
        """Genera una respuesta estructurada (JSON)."""
        ...
    
    async def embed(
        self,
        texts: list[str],
        model: str = "embedding-model",
    ) -> list[list[float]]:
        """Genera embeddings para RAG."""
        ...
```

**Providers implementados:**

```
core/cognitive/reasoning/infrastructure/llm_adapters/
├── openai_adapter.py      # OpenAI GPT-4o
├── anthropic_adapter.py   # Claude 3.5
├── ollama_adapter.py      # Ollama local
└── azure_adapter.py       # Azure OpenAI
```

**Configuración:**

```python
# config.py
class LLMConfig(BaseModel):
    provider: Literal["openai", "anthropic", "ollama", "azure"]
    model: str
    api_key: SecretStr
    base_url: str | None = None  # Para proxies y Azure
    temperature: float = 0.7
    max_tokens: int = 4096
    
    # Rate limiting
    requests_per_minute: int = 60
    requests_per_day: int = 10000
```

---

## Reasons

**1. Vendor lock-in prevention:**
- No depender de un solo provider
- Permite cambiar según costos, disponibilidad, compliance

**2. Testing:**
- Mock adapter para unit tests
- No requiere API keys en testing

**3. Fallback:**
- Si un provider falla, fallback a otro
- A/B testing de modelos

**4. Cost optimization:**
- Usar modelos más baratos para tareas simples
- Modelos potentes solo cuando necesario

---

## Consequences

**Positive:**
- Flexibility total de providers
- Testing reproducible
- Cost optimization posible
- Compliance flexibility (Azure para regulated environments)

**Negative:**
- Contrato puede ser limitante si nuevo provider tiene features únicas
- Overhead de abstracción
- Más configuración a manage
