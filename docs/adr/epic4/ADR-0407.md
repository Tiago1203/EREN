# ADR-0407: Safety & Hallucination Detection

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EREN opera en contexto médico donde:
1. **Información fabricda** puede causar daño
2. **Contenido dañino** es inaceptable
3. **PHI** debe ser protegido
4. **Alucinaciones** son un riesgo real

Epic 0 Guardrails G5, G6, G7 establecen requisitos de seguridad. Este ADR define la implementación.

---

**Safety Checks:**

```python
@dataclass
class SafetyCheckResult:
    is_safe: bool
    risk_level: RiskLevel
    violations: list[SafetyViolation]
    sanitized_content: str | None
    audit_id: str

class SafetyEngine:
    """Main safety validation engine."""
    
    async def validate_input(
        self,
        message: Message,
        context: SafetyContext,
    ) -> SafetyCheckResult:
        """
        Pre-processing safety checks:
        1. Content filter (profanity, etc.)
        2. PHI detection (patient info in input)
        3. Injection detection (prompt injection)
        4. Rate limiting
        """
        violations = []
        
        # 1. Content filter
        content_result = await self.content_filter.check(message.content)
        if not content_result.is_safe:
            violations.append(SafetyViolation(
                type=ViolationType.INAPPROPRIATE_CONTENT,
                severity=Severity.HIGH,
                details=content_result.violation,
            ))
        
        # 2. PHI detection
        phi_result = await self.phi_detector.detect(message.content)
        if phi_result.found:
            violations.append(SafetyViolation(
                type=ViolationType.PHI_DETECTED,
                severity=Severity.MEDIUM,
                details=f"PHI types: {phi_result.phi_types}",
            ))
            # Log for audit but don't block
            await self.audit_logger.log_phi_access(phi_result)
        
        # 3. Injection detection
        injection_result = await self.injection_detector.check(message.content)
        if injection_result.is_injection:
            violations.append(SafetyViolation(
                type=ViolationType.PROMPT_INJECTION,
                severity=Severity.CRITICAL,
                details=injection_result.attack_type,
            ))
        
        # 4. Rate limiting
        rate_result = await self.rate_limiter.check(context.user_id)
        if rate_result.exceeded:
            violations.append(SafetyViolation(
                type=ViolationType.RATE_LIMIT_EXCEEDED,
                severity=Severity.LOW,
                details=f"Requests: {rate_result.count}/{rate_result.limit}",
            ))
        
        is_safe = not any(v.severity == Severity.CRITICAL for v in violations)
        
        return SafetyCheckResult(
            is_safe=is_safe,
            risk_level=self.calculate_risk_level(violations),
            violations=violations,
            sanitized_content=self.sanitize(message.content, violations),
            audit_id=self.generate_audit_id(),
        )
    
    async def validate_output(
        self,
        response: LLMResponse,
        context: SafetyContext,
    ) -> SafetyCheckResult:
        """
        Post-processing safety checks:
        1. Hallucination detection
        2. Medical safety
        3. Citation verification
        4. Output content filter
        """
        violations = []
        
        # 1. Hallucination detection
        hallucination_result = await self.hallucination_detector.check(
            response.content,
            context.evidence,
        )
        if hallucination_result.is_hallucination:
            violations.append(SafetyViolation(
                type=ViolationType.HALLUCINATION,
                severity=Severity.HIGH,
                details=hallucination_result.details,
            ))
        
        # 2. Medical safety
        medical_result = await self.medical_safety.check(
            response.content,
            context.domain,
        )
        if not medical_result.is_safe:
            violations.append(SafetyViolation(
                type=ViolationType.MEDICAL_UNSAFE,
                severity=Severity.CRITICAL,
                details=medical_result.concern,
            ))
        
        # 3. Citation verification
        citation_result = await self.verify_citations(response.citations)
        if not citation_result.all_valid:
            violations.append(SafetyViolation(
                type=ViolationType.INVALID_CITATION,
                severity=Severity.MEDIUM,
                details=f"Invalid: {citation_result.invalid_citations}",
            ))
        
        # 4. Output content filter
        content_result = await self.content_filter.check(response.content)
        if not content_result.is_safe:
            violations.append(SafetyViolation(
                type=ViolationType.INAPPROPRIATE_CONTENT,
                severity=Severity.HIGH,
                details=content_result.violation,
            ))
        
        is_safe = not any(v.severity in [Severity.CRITICAL, Severity.HIGH] for v in violations)
        
        return SafetyCheckResult(
            is_safe=is_safe,
            risk_level=self.calculate_risk_level(violations),
            violations=violations,
            sanitized_content=self.sanitize(response.content, violations),
            audit_id=context.audit_id,
        )
```

**Hallucination Detection:**

```python
class HallucinationDetector:
    """Detects fabricated information in responses."""
    
    async def check(
        self,
        content: str,
        evidence: list[Evidence],
    ) -> HallucinationResult:
        """
        Detection strategies:
        1. Factual consistency with evidence
        2. Citation presence and accuracy
        3. Uncertainty markers
        4. Out-of-domain claims
        """
        issues = []
        
        # 1. Factual consistency
        for claim in self.extract_claims(content):
            if claim.requires_evidence and not claim.has_supporting_evidence:
                if not self.has_similar_evidence(claim, evidence):
                    issues.append(HallucinationIssue(
                        type=IssueType.UNSUPPORTED_CLAIM,
                        claim=claim.text,
                        severity=self.estimate_severity(claim),
                    ))
        
        # 2. Citation check
        citations = self.extract_citations(content)
        for citation in citations:
            if not self.verify_citation_exists(citation):
                issues.append(HallucinationIssue(
                    type=IssueType.INVALID_CITATION,
                    citation=citation,
                    severity=Severity.HIGH,
                ))
        
        # 3. Uncertainty check
        if self.is_overconfident_without_evidence(content, evidence):
            issues.append(HallucinationIssue(
                type=IssueType.OVERCONFIDENCE,
                details="Response lacks uncertainty markers",
                severity=Severity.MEDIUM,
            ))
        
        is_hallucination = len([i for i in issues if i.severity == Severity.HIGH]) > 0
        
        return HallucinationResult(
            is_hallucination=is_hallucination,
            issues=issues,
            confidence=1.0 - (len(issues) / 10),  # Simple scoring
        )
```

## Violation Handling

| Violation Type | Severity | Action |
|----------------|----------|--------|
| PROMPT_INJECTION | CRITICAL | Block, log, alert |
| MEDICAL_UNSAFE | CRITICAL | Block, escalate, alert |
| HALLUCINATION | HIGH | Block or warn |
| INAPPROPRIATE_CONTENT | HIGH | Block or sanitize |
| PHI_DETECTED | MEDIUM | Log, audit, warn |
| INVALID_CITATION | MEDIUM | Warn, correct |
| RATE_LIMIT | LOW | Warn |

---

## Reasons

**1. Safety-first:**
- Medical context requires highest safety bar
- CRITICAL violations = block immediately
- Audit trail for compliance

**2. Layered defense:**
- Pre-processing catches input issues
- Post-processing catches output issues
- Defense in depth

**3. Hallucination prevention:**
- Critical for medical AI
- Evidence grounding required
- Citation verification

---

## Consequences

**Positive:**
- High safety standards
- Audit trail for compliance
- Reduced liability
- User trust

**Negative:**
- Latency added by checks
- False positives may block valid content
- Complex rule management
