# ADR-0410: Explainability Requirements

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EREN debe explicar sus decisiones para:
1. **Transparency** — Usuarios saben por qué EREN sugiere algo
2. **Trust** — Explicacionesbuild trust
3. **Verification** — Profesionales pueden validar
4. **Learning** — Usuarios aprenden de explicaciones
5. **Compliance** — Audit trails requeridos

Epic 0 establece: "EREN always explains." Este ADR define los requisitos de explicabilidad.

---

## Decision

**Explanation Types:**

```python
@dataclass
class Explanation:
    """Complete explanation of a response."""
    
    # What was the question?
    query: str
    query_summary: str
    
    # What reasoning happened?
    reasoning_steps: list[ReasoningStep]
    reasoning_summary: str
    
    # What evidence was used?
    evidence: list[EvidenceItem]
    evidence_summary: str
    
    # What confidence do we have?
    confidence: ConfidenceScore
    confidence_explanation: str
    
    # What are the alternatives?
    alternatives: list[Alternative]
    alternative_summary: str
    
    # User-friendly summary
    user_summary: str
    technical_details: str
    
    # Metadata
    generated_at: datetime
    complexity_level: ComplexityLevel

@dataclass
class ReasoningStep:
    """A single step in the reasoning process."""
    
    step_number: int
    description: str
    
    # What was the input?
    input_state: str
    
    # What operation?
    operation: str  # "retrieved", "reasoned", "concluded"
    
    # What was the output?
    output_state: str
    
    # Supporting evidence
    evidence_used: list[str]  # Evidence IDs
    
    # Alternative considered?
    alternatives_considered: list[str]

@dataclass
class EvidenceItem:
    """Evidence used in the reasoning."""
    
    id: str
    source_type: SourceType  # KNOWLEDGE, ENTITY, TOOL, USER
    source_id: str
    
    # Content
    content: str
    relevance_score: float
    
    # Citation
    citation_text: str
    citation_url: str | None
    
    # User can verify?
    verifiable: bool
    verification_instructions: str | None
```

**Explanation Generator:**

```python
class ExplainabilityEngine:
    """Generates explanations for AI responses."""
    
    async def explain(
        self,
        reasoning_trace: ReasoningResult,
        confidence: ConfidenceScore,
        evidence: list[Evidence],
        context: ExplanationContext,
    ) -> Explanation:
        """
        Generate a complete explanation.
        """
        # 1. Parse reasoning trace
        steps = self.parse_reasoning_steps(reasoning_trace)
        
        # 2. Link evidence to steps
        evidence_items = await self.link_evidence(steps, evidence)
        
        # 3. Generate summaries
        reasoning_summary = self.summarize_reasoning(steps)
        evidence_summary = self.summarize_evidence(evidence_items)
        
        # 4. Generate alternatives
        alternatives = await self.generate_alternatives(
            reasoning_trace,
            context,
        )
        
        # 5. Generate user summary
        user_summary = await self.generate_user_summary(
            steps,
            evidence_items,
            confidence,
            context.user_role,
        )
        
        return Explanation(
            query=context.query,
            query_summary=self.summarize_query(context.query),
            reasoning_steps=steps,
            reasoning_summary=reasoning_summary,
            evidence=evidence_items,
            evidence_summary=evidence_summary,
            confidence=confidence,
            confidence_explanation=self.explain_confidence(confidence),
            alternatives=alternatives,
            alternative_summary=self.summarize_alternatives(alternatives),
            user_summary=user_summary,
            technical_details=self.generate_technical_details(steps, evidence),
            generated_at=datetime.utcnow(),
            complexity_level=self.assess_complexity(steps),
        )
    
    async def generate_user_summary(
        self,
        steps: list[ReasoningStep],
        evidence: list[EvidenceItem],
        confidence: ConfidenceScore,
        user_role: UserRole,
    ) -> str:
        """Generate user-friendly explanation."""
        
        # Adapt complexity to user role
        if user_role == UserRole.ENGINEER:
            return self.engineer_summary(steps, evidence, confidence)
        elif user_role == UserRole.MANAGER:
            return self.manager_summary(steps, evidence, confidence)
        elif user_role == UserRole.EXECUTIVE:
            return self.executive_summary(steps, evidence, confidence)
        else:
            return self.patient_summary(steps, evidence, confidence)
    
    def engineer_summary(
        self,
        steps: list[ReasoningStep],
        evidence: list[EvidenceItem],
        confidence: ConfidenceScore,
    ) -> str:
        """
        Generate explanation for biomedical engineers.
        Technical but not overwhelming.
        """
        lines = []
        
        # Overview
        lines.append(f"## Análisis\n")
        lines.append(f"He analizado tu consulta considerando {len(evidence)} fuentes de evidencia.")
        
        # Key findings
        lines.append(f"\n## Hallazgos Principales\n")
        for i, step in enumerate(steps[:3]):  # Top 3
            lines.append(f"{i+1}. {step.description}")
        
        # Evidence
        lines.append(f"\n## Evidencia\n")
        lines.append(f"Esta respuesta se basa en:\n")
        for e in evidence[:5]:  # Top 5
            lines.append(f"- {e.citation_text} (relevancia: {e.relevance_score:.0%})")
        
        # Confidence
        lines.append(f"\n## Confianza\n")
        lines.append(f"Nivel de confianza: {confidence.level.value} ({confidence.overall:.0%})")
        if confidence.low_evidence:
            lines.append(f"\n⚠️ Nota: La evidencia disponible es limitada.")
        
        return "\n".join(lines)
    
    def executive_summary(
        self,
        steps: list[ReasoningStep],
        evidence: list[EvidenceItem],
        confidence: ConfidenceScore,
    ) -> str:
        """
        Generate high-level summary for executives.
        Minimal technical detail.
        """
        lines = []
        
        # One-liner
        lines.append(f"## Resumen\n")
        if confidence.level == ConfidenceLevel.HIGH:
            lines.append(f"✅ Recomendación de alta confianza basada en evidencia sólida.")
        elif confidence.level == ConfidenceLevel.MEDIUM:
            lines.append(f"⚡ Recomendación razonable, con algunas áreas de incertidumbre.")
        elif confidence.level == ConfidenceLevel.LOW:
            lines.append(f"⚠️ Recomendación con incertidumbre significativa.")
        else:
            lines.append(f"❌ No tenemos suficiente información para una recomendación segura.")
        
        # Key point
        if steps:
            lines.append(f"\n{steps[0].description}")
        
        # Recommendation
        lines.append(f"\n## Recomendación\n")
        if confidence.level.value in ["HIGH", "MEDIUM"]:
            lines.append(f"La acción sugerida tiene un {confidence.overall:.0%} de confianza.")
        else:
            lines.append(f"Se recomienda revisión por un especialista antes de actuar.")
        
        return "\n".join(lines)
```

**Structured Output:**

```python
class ExplanationFormatter:
    """Formats explanations for different outputs."""
    
    def to_markdown(self, explanation: Explanation) -> str:
        """Format as Markdown."""
        ...
    
    def to_html(self, explanation: Explanation) -> str:
        """Format as HTML with styling."""
        ...
    
    def to_json(self, explanation: Explanation) -> dict:
        """Format as JSON for API responses."""
        return {
            "query": explanation.query_summary,
            "reasoning": {
                "summary": explanation.reasoning_summary,
                "steps": [
                    {
                        "number": s.step_number,
                        "description": s.description,
                        "operation": s.operation,
                    }
                    for s in explanation.reasoning_steps
                ],
            },
            "evidence": {
                "summary": explanation.evidence_summary,
                "items": [
                    {
                        "citation": e.citation_text,
                        "relevance": e.relevance_score,
                        "verifiable": e.verifiable,
                    }
                    for e in explanation.evidence
                ],
            },
            "confidence": {
                "overall": explanation.confidence.overall,
                "level": explanation.confidence.level.value,
                "explanation": explanation.confidence_explanation,
            },
            "alternatives": [
                {
                    "description": a.description,
                    "reason": a.reason,
                    "confidence": a.confidence,
                }
                for a in explanation.alternatives
            ],
            "user_summary": explanation.user_summary,
        }
    
    def to_a11y(self, explanation: Explanation) -> str:
        """Format for screen readers (accessibility)."""
        lines = []
        lines.append(f"Explicación de la respuesta.")
        lines.append(f"{explanation.user_summary}")
        lines.append(f"Fuente de información: {explanation.evidence_summary}")
        lines.append(f"Nivel de confianza: {explanation.confidence.level.value}")
        return " ".join(lines)
```

## User Role Adaptation

| Role | Complexity | Format | Depth |
|------|------------|--------|-------|
| Engineer | Medium | Markdown/HTML | Technical |
| Manager | Low | Markdown/HTML | Summary |
| Executive | Very Low | Markdown | One-liner |
| Patient | Low | Plain text | Simple |
| System | High | JSON | Full trace |

## Reasons

**1. EREN Philosophy:**
- "EREN always explains"
- Trust through transparency
- Professional verification

**2. Compliance:**
- Audit requirements
- Decision traceability
- Documentation for regulators

**3. Learning:**
- Users learn from explanations
- Improves AI literacy
- Reduces misuse

---

## Consequences

**Positive:**
- Trust through transparency
- Professional acceptance
- Compliance-ready
- Learning opportunity

**Negative:**
- Explanations add latency
- Complex to generate good ones
- May reveal too much (IP concerns)
