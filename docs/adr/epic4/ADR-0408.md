# ADR-0408: Tool Orchestration

**Status:** Proposed

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EREN necesita tools para interactuar con el mundo real:
- **Query tools** — Buscar devices, incidents, knowledge
- **Action tools** — Crear incidents, update devices, schedule maintenance
- **External tools** — FHIR queries, weather, calendar

Epic 0 define Tool Use como capability. Este ADR define la arquitectura de tools.

---

```python
class Tool(Protocol):
    """Base contract for all tools."""
    
    @property
    def name(self) -> str:
        """Tool identifier."""
        ...
    
    @property

        """Human-readable description for LLM."""
        ...

    def parameters(self) -> list[ToolParameter]:
        """Parameter schema (JSON Schema)."""
        ...
    
    @property
    def returns(self) -> ToolReturn:
        """Return type description."""
        ...
    
    @property
    def requires_approval(self) -> bool:
        """Does this tool need human approval?"""
        ...

        self,
        parameters: dict,

    ) -> ToolResult:
```

**Example Tools:**

```python
# Query Tool - Device Lookup
class DeviceLookupTool(Tool):
    name = "device_lookup"
    description = """
    Look up biomedical device information by device ID or serial number.
    Returns device details including status, location, maintenance history.
    """
    requires_approval = False
    
    parameters = [
        ToolParameter(
            name="device_id",
            type="string",
            description="Device ID (UUID)",
            required=True,
        ),
    ]
    
    returns = ToolReturn(
        type="object",
        description="Device details including status, location, last calibration",
    )
    
    async def execute(self, parameters: dict, context: ExecutionContext) -> ToolResult:
        device_id = parameters["device_id"]
        
        device = await self.device_repository.get(DeviceId(device_id))
        
        if not device:
            return ToolResult.error(f"Device not found: {device_id}")
        
        return ToolResult.success({
            "id": str(device.id),
            "serial": device.serial_number,
            "status": device.status.value,
            "location": device.location_summary,
            "last_calibration": device.last_calibration_date,
            "next_calibration": device.next_calibration_date,
        })

# Action Tool - Create Incident
class CreateIncidentTool(Tool):
    name = "create_incident"
    description = """
    Create a new engineering incident in the system.
    Requires: device_id, description, priority.
    """
    requires_approval = True  # Safety: actions need approval
    
    parameters = [
        ToolParameter(name="device_id", type="string", required=True),
        ToolParameter(name="description", type="string", required=True),
        ToolParameter(
            name="priority", 
            type="enum", 
            enum_values=["low", "medium", "high", "critical"],
            required=True,
        ),
        ToolParameter(name="reporter_notes", type="string", required=False),
    ]
    
    returns = ToolReturn(
        type="object",
        description="Created incident ID and status",
    )
    
    async def execute(self, parameters: dict, context: ExecutionContext) -> ToolResult:
        # Check approval
        if context.requires_approval and not context.approval:
            return ToolResult.pending_approval(
                f"Create incident for device {parameters['device_id']}"
            )
        
        incident = EngineeringIncident.create(
            device_id=DeviceId(parameters["device_id"]),
            description=parameters["description"],
            priority=Priority(parameters["priority"]),
            reported_by=context.user_id,
            notes=parameters.get("reporter_notes"),
        )
        
        await self.incident_repository.save(incident)
        
        return ToolResult.success({
            "incident_id": str(incident.id),
            "status": incident.status.value,
        })
```

**Tool Orchestrator:**

```python
class ToolOrchestrator:
    """Manages tool lifecycle and execution."""
    
    def __init__(
        self,
        tool_registry: ToolRegistry,
        execution_engine: ExecutionEngine,
        approval_service: ApprovalService,
    ):
        ...
    
    async def execute_tool(
        self,
        tool_call: ToolCall,
        context: ExecutionContext,
    ) -> ToolResult:
        """Execute a single tool call."""
        # 1. Validate tool exists
        tool = self.tool_registry.get(tool_call.name)
        if not tool:
            return ToolResult.error(f"Unknown tool: {tool_call.name}")
        
        # 2. Validate parameters
        validation = tool.validate_parameters(tool_call.parameters)
        if not validation.valid:
            return ToolResult.error(f"Invalid parameters: {validation.errors}")
        
        # 3. Check approval for action tools
        if tool.requires_approval:
            approved = await self.approval_service.check(
                user_id=context.user_id,
                action=tool.name,
                parameters=tool_call.parameters,
            )
            if not approved:
                return ToolResult.pending_approval(
                    f"Approval required for {tool.name}"
                )
        
        # 4. Execute
        try:
            result = await tool.execute(tool_call.parameters, context)
            await self.audit_logger.log_tool_execution(tool.name, result)
            return result
        except Exception as e:
            return ToolResult.error(f"Tool execution failed: {e}")
    
    async def execute_plan(
        self,
        plan: ExecutionPlan,
        context: ExecutionContext,
    ) -> list[ToolResult]:
        """Execute a plan with multiple tool calls."""
        results = []
        
        for step in plan.steps:
            result = await self.execute_tool(step.tool_call, context)
            results.append(result)
            
            # Stop on failure (unless plan allows recovery)
            if not result.success and not plan.continue_on_error:
                break
        
        return results
```

## Reasons

**1. Structured tool use:**
- Clear contract for all tools
- LLM knows capabilities
- Consistent error handling

**2. Safety:**
- Action tools require approval
- Audit logging
- Rate limiting

**3. Extensibility:**
- Easy to add new tools
- Tool registry pattern
- Discovery mechanism

---

## Consequences

**Positive:**
- Powerful agentic capabilities
- Safe action execution
- Easy to extend
- Good observability

**Negative:**
- Tool execution adds latency
- More complex than stateless
- Approval UX challenge
