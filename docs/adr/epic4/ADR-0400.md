# ADR-0400: AI Core Architecture

**Status:** Proposed

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EREN necesita un motor de inteligencia artificial que permita razonar, aprender y explicar decisiones clínicas. Este motor debe ser:

1. **Confiable** — Respaldado por evidencia, no fabricate información
2. **Explicable** — El usuario siempre sabe por qué EREN sugiere algo
3. **Seguro** — Ningún output debe dañar al paciente
4. **Escalable** — Debe manejar múltiples conversaciones simultáneas
5. **Flexible** — Debe soportar múltiples LLMs

Epic 0 estableció el Cognitive Model como base. Este ADR define la arquitectura de implementación.

---

## Decision

**Epic 4 implementa el AI Core como un Cognitive Runtime con 6 capas:**

```
┌─────────────────────────────────────────┐
│         CONVERSATION LAYER               │
│   Controller | Composer | Explainability │
├─────────────────────────────────────────┤
│              RAG LAYER                   │
│    Knowledge Retriever | Orchestrator    │
├─────────────────────────────────────────┤
│           REASONING LAYER                │
│  Reasoning Engine | Confidence | Feedback│
├─────────────────────────────────────────┤
│            CONTEXT LAYER                 │
│      Context Builder | Prompt Builder    │
├─────────────────────────────────────────┤
│            MEMORY LAYER                  │
│           Memory Manager                 │
├─────────────────────────────────────────┤
│            TOOLS LAYER                   │
│          Tool Orchestrator               │
├─────────────────────────────────────────┤
│            SAFETY LAYER                   │
│           Safety Engine                  │
└─────────────────────────────────────────┘
```

**Flujo de procesamiento:**

```
User Message
    ↓
Conversation Controller
    ↓
Safety Engine (pre-check)
    ↓
Context Builder (agrega contexto)
    ↓
RAG Orchestrator (recupera evidencia)
    ↓
Prompt Builder (construye prompt)
    ↓
Reasoning Engine (invoca LLM)
    ↓
Confidence Engine (calcula confianza)
    ↓
Safety Engine (post-check)
    ↓
Explainability Engine (genera explicación)
    ↓
Response Composer (formatea respuesta)
    ↓
Response + Evidence
```

---

## Reasons

**1. Separación de responsabilidades:**
- Cada capa tiene un propósito claro
- Fácil de probar, mantener y extender
- Permite desarrollo paralelo

**2. Cognitive Model alignment:**
- Sigue el modelo cognitivo de Epic 0
- Soporta todos los cognitive capabilities

**3. Principios de EREN:**
- Safety primero (Safety Engine en pre y post)
- Evidence-based (RAG obligatorio)
- Explainable (Explainability Engine dedicado)

**4. Flexibility:**
- LLM Provider abstraction permite swap de modelos
- Tool abstraction permite agregar herramientas
- Memory abstraction permite diferentes backends

---

## Consequences

**Positive:**
- Arquitectura clara y mantenible
- Separación de concerns
- Fácil de agregar nuevos LLMs o tools
- Testing independiente por capa

**Negative:**
- Más complejo que un simple wrapper de LLM
- Latencia adicional por capas
- Más puntos de failure que requieren manejo
