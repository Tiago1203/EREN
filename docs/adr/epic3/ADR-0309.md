# ADR-0309: Department Hierarchy Model

**Status:** Proposed

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

This ADR details the Department hierarchy implementation, complementing ADR-0052 (Department Hierarchy Model) from Epic 0.

Departments are the organizational units of a hospital and cross-cut the physical hierarchy.

---

## Decision

**Department Hierarchy:**

```
Organization
└── Department Group (grouping: "Critical Care Services", "Imaging Services")
    └── Department
        └── Unit (optional — for large departments)
```

**Key rules:**

1. **Department Group:** Groups related departments for reporting and administration (e.g., "Surgical Services" groups General Surgery, Orthopedics, Neurosurgery)
2. **Department:** The operational unit (ICU, Emergency, Radiology, Biomedical Engineering)
3. **Unit:** Optional sub-division for large departments (e.g., ICU-North, ICU-South)
4. **Department ↔ Physical:** Independent hierarchies linked via `DepartmentAssignment` and `UnitAssignment`

**Data model:**

```sql
hospital.department_group (
    tenant_id UUID PK,
    department_group_id UUID PK,
    organization_id UUID FK,
    group_name TEXT,
    group_type TEXT,
    ...
)

hospital.department (
    tenant_id UUID PK,
    department_id UUID PK,
    organization_id UUID FK,
    parent_department_id UUID FK (nullable — for hierarchy),
    department_group_id UUID FK,
    department_code TEXT,
    department_name TEXT,
    department_type TEXT,
    ...
)

hospital.unit (
    tenant_id UUID PK,
    unit_id UUID PK,
    organization_id UUID FK,
    department_id UUID FK,
    unit_code TEXT,
    unit_name TEXT,
    ...
)

hospital.department_assignment (
    tenant_id UUID PK,
    assignment_id UUID PK,
    staff_id UUID FK (from Staffing Context),
    department_id UUID FK,
    unit_id UUID FK (nullable),
    assignment_type TEXT,
    start_date DATE,
    end_date DATE (nullable),
    ...
)
```

---

## Consequences

**Positive:**
- Flexible hierarchy (departments can be nested up to 3 levels)
- Department assignments link staff to organizational structure
- Independent of physical hierarchy — same staff can work in multiple departments

**Negative:**
- Staff with multiple department assignments require careful permission checking
- Department restructuring cascades to assignments
