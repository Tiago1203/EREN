# ADR-0307: Inventory Bounded Context

**Status:** Proposed

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

The Inventory Context manages spare parts, consumables, warehouses, purchase orders, and suppliers. It supports both planned maintenance (from Maintenance Context) and reactive repairs (from Incident Context) by ensuring parts are available when needed.

---

## Decision

**Aggregate: SparePart** (aggregate root)

```
SparePart (aggregate root)
├── tenant_id, spare_part_id
├── part_number: string (manufacturer part number)
├── part_name: string
├── part_description: string
├── category: enum (electrical, mechanical, optical, fluid, consumable, accessory)
├── manufacturer_id (FK — from Asset Context)
├── unit_of_measure: enum (piece, meter, liter, kg, kit)
├── unit_cost: decimal
├── reorder_point: integer (minimum stock level)
├── reorder_quantity: integer
├── current_stock: integer
├── warehouse_id (FK)
├── storage_location: string (e.g., "Shelf A-3")
├── lot_tracking_enabled: boolean
├── expiry_tracking_enabled: boolean
├── shelf_life_days: integer (nullable)
└── status: enum (active, discontinued, recalled)

Warehouse
├── tenant_id, warehouse_id
├── warehouse_code: string
├── warehouse_name: string
├── warehouse_type: enum (main, satellite, on_site)
├── address: Address
├── capacity_sqft: integer
└── status: enum (active, inactive, full)

PurchaseOrder
├── tenant_id, purchase_order_id
├── po_number: string (auto-generated)
├── supplier_id (FK)
├── warehouse_id (FK — destination)
├── order_date: date
├── expected_delivery_date: date
├── status: enum (draft, submitted, approved, partially_received, received, cancelled)
├── total_value: decimal
├── approval_required: boolean
├── approved_by_staff_id (FK — nullable)
├── approved_at: datetime (nullable)
└── notes: string

PurchaseOrderLineItem
├── tenant_id, line_item_id, purchase_order_id (FK)
├── spare_part_id (FK)
├── quantity_ordered: integer
├── quantity_received: integer
├── unit_cost: decimal
├── line_total: decimal
└── status: enum (pending, partial, fulfilled, cancelled)

Supplier
├── tenant_id, supplier_id
├── supplier_name: string
├── supplier_code: string
├── supplier_type: enum (manufacturer, distributor, direct)
├── tax_id: string
├── address: Address
├── contact_person: string
├── contact_email: string
├── contact_phone: string
├── payment_terms: string
├── lead_time_days: integer
├── minimum_order_value: decimal
└── status: enum (active, on_hold, discontinued)
```

**Invariants:**
- `current_stock` ≥ 0 always
- `reorder_point` > 0 if reorder enabled
- PurchaseOrder total = sum of line_item totals
- `quantity_received` ≤ `quantity_ordered` for each line item
- Supplier must be active to create PurchaseOrder

**Domain Events:**
- `SparePartAdded`, `SparePartConsumed`, `SparePartLowStock`, `SparePartOutOfStock`
- `SparePartRecalled`, `SparePartExpired`
- `WarehouseConfigured`, `WarehouseCapacityAlert`
- `PurchaseOrderCreated`, `PurchaseOrderApproved`, `PurchaseOrderReceived`, `PurchaseOrderCancelled`
- `SupplierRegistered`, `SupplierOnHold`

---

## Consequences

**Positive:**
- SparePart is aggregate root — inventory centered on part lifecycle
- Low stock alerts trigger reorder workflow
- Links to Asset Context (parts for assets) and Maintenance Context (parts used in maintenance)
- Supplier lead time used in reorder calculations

**Negative:**
- Stock consumption during reactive repairs (from Incident Context) requires event subscription
- Multi-warehouse tracking adds complexity
