# ADR-0305: Asset Management Bounded Context

**Status:** Proposed

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

Asset Management tracks the physical medical equipment lifecycle: acquisition, vendor relationships, contracts, warranties, and deprecation. It complements the Device Context (from EPIC 2) by managing the business side of equipment — who bought it, under what contract, with what warranty.

---

## Decision

**Aggregate: Asset** (aggregate root)

```
Asset (aggregate root)
├── tenant_id, asset_id
├── asset_tag: string (hospital inventory number)
├── device_id (FK — links to Device Context from EPIC 2)
├── manufacturer_id (FK)
├── vendor_id (FK)
├── acquisition_date: date
├── acquisition_cost: decimal
├── current_value: decimal
├── depreciation_method: enum (straight_line, declining_balance, units_of_production)
├── useful_life_years: integer
├── status: enum (active, in_storage, under_maintenance, decommissioned, disposed)
├── location_id: room_id (FK — from Capacity Context)
├── department_id (FK — owning department)
└── notes: string

Manufacturer
├── tenant_id, manufacturer_id
├── manufacturer_name: string
├── manufacturer_code: string
├── country_of_origin: string
├── website: string
└── support_contact: ContactInfo

Vendor
├── tenant_id, vendor_id
├── vendor_name: string
├── vendor_type: enum (distributor, manufacturer_direct, service_provider, third_party)
├── tax_id: string
├── address: Address
└── credit_limit: decimal

Contract
├── tenant_id, contract_id, asset_id (FK), vendor_id (FK)
├── contract_number: string
├── contract_type: enum (purchase, service, lease, rental, maintenance_agreement)
├── start_date: date
├── end_date: date
├── total_value: decimal
├── payment_terms: string
├── auto_renew: boolean
├── renewal_terms: string
└── status: enum (active, expiring_soon, expired, terminated)

Warranty
├── tenant_id, warranty_id, asset_id (FK)
├── warranty_type: enum (manufacturer, extended, service)
├── start_date: date
├── end_date: date
├── coverage_details: string
├── warranty_provider: string
└── claim_contact: string
```

**Invariants:**
- An Asset must have exactly one Manufacturer and one Vendor
- An Asset may have zero or one active Warranty
- An Asset may have zero or more Contracts
- `acquisition_date` must be in the past or today
- `current_value` ≤ `acquisition_cost` (depreciation)
- `warranty.end_date` must be after `warranty.start_date`
- `contract.end_date` must be after `contract.start_date`

**Domain Events:**
- `AssetRegistered`, `AssetUpdated`, `AssetAssigned`, `AssetTransferred`, `AssetRetired`, `AssetDisposed`
- `ManufacturerRegistered`, `VendorRegistered`, `VendorCreditLimitChanged`
- `ContractSigned`, `ContractRenewed`, `ContractExpiring`, `ContractTerminated`
- `WarrantyActivated`, `WarrantyExpired`, `WarrantyClaimFiled`

---

## Consequences

**Positive:**
- Links to Device Context (EPIC 2) via device_id — unified device + asset view
- Contract and warranty tracking prevents expired coverage
- Vendor credit limits prevent over-ordering
- Depreciation tracking for financial reporting

**Negative:**
- Dual tracking (Device + Asset) requires careful synchronization
- Asset decommissioning must update Device Context
