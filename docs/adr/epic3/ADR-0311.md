# ADR-0311: Multi-Tenant Data Isolation

**Status:** Proposed

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

All Hospital domain entities must be multi-tenant. This ADR defines how data isolation is enforced across all 6 EPIC 3 bounded contexts, building on the multi-tenancy strategy from Epic 0.

---

## Decision

**Multi-tenancy enforcement strategy: Shared Database + RLS + Application**

**1. Database level (PostgreSQL RLS):**

All hospital tables include `tenant_id` as the first column of the primary key and in RLS policies:

```sql
-- Example: hospital.bed table
CREATE TABLE hospital.bed (
    tenant_id UUID NOT NULL,
    campus_id UUID NOT NULL,
    building_id UUID NOT NULL,
    floor_id UUID NOT NULL,
    room_id UUID NOT NULL,
    bed_id UUID NOT NULL,
    bed_number TEXT NOT NULL,
    bed_type TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'available',
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (tenant_id, campus_id, building_id, floor_id, room_id, bed_id),
    FOREIGN KEY (tenant_id) REFERENCES tenants(tenant_id),
    FOREIGN KEY (tenant_id, campus_id, building_id, floor_id, room_id)
        REFERENCES hospital.room(tenant_id, campus_id, building_id, floor_id, room_id)
);

ALTER TABLE hospital.bed ENABLE ROW LEVEL SECURITY;

CREATE POLICY bed_tenant_isolation ON hospital.bed
    FOR ALL
    USING (tenant_id = current_setting('app.tenant_id')::UUID)
    WITH CHECK (tenant_id = current_setting('app.tenant_id')::UUID);
```

**2. Application level (FastAPI middleware):**

```python
# apps/api/app/middleware/request_context.py
def get_tenant_id() -> UUID:
    tenant_id = request.headers.get("X-Tenant-ID")
    if not tenant_id:
        raise HTTPException(401, "X-Tenant-ID header required")
    return UUID(tenant_id)

# Usage in repository
async def get_bed(tenant_id: UUID, bed_id: UUID) -> Bed:
    await session.execute(
        text("SET LOCAL app.tenant_id = :tenant_id"),
        {"tenant_id": str(tenant_id)}
    )
    # RLS enforces tenant isolation automatically
```

**3. Every repository method:**

```python
class BedRepository:
    async def get(self, tenant_id: UUID, bed_id: UUID) -> Bed | None:
        # tenant_id is ALWAYS the first filter
        result = await self.session.execute(
            select(BedModel).where(
                BedModel.tenant_id == tenant_id,
                BedModel.bed_id == bed_id
            )
        )
        return result.scalar_one_or_none()
```

## Tenant Isolation Matrix

| Context | Entity | tenant_id | Additional Scope |
|---------|--------|-----------|-----------------|
| Capacity | Bed, Room, Floor, Building, Campus | tenant_id | campus_id |
| Capacity | OccupancyRecord | tenant_id | bed_id, recorded_at |
| Staffing | Staff, Role, Team, Shift | tenant_id | — |
| Staffing | Schedule | tenant_id | staff_id, shift_id |
| Organization | Hospital, Organization, Campus | tenant_id | — |
| Department | Department, DepartmentGroup, Unit | tenant_id | organization_id |
| Department | DepartmentAssignment | tenant_id | staff_id, department_id |
| Asset | Asset, Manufacturer, Vendor | tenant_id | — |
| Asset | Contract, Warranty | tenant_id | asset_id, vendor_id |
| Inventory | SparePart, Warehouse, Supplier | tenant_id | warehouse_id |
| Inventory | PurchaseOrder, LineItem | tenant_id | supplier_id, warehouse_id |

## Guardrail

**G2.2 from EREN_ARCHITECTURAL_GUARDRAILS.md:**
> Every API endpoint MUST require and validate `X-Tenant-ID` header. Every database query MUST filter by `tenant_id`.

---

## Consequences

**Positive:**
- PostgreSQL RLS provides defense-in-depth
- Application-level enforcement catches errors before RLS
- Cross-context queries always include tenant_id
- Compatible with existing EPIC 1 migration 007 (RLS enforcement)

**Negative:**
- Composite primary keys (5-6 columns for physical hierarchy) add complexity
- tenant_id must be propagated through all domain events
- Development/testing requires tenant context management
