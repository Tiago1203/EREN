# ADR-0601: FHIR R4 Integration

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

FHIR R4 is the standard for healthcare data exchange. EREN must support FHIR for interoperability with EHR systems.

## Decision

We will implement **FHIR R4 client and server** capabilities:

### Supported Resources

| Resource | Direction | Use Case |
|----------|-----------|----------|
| Device | Bidirectional | Device inventory sync |
| Patient | Read | Patient context |
| Location | Read | Department mapping |
| Organization | Read | Hospital structure |
| Procedure | Write | Maintenance records |
| Observation | Write | Device telemetry |

### Implementation

```python
class FHIRClient:
    async def get_device(self, device_id: str) -> Device:
        bundle = await self.search("Device", identifier=device_id)
        return bundle.entry[0].resource
    
    async def create_procedure(self, maintenance: Maintenance) -> Procedure:
        return await self.create("Procedure", self.map_to_fhir(maintenance))
```

## Consequences

### Positive
- EHR interoperability
- Standard compliance
- Data portability

### Negative
- FHIR complexity
- Version compatibility
