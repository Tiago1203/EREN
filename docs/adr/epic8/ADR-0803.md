# ADR-0803: RBAC Implementation

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Role-Based Access Control (RBAC) ensures proper authorization across EREN features.

## Decision

We will implement **Hierarchical RBAC**:

### Roles

| Role | Level | Permissions |
|------|-------|-------------|
| Super Admin | 1 | Full system access |
| Hospital Admin | 2 | Hospital-wide access |
| Department Manager | 3 | Department access |
| Engineer | 4 | Device/incident management |
| Viewer | 5 | Read-only access |

### Permissions Matrix

| Resource | Super Admin | Hospital Admin | Manager | Engineer | Viewer |
|----------|-------------|----------------|---------|----------|--------|
| Devices | CRUD | CRUD | RU | RU | R |
| Incidents | CRUD | CRUD | CRU | CRU | R |
| Reports | CRUD | CRUD | RU | R | R |
| Settings | CRUD | U | - | - | - |

### Implementation

```python
class RBACMiddleware:
    async def authorize(self, user: User, action: str, resource: str) -> bool:
        role = await self.get_role(user)
        permissions = ROLE_PERMISSIONS.get(role, [])
        return f"{resource}:{action}" in permissions
```
