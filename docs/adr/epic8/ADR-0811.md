# ADR-0811: High Availability Design

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

High availability ensures EREN remains operational during component failures.

## Decision

We will implement **99.99% Uptime SLA**:

### HA Components

| Component | HA Strategy | RTO | RPO |
|-----------|-------------|-----|-----|
| API | Horizontal scaling | 0 | 0 |
| Database | PgBouncer + Patroni | < 30s | < 5min |
| Cache | Redis Cluster | 0 | < 1min |
| Queue | RabbitMQ Cluster | < 1min | < 5min |
| Storage | S3 Multi-AZ | 0 | 0 |

### Availability Targets

| Tier | Availability | Downtime/Year |
|------|--------------|---------------|
| 99.99% | Four nines | 52 min |
| 99.9% | Three nines | 8.7 hours |
| 99% | Two nines | 87.6 hours |

### Resilience Patterns

```python
PATTERNS = {
    "circuit_breaker": {
        "failure_threshold": 5,
        "timeout": 30,
    },
    "retry": {
        "max_attempts": 3,
        "backoff": "exponential",
    },
    "bulkhead": {
        "max_concurrent": 100,
    },
}
```
