## Purpose

This document defines the single, authoritative terminology for EREN. Every team member, every document, and every piece of code must use these terms consistently.

**Rule:** When in doubt, refer to this document. If a term is missing, propose an addition via ADR.

---

## Clinical Domain Terms

### Patient

**Definition:** An individual who receives healthcare services in the hospital.

```
✅ CORRECT:
patient = Patient(id="123", name="John Smith")

❌ WRONG:
patient_record = ...  # Use "Patient"
patient_model = ...   # Use "Patient"
patient_entity = ...  # Use "Patient"
client = ...          # Do not use in clinical context
```

**Key Attributes:**
- `patient_id` (UUID)
- `mrn` (Medical Record Number)
- `demographics`
- `medical_history`

---

### Diagnosis

**Definition:** A clinical judgment about a patient's health condition, coded using ICD-10.

```
✅ CORRECT:
diagnosis = Diagnosis(code="J18.9", patient_id="123")

❌ WRONG:
diagnostic = ...      # Use "Diagnosis"
diagnosis_record = ... # Use "Diagnosis"
```

---

### Treatment

**Definition:** An intervention designed to address a diagnosed condition.

```
✅ CORRECT:
treatment = Treatment(patient_id="123", diagnosis_id="456", type="medication")

❌ WRONG:
therapy = ...          # Only use for specific therapies
prescription = ...     # Use for medications specifically
```

---

### Vital Signs

**Definition:** Objective measurements of patient's physiological status.

```
✅ CORRECT:
vital_signs = VitalSigns(patient_id="123", heart_rate=72, bp="120/80")

❌ WRONG:
vitals_reading = ...  # Use "VitalSigns"
vitals_data = ...      # Use "VitalSigns"
```

---

### Clinical Alert

**Definition:** A notification requiring clinical attention, generated by CDS or device.

```
✅ CORRECT:
alert = ClinicalAlert(type="sepsis_risk", patient_id="123", severity="high")

❌ WRONG:
alarm = ...            # Use for devices only
notification = ...     # Too generic
warning = ...         # Use for severity levels
```

## Biomedical Domain Terms

### Device (Medical Device)

**Definition:** A medical equipment item with specific functionality, registered and tracked.

```
✅ CORRECT:
device = Device(id="M-001", type="monitor", manufacturer="Philips")

❌ WRONG:
equipment = ...        # Too generic
asset = ...           # Use "Asset" for inventory tracking
machine = ...         # Avoid in biomedical context
```

---

### Alarm (Device Alarm)

**Definition:** A signal generated by a medical device indicating a condition requiring attention.

```
✅ CORRECT:
alarm = DeviceAlarm(device_id="M-001", severity="critical", patient_id="123")

❌ WRONG:
alert = ...            # Use for clinical alerts
beep = ...            # Informal
```

---

### Calibration

**Definition:** Verification that a device measures accurately against a standard.

```
✅ CORRECT:
calibration = Calibration(device_id="M-001", standard="NIST", result="pass")

❌ WRONG:
calibration_record = ... # Use "Calibration"
cal_check = ...         # Use "Calibration"
```

---

### Maintenance

**Definition:** Work performed to keep medical equipment operational.

```
✅ CORRECT:
maintenance = Maintenance(device_id="M-001", type="preventive", performed_by="BMET-01")

❌ WRONG:
repair = ...            # Use "Corrective Maintenance"
service = ...          # Too generic
```

---

### Work Order

**Definition:** A formal request for maintenance work on a device.

```
✅ CORRECT:
work_order = WorkOrder(device_id="M-001", type="calibration", priority="high")

❌ WRONG:
ticket = ...            # Too generic
maintenance_request = ... # Use "WorkOrder"
```

## Hospital Domain Terms

### Bed

**Definition:** A licensed inpatient location for patient care.

```
✅ CORRECT:
bed = Bed(id="ICU-204-B", unit="ICU", status="occupied")

❌ WRONG:
cot = ...              # Informal
stretcher = ...        # Not a bed
```

---

### Staff

**Definition:** Hospital personnel with assigned roles.

```
✅ CORRECT:
staff = Staff(id="RN-001", role="nurse", department="ICU")

❌ WRONG:
employee = ...         # Too generic
user = ...             # Use for authentication
personnel = ...        # Use "Staff"
```

---

### Unit

**Definition:** An organizational hospital area (ICU, Med-Surg, ED, etc.).

```
✅ CORRECT:
unit = Unit(id="ICU", name="Intensive Care Unit", floor=3)

❌ WRONG:
department = ...        # Use "Unit"
ward = ...             # Use "Unit"
area = ...             # Too generic
```

---

### Capacity

**Definition:** The ability to accept new patients, measured by bed availability.

```
✅ CORRECT:
capacity = Capacity(unit="ICU", beds_total=20, beds_available=3)

❌ WRONG:
occupancy = ...        # Related but different concept
availability = ...     # Use for specific resources
```

## Technical Terms

### Principal

**Definition:** An authenticated identity (human, device, or system) in EREN.

```
✅ CORRECT:
principal = Principal(id="user-123", type="human")

❌ WRONG:
user = ...             # Only for authenticated humans
identity = ...         # Too generic
account = ...          # Avoid
```

### Session

**Definition:** A temporary authenticated state of interaction.

```
✅ CORRECT:
session = Session(principal_id="user-123", expires_at=datetime.now())

❌ WRONG:
token = ...            # Use for authentication tokens
connection = ...       # Too generic
```

### Tenant

**Definition:** An isolated organizational unit (hospital) within EREN.

```
✅ CORRECT:
tenant = Tenant(id="hospital-001", name="General Hospital")

❌ WRONG:
organization = ...     # "Tenant" is the correct term
hospital = ...         # The hospital is a tenant
account = ...          # Use for billing
```

## CDS-Specific Terms

### Recommendation

**Definition:** A CDS output suggesting an action with evidence and confidence.

```
✅ CORRECT:
recommendation = Recommendation(
    action="order_lactate",
    confidence=0.85,
    evidence=["vital_signs", "lab_results"]
)

❌ WRONG:
suggestion = ...       # Too weak
advice = ...           # Avoid
decision = ...         # CDS recommends, humans decide
```

### Evidence

**Definition:** Information used to support a clinical recommendation.

```
✅ CORRECT:
evidence = Evidence(source="AHA Guidelines", trust=0.95, content="...")

❌ WRONG:
data = ...             # Too generic
information = ...       # Too generic
fact = ...             # Evidence implies verified
```

### Trust Score

**Definition:** An evaluation of credibility/reliability (0.0 to 1.0).

```
✅ CORRECT:
trust_score = TrustScore(level="high", score=0.92)

❌ WRONG:
confidence = ...        # Related but different
reliability = ...       # Use for devices
trustworthiness = ...    # Too informal
```

## Common Anti-Patterns (Terms to AVOID)

| ❌ Wrong | ✅ Correct |
|---------|----------|
| PatientRecord | Patient |
| PatientModel | Patient |
| PatientEntity | Patient |
| PatientDTO | Patient (API layer) |
| Diagnostic | Diagnosis |
| Therapy | Treatment (or specify type) |
| Alarm | DeviceAlarm (devices) / ClinicalAlert (clinical) |
| Machine | Device |
| Equipment | Device (or Asset for inventory) |
| User | Principal (authenticated) / Staff (hospital) |
| Account | Tenant / Principal |
| Department | Unit (clinical) / Team (operational) |
| EMR | EHR (Electronic Health Record) |
