# ADR-3001: Clinical DTO Design

**Título:** Diseño de Clinical DTOs
**Estado:** ✅ ACEPTADO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

Los DTOs (Data Transfer Objects) de Clinical Intelligence son la interfaz principal entre el sistema de IA y el resto de EREN. Deben:

1. Representar hallazgos clínicos de manera estructurada
2. Incluir metadatos de confianza y evidencia
3. Ser serializables para APIs
4. Ser inmutables una vez creados

---

## Decisión

### Principios de Diseño

1. **Inmutabilidad**: Usar `@dataclass(frozen=True)`
2. **Tipado Estricto**: Todos los campos tienen tipos definidos
3. **Valores por Defecto**: Campos opcionales tienen defaults
4. **Serialización**: Soporte para `to_dict()` y `from_dict()`

### DTOs Principales

```python
@dataclass(frozen=True)
class ClinicalFinding:
    """Hallazgo clínico."""
    finding_id: str
    concept_id: str                    # SNOMED, ICD-10, LOINC
    description: str
    evidence_level: EvidenceLevel
    source: EvidenceSource
    confidence: float                 # 0.0 - 1.0
    patient_context: dict
    clinical_significance: ClinicalSignificance
    timestamp: datetime
    
    def to_dict(self) -> dict:
        return {
            "finding_id": self.finding_id,
            "concept_id": self.concept_id,
            "description": self.description,
            "evidence_level": self.evidence_level.value,
            "source": self.source.to_dict(),
            "confidence": self.confidence,
            "clinical_significance": self.clinical_significance.value,
            "timestamp": self.timestamp.isoformat(),
        }
```

## Niveles de Significancia Clínica

```python
class ClinicalSignificance(Enum):
    """Significancia clínica de un hallazgo."""
    CRITICAL = "critical"      # Requiere acción inmediata
    HIGH = "high"              # Requiere atención urgente
    MODERATE = "moderate"      # Requiere revisión
    LOW = "low"                # Informacional
    UNKNOWN = "unknown"        # Por determinar
```

---

## Consecuencias

### Positivas
- ✅ DTOs inmutables evitan bugs de mutación
- ✅ Tipado estricto facilita IDE support
- ✅ Serialización built-in para APIs

### Negativas
- ⚠️ DTOs inmutables requieren builders para modificaciones
