# ADR-3000: Clinical Intelligence Architecture

**Título:** Arquitectura de Clinical Intelligence
**Estado:** ✅ ACEPTADO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

EREN necesita un sistema de Clinical Intelligence que permita:
- Razonamiento clínico automatizado
- Generación de recomendaciones basadas en evidencia
- Validación de decisiones médicas
- Explicabilidad de recomendaciones

Este ADR define la arquitectura base sobre la cual se construirán todos los componentes de Clinical Intelligence.

### Principios de Diseño

1. **Separación de Responsabilidades**: Cada componente tiene una responsabilidad clara
2. **Interfaces Contratadas**: Componentes se comunican via interfaces bien definidas
3. **Inmutabilidad de DTOs**: DTOs son inmutables una vez creados
4. **Traza Completa**: Cada decisión puede ser rastreada hasta su evidencia

---

## Consecuencias

### Positivas
- ✅ Arquitectura extensible y mantenible
- ✅ Fácil de testear (interfaces mockeables)
- ✅ Separación clara de concerns
- ✅ Reusabilidad de foundation

### Negativas
- ⚠️ Más abstracciones inicialmente
- ⚠️ Curva de aprendizaje para nuevos desarrolladores
