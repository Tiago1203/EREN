# ADR-3005: Validation Model Design

**Título:** Diseño de Modelos de Validación Clínica
**Estado:** ✅ ACEPTADO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

Toda recomendación clínica debe ser validada antes de presentarse al usuario. Necesitamos:

1. Pipeline de validación multi-etapa
2. Reglas de validación configurables
3. Severidad de fallos de validación
4. Capacidad de skip bajo condiciones controladas

---

## Decisión

### Validation Result

```python
@dataclass(frozen=True)
class ValidationResult:
    """Resultado de validación."""
    is_valid: bool
    validation_id: str
    rule_id: str
    severity: ValidationSeverity
    message: str
    details: dict | None
    failed_at: datetime
    can_override: bool
```

### Validation Rule

```python
@dataclass(frozen=True)
class ValidationRule:
    """Regla de validación."""
    rule_id: str
    rule_type: ValidationRuleType
    description: str
    severity: ValidationSeverity
    check_function: Callable
    enabled: bool = True
    can_skip: bool = False
    skip_conditions: list[SkipCondition] | None
```

### Severidades de Validación

```python
class ValidationSeverity(Enum):
    """Severidad de fallo de validación."""
    BLOCKING = "blocking"        # Impide la recomendación
    HIGH = "high"                // Recomendación no deseable
    MEDIUM = "medium"            # Advertencia
    LOW = "low"                  # Información
    INFO = "info"                # Para documentación
```

## Tipos de Reglas

```python
class ValidationRuleType(Enum):
    """Tipos de reglas de validación."""
    SAFETY = "safety"            # Seguridad del paciente
```
