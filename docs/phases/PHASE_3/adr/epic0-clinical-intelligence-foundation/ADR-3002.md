# ADR-3002: Evidence Model Design

**Título:** Diseño de Modelos de Evidencia
**Estado:** ✅ ACEPTADO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

La evidencia es la base de todo razonamiento clínico. Necesitamos un modelo que:

1. Represente diferentes niveles de evidencia
2. Capture la fuente de cada pieza de evidencia
3. Permita encadenar evidencia
4. Asigne pesos relativos a cada evidencia

---

## Decisión

### Jerarquía de Evidencia

```
EvidenceLevel (Enum)
├── A: Systematic reviews, RCTs de alta calidad
├── B: Estudios de cohorte, RCTs de baja calidad
├── C: Estudios de caso-control, series de casos
└── D: Opinión de expertos, investigación básica
```

### Modelo de Evidencia

```python
@dataclass(frozen=True)
class Evidence:
    """Evidencia clínica."""
    evidence_id: str
    level: EvidenceLevel
    source: EvidenceSource
    content: str
    weight: float                    # 0.0 - 1.0
    applicability: ApplicabilityScore
    quality: EvidenceQuality
    citation: Citation | None
    retrieved_at: datetime
    expires_at: datetime | None
```

### Cadena de Evidencia

```python
@dataclass(frozen=True)
class EvidenceChain:
    """Cadena de evidencia para un hallazgo."""
    chain_id: str
    primary_evidence: list[Evidence]
    supporting_evidence: list[Evidence]
    contradicting_evidence: list[Evidence]
    overall_assessment: EvidenceAssessment
    confidence_modifier: float      # Modificador basado en consistencia
```

- ✅ Expiración de evidencia antigua

### Negativas
- ⚠️ Complejidad adicional en gestión de evidencia
