# ADR-3006: Knowledge Interface Design

**Título:** Diseño de Interfaces de Conocimiento Médico
**Estado:** ✅ ACEPTADO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

El Clinical Intelligence debe interactuar con múltiples fuentes de conocimiento médico:

1. Bases de conocimiento clínico (SNOMED, LOINC, ICD)
2. Guías clínicas institucionales
3. Ontologías médicas
4. Literatura médica

Necesitamos interfaces abstractas que permitan esta interacción.

---

## Conceptos Médicos

```python
@dataclass(frozen=True)
class MedicalConcept:
    """Concepto médico."""
    concept_id: str
    code_system: CodeSystem

    definitions: list[str]
```

```python
class CodeSystem(Enum):
```

## Consecuencias

### Positivas
- ✅ Interfaces abstractas permiten múltiples implementaciones
- ✅ Fácil de testear con mocks
- ✅ Soporte para diferentes sistemas de codificación

### Negativas
- ⚠️ Abstacción puede ocultar detalles de implementación
