# ADR-3020: Reasoning Engine Architecture

**Título:** Arquitectura del Motor de Razonamiento
**Estado:** ✅ COMPLETO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

El Reasoning Engine es el "cerebro" de EREN. Necesita procesar información de múltiples fuentes (memoria, conocimiento, dominio, conversación) para generar razonamientos lógicos que produzcan hipótesis, cadenas de razonamiento, alternativas e inferencias.

El motor NO responde directamente. **Razona** para generar conocimiento accionable.

### Modelo de Datos Principal

```python
@dataclass
class ReasoningContext:
    """Contexto de razonamiento."""
    memory: MemoryContext
    knowledge: KnowledgeContext
    domain: DomainContext
    conversation: ConversationContext
    constraints: list[Constraint]

@dataclass
class ReasoningResult:
    """Resultado del razonamiento."""
    hypotheses: list[Hypothesis]
    reasoning_chains: list[ReasoningChain]
    alternatives: list[Alternative]
    inferences: list[Inference]
    confidence: ConfidenceScore
    evidence_used: list[Evidence]

class ReasoningEngine:
    """Motor de razonamiento principal."""
    
    async def reason(
        self,
        context: ReasoningContext,
        query: str,
    ) -> ReasoningResult:
        """Ejecuta el pipeline de razonamiento completo."""
        # 1. Construir contexto
        full_context = await self.context_builder.build(context)
        
        # 2. Generar hipótesis
        hypotheses = await self.hypothesis_engine.generate(full_context)
        
        # 3. Inferencia
        inferred = await self.inference_engine.infer(hypotheses, full_context)
        
        # 4. Diagnóstico
        diagnosis = await self.diagnostic_engine.diagnose(inferred, full_context)
        
        # 5. Construir cadenas
        chains = await self.chain_builder.build(hypotheses, diagnosis)
        
        # 6. Evaluar confianza
        confidence = await self.confidence_calculator.calculate(chains)
        
        return ReasoningResult(
            hypotheses=hypotheses,
            reasoning_chains=chains,
            alternatives=diagnosis.alternatives,
            inferences=inferred,
            confidence=confidence,
            evidence_used=diagnosis.evidence,
        )
```

### Pipeline de Razonamiento

```python
class ReasoningPipeline:
    """Pipeline de razonamiento con stages."""
    
    async def execute(
        self,
        input_data: ReasoningInput,
    ) -> ReasoningResult:
        
        # Stage 1: Context Preparation
        context = await self.prepare_context(input_data)
        
        # Stage 2: Hypothesis Generation
        hypotheses = await self.generate_hypotheses(context)
        
        # Stage 3: Evidence Collection
        evidence = await self.collect_evidence(hypotheses, context)
        
        # Stage 4: Hypothesis Evaluation
        evaluated = await self.evaluate_hypotheses(hypotheses, evidence)
        
        # Stage 5: Diagnostic Analysis
        diagnosis = await self.analyze_diagnosis(evaluated)
        
        # Stage 6: Chain Construction
        chains = await self.construct_chains(diagnosis)
        
        # Stage 7: Final Synthesis
        result = await self.synthesize(chains, diagnosis)
        
        return result
```

## Consecuencias

### Positivas
- ✅ Arquitectura modular y extensible
- ✅ Separación clara de responsabilidades
- ✅ Integración con EPIC 0 (Foundation) y EPIC 1 (Knowledge)
- ✅ Pipeline configurable

### Negativas
- ⚠️ Complejidad en la coordinación de componentes
- ⚠️ Requiere buen diseño de contexto
