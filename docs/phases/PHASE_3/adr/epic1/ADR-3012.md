# ADR-3012: Equipment Taxonomy Design

**Título:** Diseño de Taxonomía de Equipos Médicos
**Estado:** ✅ COMPLETO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

EREN necesita categorizar y estructurar el conocimiento sobre equipos médicos para:

1. Clasificar equipos por tipo, fabricante, uso clínico
2. Identificar modos de falla comunes por categoría
3. Definir lógicas de mantenimiento predictivo
4. Mapear equipos a estándares aplicables (IEC 60601)

---

### Clases de Riesgo

```python
class RiskClass(Enum):
    """Clase de riesgo según MDR/AAMI."""
    CLASS_I = "class_i"      # Bajo riesgo
    CLASS_IIA = "class_iia"  # Riesgo moderado
    CLASS_IIB = "class_iib"  # Riesgo alto
    CLASS_III = "class_iii"  # Riesgo crítico

class FailureSeverity(Enum):
    """Severidad de falla."""
    CRITICAL = "critical"    # Puede causar lesión o muerte
    MAJOR = "major"          # Falla funcional significativa
    MINOR = "minor"          # Impacto menor
    COSMETIC = "cosmetic"    # Solo apariencia

class MaintenanceType(Enum):
    """Tipos de mantenimiento."""
    PREVENTIVE = "preventive"      # Basado en tiempo/uso
    PREDICTIVE = "predictive"      # Basado en condición
    CORRECTIVE = "corrective"       # Reparación de falla
    CALIBRATION = "calibration"    # Verificación de precisión
    SAFETY = "safety"              # Verificación de seguridad
```

## Consecuencias

### Positivas
- ✅ Estructura jerárquica de equipos
- ✅ Identificación de modos de falla por categoría
- ✅ Programación de mantenimiento basada en categoría
- ✅ Compliance automático con estándares

### Negativas
- ⚠️ Mantenimiento de taxonomía actualizada
- ⚠️ Clasificaciones pueden variar por país
