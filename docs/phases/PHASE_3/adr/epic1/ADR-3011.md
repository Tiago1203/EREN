# ADR-3011: Medical Ontology Design

**Título:** Diseño de Ontología Médica
**Estado:** ✅ COMPLETO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

El sistema necesita integrar múltiples sistemas de codificación médica estándar para permitir búsqueda y mapeo entre:

1. SNOMED-CT (Systematized Nomenclature of Medicine)
2. ICD-10/ICD-11 (International Classification of Diseases)
3. LOINC (Logical Observation Identifiers Names and Codes)
4. RxNorm (Normalized naming of drugs)

La ontología médica debe permitir navegación jerárquica y mapeo entre sistemas.

---

## Decisión

### Arquitectura de Ontología

```
MedicalOntology
├── SNOMEDIntegration
│   ├── Concept search
│   ├── Hierarchy navigation
│   └── Relationship traversal
├── ICDIntegration
│   ├── ICD-10 mapping
│   └── ICD-11 mapping
├── LOINCIntegration
│   ├── Laboratory observations
│   └── Clinical measurements
└── HierarchyNavigator
    ├── Parent/child traversal
    ├── Sibling finding
    └── Ancestor/descendant paths
```

### Modelo de Datos

```python
@dataclass(frozen=True)
class MedicalOntologyConcept:
    """Concepto de ontología médica."""
    concept_id: str
    fully_qualified_name: str
    semantic_tag: str
    is_active: bool
    module_id: str
    definitions: list[str]
    synonyms: list[str]
    parents: list[str]
    children: list[str]
    relationships: list[OntologyRelationship]

@dataclass(frozen=True)
class OntologyRelationship:
    """Relación entre conceptos."""
    type_id: str
    target_concept_id: str
    relationship_group: str | None
    characteristic_type: str
    modifier: str | None

class MedicalOntology:
    """Ontología médica integrada."""
    
    async def get_concept(
        self,
        concept_id: str,
        code_system: CodeSystem,
    ) -> MedicalOntologyConcept | None:
        """Obtiene concepto por ID."""
        ...
    
    async def search_concepts(
        self,
        query: str,
        code_system: CodeSystem | None = None,
        limit: int = 10,
    ) -> list[MedicalOntologyConcept]:
        """Busca conceptos por término."""
        ...
    
    async def get_hierarchy(
        self,
        concept_id: str,
    ) -> ConceptHierarchy:
        """Obtiene jerarquía completa."""
        ...

        self,
        code: str,

        target_system: CodeSystem,

        """Mapea código entre sistemas."""
        ...
    
    async def find_narrower(
```

## Consecuencias

### Positivas
- ✅ Integración con estándares médicos globales
- ✅ Navegación jerárquica de conceptos
- ✅ Mapeo automático entre sistemas de codificación
- ✅ Búsqueda semántica de términos

### Negativas
- ⚠️ Licencias de SNOMED-CT (requiere acuerdos)
- ⚠️ Complejidad en mapeo 1:N entre sistemas
- ⚠️ Actualización periódica de ontologías
