# ADR-3013: Standards Repository Design

**Título:** Diseño del Repositorio de Estándares
**Estado:** 🚧 IN PROGRESS
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

EREN necesita acceder y aplicar estándares de ingeniería biomédica para:

1. Validar cumplimiento de equipos médicos
2. Guiar mantenimiento preventivo
3. Identificar requisitos de seguridad
4. Mantener documentación regulatoria

Los estándares incluyen IEC, ISO, AAMI, NFPA y regulaciones FDA.

---

## Decisión

### Arquitectura del Repositorio

```
StandardsRepository
├── IECStandards
│   ├── IEC_60601 (General requirements)
│   ├── IEC_60601_1_8 (Alarm requirements)
│   ├── IEC_62304 (Software lifecycle)
│   └── IEC_80001 (Risk management)
├── ISOStandards
│   ├── ISO_13485 (Quality management)
│   ├── ISO_14971 (Risk management)
│   ├── ISO_15288 (System lifecycle)
│   └── ISO_20485 (Vibration testing)
├── AAMIStandards
│   ├── AAMI_TIR45 (Agile development)
│   └── AAMI_CR500 (Quality fundamentals)
├── NFPACodes
│   ├── NFPA_99 (Health care facilities)
│   └── NFPA_70 (National Electrical Code)
└── RegulatoryReferences
    ├── FDA_21_CFR_820 (QSR)
    ├── FDA_21_CFR_11 (Electronic records)
    └── EU_MDR (European Medical Device Regulation)
```

### Modelo de Datos

```python
@dataclass(frozen=True)
class Standard:
    """Estándar de ingeniería biomédica."""
    standard_id: str
    code: str                          # e.g., "IEC 60601-1"
    title: str
    version: str
    publication_date: date
    organization: StandardOrganization
    scope: str
    description: str
    key_requirements: list[str]
    related_standards: list[str]
    applicable_device_classes: list[RiskClass]
    url: str | None
    is_mandatory: bool

@dataclass(frozen=True)
class StandardRequirement:
    """Requisito específico de un estándar."""
    requirement_id: str
    standard_id: str
    clause: str                        # e.g., "4.2"
    title: str
    description: str
    requirement_text: str
    verification_method: VerificationMethod
    acceptance_criteria: str
    documentation_required: list[str]

@dataclass(frozen=True)
class ComplianceCheck:
    """Verificación de cumplimiento."""
    check_id: str
    device_id: str
    standard_id: str
    requirement_id: str | None
    status: ComplianceStatus
    evidence: str
    verified_at: datetime
    verified_by: str
    notes: str | None

class StandardsRepository:
    """Repositorio de estándares."""
    
    async def get_standard(
        self,
        standard_id: str,
    ) -> Standard | None:
        """Obtiene estándar por ID."""
        ...
    
    async def search_standards(
        self,
        query: str,
        organization: StandardOrganization | None = None,
        device_class: RiskClass | None = None,
    ) -> list[Standard]:
        """Busca estándares."""
        ...
    
    async def get_requirements(
        self,
        standard_id: str,
    ) -> list[StandardRequirement]:
        """Obtiene requisitos de un estándar."""
        ...
    
    async def get_applicable_standards(
        self,
        device_category_id: str,
    ) -> list[Standard]:
        """Obtiene estándares aplicables a una categoría."""
        ...
    
    async def check_compliance(
        self,
        device_id: str,
        standard_id: str,
    ) -> ComplianceReport:
        """Genera reporte de cumplimiento."""
        ...
    
    async def get_related_standards(
        self,
        standard_id: str,
    ) -> list[Standard]:
        """Obtiene estándares relacionados."""
        ...
```

### Organizaciones de Estándares

```python
class StandardOrganization(Enum):
    """Organizaciones de estándares."""
    IEC = "iec"                    # International Electrotechnical Commission
    ISO = "iso"                   # International Organization for Standardization
    AAMI = "aami"                # Association for the Advancement of Medical Instrumentation
    NFPA = "nfpa"                # National Fire Protection Association
    IEEE = "ieee"                # Institute of Electrical and Electronics Engineers
    WHO = "who"                  # World Health Organization
    FDA = "fda"                  # US Food and Drug Administration
    EU = "eu"                    # European Union
    CUSTOM = "custom"             # Estándares institucionales

class VerificationMethod(Enum):
    """Métodos de verificación."""
    INSPECTION = "inspection"
    TESTING = "testing"
    ANALYSIS = "analysis"
    REVIEW = "review"
    DEMONSTRATION = "demonstration"

class ComplianceStatus(Enum):
    """Estado de cumplimiento."""
    COMPLIANT = "compliant"
    NON_COMPLIANT = "non_compliant"
    NOT_APPLICABLE = "not_applicable"
    PENDING = "pending"
    EXEMPT = "exempt"
```

## Consecuencias

### Positivas
- ✅ Acceso centralizado a estándares
- ✅ Compliance checking automatizado
- ✅ Versionamiento de estándares
- ✅ Mapeo a categorías de equipos

### Negativas
- ⚠️ Mantenimiento de estándares actualizados
- ⚠️ Costos de licencias para algunos estándares
- ⚠️ Interpretación puede variar
