# ADR-3010: Knowledge Graph Architecture

**Título:** Arquitectura del Grafo de Conocimiento Biomédico
**Estado:** 🚧 IN PROGRESS
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

El Biomedical Knowledge Engine necesita almacenar y navegar conocimiento biomédico de manera eficiente. Las relaciones entre conceptos médicos, equipos, fallas y estándares son complejas y requieren un modelo de datos que capture:

1. Conceptos médicos (enfermedades, procedimientos, dispositivos)
2. Relaciones jerárquicas (es-un, parte-de)
3. Relaciones causales (causa, previene, trata)
4. Relaciones temporales (ocurre-antes, overlap)

---

## Decisión

### Arquitectura de Grafo

```
BiomedicalKnowledgeGraph
├── Nodes
│   ├── ConceptNode (conceptos médicos)
│   ├── EntityNode (entidades específicas)
│   ├── StandardNode (estándares ISO/IEC)
│   └── DeviceNode (equipos médicos)
└── Edges
    ├── HierarchicalEdge (jerarquía)
    ├── CausalEdge (causalidad)
    ├── SemanticEdge (semántica)
    └── TemporalEdge (temporal)
```

### Tipos de Relaciones

```python
class RelationType(Enum):
    """Tipos de relaciones en el grafo."""
    # Jerárquicas
    IS_A = "is_a"                    # Relación de herencia
    PART_OF = "part_of"              # Relación de composición
    LOCATED_IN = "located_in"       # Localización anatómica
    
    # Causales
    CAUSES = "causes"               # Causa directa
    PREVENTS = "prevents"           # Previene
    TREATS = "treats"               # Trata
    DIAGNOSES = "diagnoses"         # Diagnostica
    
    # Semánticas
    ASSOCIATED_WITH = "associated_with"
    RELATED_TO = "related_to"
    SIMILAR_TO = "similar_to"
    
    # Temporales
    OCCURS_BEFORE = "occurs_before"
    OVERLAPS = "overlaps"
    
    # De equipo
    USES_DEVICE = "uses_device"
    HAS_FAILURE_MODE = "has_failure_mode"
    REQUIRES_MAINTENANCE = "requires_maintenance"
```

## Consecuencias

### Positivas
- ✅ Modelo flexible para relaciones complejas
- ✅ Navegación eficiente del conocimiento
- ✅ Soporte para inferencia basada en grafos
- ✅ Fácil adición de nuevos tipos de relaciones

### Negativas
- ⚠️ Requiere base de datos de grafos (Neo4j o similar)
- ⚠️ Curva de aprendizaje para queries de grafo
