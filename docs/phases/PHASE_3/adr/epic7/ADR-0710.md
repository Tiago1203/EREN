# ADR-0710: Notification System

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Timely notifications keep engineers informed of critical events.

## Decision

We will implement multi-channel notifications:

### Channels

| Channel | Use Case |
|---------|----------|
| In-App | All notifications |
| Email | Digest, critical |
| SMS | Critical alerts only |
| Push | Mobile, on-the-go |
| Webhook | External systems |

### Notification Types

1. **Alert** — Critical device issues
2. **Recommendation** — AI suggestions
3. **Reminder** — Maintenance due
4. **Update** — Status changes
5. **System** — Scheduled reports

### Preferences

```typescript
interface NotificationPrefs {
  channels: Channel[];
  quietHours: { start: string; end: string };
  severityThreshold: "low" | "medium" | "high" | "critical";
}
```
