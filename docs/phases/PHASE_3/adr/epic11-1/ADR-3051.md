# ADR-3051: Enum Consolidation Strategy

## Estado

✅ Accepted - 2026-07-22

---

## Contexto

Durante la auditoría de FASE 3, se identificaron **9 enums duplicados** en múltiples módulos, causando:
- Inconsistencia en valores
- Dificultad de mantenimiento
- Violación del principio DRY

### Enums Duplicados Identificados

| Enum | Módulos con Duplicados |
|------|------------------------|
| ConfidenceLevel | confidence, models, foundation |
| EvidenceLevel | models, evidence, knowledge, reasoning |
| ValidationSeverity | models, validation, rules |
| RiskLevel | confidence, rules, safety |
| SafetyLevel | models, safety |
| ComplianceStatus | evidence/bundle |
| Priority | evidence/bundle |
| LanguageStyle | explainability |
| QualityDimension | confidence, learning |
| UncertaintyType | confidence |

---

## Decisión

Implementar **Estrategia de Consolidación Centralizada**:

> **Single Source of Truth**: `core.intelligence.foundation.enums`

### Arquitectura de Enums

```
core/PHASE_3/intelligence/foundation/enums.py  ← SINGLE SOURCE OF TRUTH
    ├── Severity (4 values)
    ├── RiskLevel (4 values)
    ├── ConfidenceLevel (6 values)
    ├── UncertaintyType (6 values)
    ├── ValidationDecision (3 values)
    ├── ValidationSeverity (5 values)
    ├── ComplianceStatus (5 values)
    ├── Priority (5 values)
    ├── AutomationLevel (4 values)
    ├── LanguageStyle (4 values)
    ├── EvidenceLevel (8 values)
    ├── OutcomeType (3 values)
    ├── FeedbackType (3 values)
    ├── PatternType (4 values)
    ├── RevisionStatus (5 values)
    ├── ApprovalDecision (3 values)
    ├── RollbackTrigger (4 values)
    └── QualityDimension (4 values)
```

### Reglas de Nombrado

| Tipo | Naming Convention | Ejemplo |
|------|-------------------|---------|
| **Shared** | Nombre directo | `ConfidenceLevel` |
| **Local similar pero diferente** | Prefijo del módulo | `EvidencePriority` (evidence) |

## Opciones Consideradas

| Opción | Descripción | Resultado |
|--------|-------------|-----------|
| **Opción 1** | Enums en cada módulo | ❌ Duplicación completa |
| **Opción 2** | Enums en Foundation + Local con prefijo | ✅ **SELECCIONADA** |

---

## Consecuencias

### Positivas

1. ✅ **Single Source of Truth** - Un lugar para modificar enums
2. ✅ **Consistencia** - Todos usan los mismos valores
3. ✅ **Mantenibilidad** - Cambios en un lugar
4. ✅ **Type Safety** - Imports tipados correctamente
5. ✅ **IDE Support** - Autocomplete funciona correctamente

### Negativas

1. ⚠️ **Breaking Changes** - Algunos imports cambian
2. ⚠️ **Migration** - Requiere actualizar código existente

---

## Implementación

### Enums Centralizados en Foundation

**Archivo:** `core/PHASE_3/intelligence/foundation/enums.py`

### Imports en Módulos

```python
# ANTES (duplicado):
from core.intelligence.confidence import ConfidenceLevel  # Local

# DESPUÉS (centralizado):
from core.intelligence.foundation.enums import ConfidenceLevel  # Foundation
```

### Enums Renombrados

| Antes | Después | Módulo |
|-------|---------|--------|
| `Priority` | `EvidencePriority` | evidence/bundle |
| `QualityDimension` | `ConfidenceQualityDimension` | confidence |

### Enums sin Cambios (ya eran únicos)

- `ConfidenceQualityDimension` - confidence (valores diferentes a Foundation)
