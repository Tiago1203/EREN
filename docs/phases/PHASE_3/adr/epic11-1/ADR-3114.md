# ADR-3114: Foundation Enums Architecture

**Título:** Arquitectura de Enums Compartidos en Foundation
**Estado:** ✅ COMPLETO
**Fecha:** 2026-07-22
**Decisor:** Architecture Team

---

## Contexto

Durante la auditoría de FASE 3 se identificó que varios Enums estaban duplicados en múltiples módulos:

### Duplicaciones Detectadas

| Enum | Módulos donde aparecía |
|------|----------------------|
| Severity | rules, safety, decision |
| RiskLevel | rules, safety, decision |
| ValidationDecision | validation, foundation |
| Priority | decision, foundation |

Esta duplicación viola el principio DRY y dificulta el mantenimiento.

---

## Decisión

### Crear `core/intelligence/foundation/enums.py`

Se crea un nuevo módulo que centraliza todos los Enums compartidos:

```python
# core/intelligence/foundation/enums.py

class Severity(Enum):
    CRITICAL = "critical"
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"

class RiskLevel(Enum):
    CRITICAL = "critical"
    HIGH = "high"
    MEDIUM = "medium"
    LOW = "low"

class ConfidenceLevel(Enum):
    VERY_HIGH = "very_high"
    HIGH = "high"
    MODERATE = "moderate"
    LOW = "low"
    VERY_LOW = "very_low"
    UNCERTAIN = "uncertain"

# ... más enums compartidos
```

### Regla de Arquitectura

```
┌────────────────────────────────────────────────────────────────┐
│                    SINGLE SOURCE OF TRUTH                        │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  1. Enums USADOS POR MÚLTIPLES MÓDULOS → van en Foundation  │
│                                                                │
│  2. Enums ESPECÍFICOS DE UN MÓDULO → quedan en ese módulo    │
│                                                                │
│  3. Foundation NUNCA depende de Engines                       │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

---

## Consecuencias

### Positivas
- ✅ SINGLE SOURCE OF TRUTH para enums compartidos
- ✅ Eliminación de duplicación (DRY)
- ✅ Mantenimiento más fácil
- ✅ Consistencia garantizada
- ✅ Type checking más robusto

### Negativas
- ⚠️ Módulos deben importar desde Foundation
- ⚠️ Breaking change si alguien referenciaba enums locales

### Mitigación
- Los módulos re-exportan los enums para backwards compatibility
- Se agregaron comentarios claros en cada módulo

## Implementación

### Archivo: `core/intelligence/foundation/enums.py`

```python
Shared Enumerations for Clinical Intelligence

This module centralizes all shared Enums used across the Clinical Intelligence
Bounded Context to ensure consistency and avoid duplication.
"""
```
