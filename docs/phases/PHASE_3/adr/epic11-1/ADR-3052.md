# ADR-3052: Unit Test Coverage Strategy for FASE 3

## Estado

✅ Accepted - 2026-07-22

---

## Contexto

FASE 3 (Clinical Intelligence) no tenía tests unitarios, lo que impedía:
- Validar implementaciones
- Detectar regresiones
- Mantener calidad de código
- Soportar CI/CD

---

## Estrategia de Testing

### Principios

1. **Tests de Enum** - Verificar valores, conteos, ordenamiento
2. **Tests de Dataclasses** - Creación, propiedades, validaciones
3. **Tests de Clases** - Métodos principales, edge cases
4. **Tests de Integración** - Flujos completos

### Patrón de Tests

```python
class TestConfidenceScore:
    """Tests for ConfidenceScore dataclass."""

    def test_confidence_score_creation(self):
        """Test ConfidenceScore can be created."""
        score = ConfidenceScore(...)
        assert score.score == 0.85

    def test_confidence_score_is_uncertain(self):
        """Test is_uncertain property."""
        uncertain_score = ConfidenceScore(...)
        assert uncertain_score.is_uncertain is True
```
