# ADR-3030: Evidence Retrieval Architecture

**Título:** Arquitectura de Recuperación de Evidencia
**Estado:** ✅ COMPLETO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

El Evidence Retrieval Engine busca y recupera evidencia para respaldar los razonamientos generados por EPIC 2. Debe integrar múltiples fuentes de evidencia y priorizar por relevancia.

---

## Decisión

### Arquitectura General

```
EvidenceRetrieval
├── EvidenceSearcher
│   ├── KnowledgeGraphSearcher → EPIC 1 integration
│   ├── ManualSearcher → Equipment manuals
│   ├── StandardSearcher → IEC/ISO/AAMI/FDA
│   └── HistoricalSearcher → Similar incidents
│
├── EvidenceCollector
│   ├── SourceCollector → Collect from sources
│   ├── EvidenceAggregator → Combine evidence
│   └── RelevanceFilter → Filter by relevance
│
└── EvidenceRanker
    ├── QualityScorer → Evidence quality
    ├── RelevanceScorer → Query relevance
    └── RecencyScorer → Temporal relevance
```

### Modelo de Datos

```python
@dataclass(frozen=True)
class EvidenceItem:
    """Evidencia individual."""
    evidence_id: str
    content: str
    source_type: EvidenceSource
    source_id: str
    relevance_score: float
    quality_score: float
    timestamp: datetime
    url: str | None
    citation: str | None

@dataclass(frozen=True)
class EvidenceQuery:
    """Consulta de evidencia."""
    query: str
    hypothesis_id: str | None
    equipment_type: str | None
    context: dict

class EvidenceRetriever:
    """Motor de recuperación de evidencia."""
    
    async def retrieve(
        self,
        query: EvidenceQuery,
    ) -> list[EvidenceItem]:
        """Recupera evidencia para una consulta."""
        # Search multiple sources in parallel
        sources = await asyncio.gather(
            self.search_knowledge_graph(query),
            self.search_manuals(query),
            self.search_standards(query),
            self.search_historical(query),
        )
        
        # Flatten and deduplicate
        all_evidence = []
        for source_evidence in sources:
            all_evidence.extend(source_evidence)
        
        # Rank and return top evidence
        ranked = await self.ranker.rank(all_evidence, query)
        return ranked[:self.max_evidence]
    
    async def search_knowledge_graph(
        self,
        query: EvidenceQuery,
    ) -> list[EvidenceItem]:
        """Busca en Knowledge Graph (EPIC 1)."""
        # Integration with EPIC 1
        concepts = await self.kg.search(query.query)
        evidence = []
        for concept in concepts:
            evidence.append(EvidenceItem(
                evidence_id=f"kg_{concept.id}",
                content=concept.description,
                source_type=EvidenceSource.KNOWLEDGE_GRAPH,
                source_id=concept.id,
                relevance_score=concept.relevance,
                quality_score=0.9,  # Knowledge Graph is high quality
                timestamp=datetime.now(),
            ))
        return evidence
    
    async def search_standards(
        self,
        query: EvidenceQuery,
    ) -> list[EvidenceItem]:
        """Busca en estándares (IEC, ISO, etc.)."""
        # Search standards repository
        standards = await self.standards_repo.search(query.query)
        return [
            EvidenceItem(
                evidence_id=f"std_{s.id}",
                content=s.requirement_text,
                source_type=EvidenceSource.STANDARD,
                source_id=s.id,
                relevance_score=0.8,
                quality_score=1.0,  # Standards are authoritative
                timestamp=s.publication_date,
                url=s.url,
                citation=s.code,
            )
            for s in standards
        ]
```

## Consecuencias

### Positivas
- ✅ Integración con EPIC 1 (Knowledge Graph)
- ✅ Múltiples fuentes de evidencia
- ✅ Ranking por relevancia y calidad

### Negativas
- ⚠️ Dependencia de calidad de EPIC 1
