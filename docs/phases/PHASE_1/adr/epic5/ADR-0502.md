# ADR-0502: Root Cause Analysis Engine

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

When incidents occur, identifying the root cause is critical for preventing recurrence. The Root Cause Analysis (RCA) Engine provides structured methods for analyzing failures.

## Decision

We will implement an **AI-assisted RCA Engine** supporting multiple methodologies:

### 1. 5 Whys Analysis
```
Problem вҶ’ Why? вҶ’ Answer вҶ’ Why? вҶ’ Answer вҶ’ ...
```
- Iterative questioning
- Surface cause вҶ’ Root cause
- Maximum 5 iterations

## Implementation

```python
class RootCauseAnalyzer:
    async def analyze(
        self,
        incident_id: str,
        methodology: RCAMethodology,  # FIVE_WHYS, FISHBONE, FAULT_TREE
    ) -> RootCauseResult:
        # 1. Gather evidence
        evidence = await self.gather_evidence(incident_id)
        
        # 2. Apply methodology
        analysis = await self.apply_methodology(evidence, methodology)
        
        # 3. Generate recommendations
        recommendations = await self.generate_fixes(analysis)
        
        return RootCauseResult(
            root_cause=analysis.root_cause,
            evidence_chain=analysis.chain,
            recommendations=recommendations,
            confidence=analysis.confidence,
        )
```

## Consequences

### Positive
- Systematic problem solving
- Prevents recurrence
- Continuous improvement
- Knowledge capture

### Negative
- Time-consuming for complex issues
- Requires complete data
