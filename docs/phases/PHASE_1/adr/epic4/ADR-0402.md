# ADR-0402: Cognitive Runtime Design

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

El Cognitive Runtime es el orchestra central que coordina todas las capas del AI Core. Debe:

1. Recibir mensajes del usuario
2. Coordinar el pipeline completo
3. Manejar errores gracefulmente
4. Proporcionar observabilidad completa
5. Supportar streaming de respuestas

Epic 0 Cognitive Model define qué capacidades debe tener. Este ADR define cómo se implementan.

---

**Pipeline execution:**

```python
class PipelineOrchestrator:
    """Orquestra la ejecución del pipeline cognitivo."""
    
    async def execute(
        self,
        context: ProcessingContext,
    ) -> CognitiveResult:
        
        # 1. Pre-safety check
        safety_result = await self.safety_engine.validate_input(context.message)
        if not safety_result.is_safe:
            return CognitiveResult.blocked(safety_result.reason)
        
        # 2. Build context
        enriched_context = await self.context_builder.build(context)
        
        # 3. RAG retrieval
        evidence = await self.rag_orchestrator.retrieve(enriched_context)
        
        # 4. Reasoning
        reasoning_trace = await self.reasoning_engine.reason(
            enriched_context,
            evidence,
        )
        
        # 5. Confidence
        confidence = await self.confidence_engine.calculate(
            reasoning_trace,
            evidence,
        )
        
        # 6. Explainability
        explanation = await self.explainability_engine.explain(
            reasoning_trace,
            confidence,
        )
        
        # 7. Post-safety check
        if not await self.safety_engine.validate_output(reasoning_trace):
            return CognitiveResult.blocked("Output failed safety check")
        
        # 8. Compose response
        response = await self.response_composer.compose(
            reasoning_trace,
            confidence,
            explanation,
            evidence,
        )
        
        return CognitiveResult.success(response)
```

---

## Reasons

**1. Single entry point:**
- Clients (API, Chat UI) solo hablan con CognitiveRuntime
- No necesitan conocer implementación de capas internas

**2. Pipeline como orchestrator:**
- Describe claramente el flujo de procesamiento
- Cada paso es independiente y testeable
- Easy to add/remove/modify steps

**3. Event-driven:**
- Permite logging, monitoring, debugging
- Publish/subscribe para extensiones futuras

**4. Streaming support:**
- Respuestas parciales mientras se generan
- Mejor UX para chat interface

---

## Consequences

**Positive:**
- Clear separation between orchestration y execution
- Event-driven permite extensibilidad
- Streaming para mejor UX
- Complete observability

**Negative:**
- Más código que un simple wrapper
- Más puntos de failure que manejar
- Latencia acumulada del pipeline
