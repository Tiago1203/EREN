# ADR-0306: Maintenance Bounded Context

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

The Maintenance Context manages maintenance planning and scheduling for medical equipment. It is distinct from the Incident Context (EPIC 2) — Incident handles reactive troubleshooting (something broke), while Maintenance handles proactive and planned maintenance (preventive schedules, maintenance plans).

---

## Decision

**Aggregate: MaintenancePlan** (aggregate root)

```
MaintenancePlan (aggregate root)
├── tenant_id, maintenance_plan_id
├── asset_id (FK — from Asset Context)
├── plan_type: enum (preventive, predictive, corrective)
├── frequency: enum (daily, weekly, monthly, quarterly, semi_annual, annual)
├── next_due_date: date
├── last_completed_date: date (nullable)
├── estimated_duration_hours: decimal
├── assigned_team_id (FK — from Staffing Context)
├── checklist_items: list[ChecklistItem] (value object)
├── priority: enum (low, medium, high, critical)
└── status: enum (active, suspended, completed, cancelled)

MaintenanceSchedule
├── tenant_id, schedule_id, maintenance_plan_id (FK)
├── scheduled_date: date
├── scheduled_start_time: time
├── assigned_staff_id (FK — from Staffing Context)
├── estimated_duration: duration
├── actual_start_time: datetime (nullable)
├── actual_end_time: datetime (nullable)
├── status: enum (scheduled, in_progress, completed, cancelled, missed)
└── notes: string

MaintenanceRecord
├── tenant_id, record_id, maintenance_schedule_id (FK)
├── work_performed: string (rich text)
├── parts_replaced: list[PartReplacement] (value object)
├── labor_hours: decimal
├── technician_notes: string
├── is_compliant: boolean
├── iec_60601_check_passed: boolean (safety standard)
└── next_recommended_date: date (nullable)

ChecklistItem (value object)
├── item_number: integer
├── description: string
├── is_required: boolean
└── checked: boolean

PartReplacement (value object)
├── spare_part_id (FK — from Inventory Context)
├── quantity_used: integer
└── lot_number: string
```

**Invariants:**
- A MaintenancePlan must reference exactly one Asset
- `next_due_date` must be updated after each completion
- MaintenanceSchedule must have exactly one assigned staff member
- `scheduled_date` must be in the future when created
- Completion record must have `actual_start_time` and `actual_end_time` set

**Integration with EPIC 2 (Incident Context):**
- When an Incident is resolved, a corrective MaintenancePlan can be auto-created
- MaintenanceRecord can reference an IncidentId (optional FK)

**Domain Events:**
- `MaintenancePlanCreated`, `MaintenancePlanUpdated`, `MaintenancePlanSuspended`
- `MaintenanceScheduled`, `MaintenanceStarted`, `MaintenanceCompleted`, `MaintenanceCancelled`
- `PreventiveMaintenanceOverdue`, `MaintenanceComplianceCheckFailed`

---

## Consequences

**Positive:**
- Clear separation from Incident Context (reactive vs proactive)
- Links to Staffing Context (assigned technicians)
- Links to Inventory Context (spare parts used)
- Links to Asset Context (planned maintenance for assets)
- Links to Incident Context (corrective maintenance from incidents)

**Negative:**
- Integration with 3 other contexts (Staffing, Inventory, Incident) requires event coordination
- Predictive maintenance requires ML model (EPIC 9) — placeholder for now
