# ADR-0303: Organization Bounded Context

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

The Organization Context manages the hospital organization hierarchy and multi-campus structure. It is the top-level context — all other Hospital contexts (Capacity, Staffing, Department, Asset, Inventory) reference Organization entities.

This implements ADR-0051 (Organization Multi-Campus Strategy).

---

## Decision

**Aggregate: Hospital** (aggregate root)

```
Hospital (aggregate root)
├── tenant_id, organization_id, hospital_id
├── hospital_code: string (e.g., "HH-NYC-01")
├── hospital_name: string
├── hospital_type: enum (academic, community, specialty, trauma_center)
├── license_number: string
├── accreditation_status: enum (accredited, pending, expired)
├── license_expiry_date: date
└── contact_info: ContactInfo (value object)

Organization
├── tenant_id, organization_id
├── legal_name: string
├── doing_business_as: string (optional)
├── tax_id: string
├── ownership_type: enum (public, private, non_profit, government)
├── founded_date: date
└── status: enum (active, suspended, dissolved)

Campus
├── tenant_id, organization_id, campus_id
├── campus_code: string
├── campus_name: string
├── address: Address (value object)
├── geographic_coordinates: GeoPoint (value object)
├── timezone: string (IANA timezone)
├── campus_type: enum (main, satellite, outpatient, emergency)
└── status: enum (active, inactive, planning)
```

**Invariants:**
- A Hospital must belong to exactly one Organization
- A Campus must belong to exactly one Organization
- An Organization can have multiple Campuses
- A Campus can have multiple Buildings
- license_expiry_date must be in the future for `accreditation_status = accredited`
- campus timezone must be a valid IANA timezone identifier

**Domain Events:**
- `HospitalRegistered`, `HospitalUpdated`, `HospitalSuspended`
- `OrganizationRegistered`, `OrganizationUpdated`
- `CampusRegistered`, `CampusConfigured`, `CampusDeactivated`

---

## Consequences

**Positive:**
- Hospital is aggregate root — organizational identity centralized
- Multi-campus support via campus_id across all physical entities
- All other Hospital contexts reference Organization context via FK
- Consistent organizational identity across the platform

**Negative:**
- Organizational changes cascade to all dependent contexts
- Multi-campus queries require campus_id filtering
