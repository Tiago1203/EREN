# ADR-0045: Medical Device Vendor Adapters

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Integration Team

---

## Context

Medical devices are supplied by multiple vendors (Philips, GE, Dräger, Mindray). Each has a proprietary API for device data, configuration, and remote management.

---

## Decision

**Each vendor has a dedicated adapter. All adapters implement a common VendorDevicePort interface.**

```
VendorDevicePort (interface)
├── PhilipsAdapter    → Philips IntelliVue, SureSigns
├── GEAdapter         → GE CARESCAPE, MAC
├── DraegerAdapter    → Dräger medical devices
├── MindrayAdapter   → Mindray monitors
└── GenericAdapter   → Standard MQTT/Bluetooth devices
```

---

## Adapter Responsibilities

Each vendor adapter:
1. Translates vendor-specific API to canonical device data model
2. Handles vendor-specific authentication (API keys, OAuth)
3. Implements circuit breaker per vendor
4. Logs vendor-specific errors for debugging

---

## Circuit Breaker Per Vendor

Each vendor adapter has its own circuit breaker:
- 5 consecutive failures → open circuit
- 60 second recovery timeout
- PagerDuty alert on P1 (device data unavailable)
