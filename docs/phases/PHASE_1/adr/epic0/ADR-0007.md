# ADR-0007: Contract-First Development

**Status:** Accepted

**Date:** 2026-03-01

**Deciders:** Architecture Board

---

## Context

EREN has multiple bounded contexts that must communicate. Without explicit contracts, integration breaks silently when one context changes its API.

---

## Decision

**All bounded context interfaces are defined by contracts BEFORE implementation.**

Contracts are defined in code (Python protocols + Pydantic models) and enforced via automated tests. See `EREN_CONTRACTS_FOUNDATION.md` (Epic 0).

---

## Reasons

1. **Early validation:** API shape is validated before implementation
2. **Parallel development:** Teams can work independently on contract-defined interfaces
3. **Breaking change detection:** Contract tests fail if API changes
4. **Documentation:** Contracts serve as living API documentation

---

## Contract Types

| Type | Tool | Enforcement |
|------|------|------------|
| REST API | OpenAPI/Swagger | Contract tests |
| Kafka events | JSON Schema | Schema registry |
| gRPC | Protobuf | Proto compiler |
| Python ports | Pydantic + Protocol | Type checking + tests |
