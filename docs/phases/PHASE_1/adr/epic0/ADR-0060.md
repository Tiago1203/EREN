# ADR-0060: Clinical Intelligence Context Architecture

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

EPIC 5 introduces Clinical Intelligence: reasoning capabilities that operate across the Clinical and Biomedical domains. These need dedicated bounded contexts.

---

## Decision

**Clinical Intelligence has 9 dedicated bounded contexts, each focused on a specific reasoning capability.**

These contexts use the AI Core capabilities (Reason, Decide, Explain, Trust) but add domain-specific logic.

---

## Clinical Intelligence Contexts

| Context | Responsibility | Owns |
|---------|---------------|------|
| RootCauseAnalysis | Identify root causes of failures | Root cause findings |
| DifferentialDiagnosis | Generate diagnostic hypotheses | Diagnoses, candidates |
| FailurePrediction | Predict device failures | Predictions, risk scores |
| RiskAssessment | Evaluate clinical/operational risks | Risk evaluations |
| CalibrationAdvisor | Recommend calibration schedules | Calibration recommendations |
| ComplianceAdvisor | Monitor regulatory compliance | Compliance evaluations |
| Troubleshooting | Guide device troubleshooting | Troubleshooting workflows |
| MaintenanceSuggestions | Generate maintenance recommendations | Maintenance suggestions |
| CalibrationAdvisor | Recommend calibration schedules | Calibration recommendations |

See `EREN_BOUNDED_CONTEXT_MAP.md` (Epic 0 v1.1) for full context map.

---

## Epic 5 Alignment

This ADR drives EPIC 5 (Clinical Intelligence) implementation.
