# ADR-0011: Audit Events Immutable

**Status:** Accepted

**Date:** 2026-03-15

**Deciders:** Security Team

---

## Context

HIPAA requires audit trails for all PHI access. Audit records must never be modified or deleted after creation.

---

## Decision

**Audit log entries are append-only and immutable.**

- Audit events are written to a dedicated `audit_log` table
- No UPDATE or DELETE permissions on audit_log table
- Audit table is partitioned by month for efficient querying
- 7-year retention (HIPAA requirement)

---

## Audit Events

| Event | Trigger |
|-------|---------|
| `PHI_ACCESSED` | Any read of patient, diagnosis, treatment data |
| `DATA_EXPORTED` | Any export of patient data |
| `POLICY_VIOLATED` | Guardrail violation detected |
| `CROSS_TENANT_ACCESS_ATTEMPTED` | Cross-tenant access blocked |
| `AUTH_SUCCESS` | Successful authentication |
| `AUTH_FAILURE` | Failed authentication attempt |
| `ROLE_CHANGED` | User role modified |
| `CDS_RECOMMENDATION_GENERATED` | Clinical decision support response |
| `CDS_HALLUCINATION_DETECTED` | AI hallucination blocked |
