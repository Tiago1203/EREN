# ADR-0044: HL7 V2/V3 Integration

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Integration Team

---

## Context

Many hospital HIS systems still use HL7 V2 (V2.x messages) for order entry, results reporting, and ADT (Admit-Discharge-Transfer) notifications.

---

## Decision

**HL7 V2 messages are received via an HL7 V2 to FHIR converter and processed as FHIR events.**

EREN does not store HL7 V2 natively. An integration adapter converts HL7 V2 messages to FHIR R4 resources, which are then processed by the FHIR adapter.

---

## HL7 V2 Message Types Supported

| Message | Description | Mapped to FHIR |
|---------|------------|---------------|
| ADT^A01 | Patient admission | Patient |
| ADT^A04 | Patient registration | Patient |
| ADT^A08 | Patient update | Patient |
| ORU^R01 | Observation result | Observation |
| ORM^O01 | Order message | ServiceRequest |

---

## HL7 V2 to FHIR Mapping

HL7 V2 data is mapped to FHIR R4 resources:
- Patient demographics → FHIR Patient
- Observations → FHIR Observation
- Orders → FHIR ServiceRequest
