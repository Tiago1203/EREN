# ADR-0040: FHIR R4 Integration

**Status:** Accepted

**Date:** 2026-06-01

**Deciders:** Integration Team

---

## Context

EREN must exchange patient data with Hospital Information Systems (HIS) and Electronic Health Record (EHR) systems. The industry standard for this exchange is FHIR R4.

---

## Decision

**FHIR R4 is the primary standard for clinical data exchange.**

EREN implements a FHIR R4 adapter that translates between internal domain models and FHIR resources.

---

## FHIR Resources Supported

| EREN Entity | FHIR Resource |
|------------|--------------|
| Patient | Patient |
| Diagnosis | Condition |
| Treatment | CarePlan / MedicationRequest |
| Observation | Observation (vitals) |
| Device | Device |
| Location | Location |

---

## FHIR Adapter

The FHIR adapter lives in the Clinical context and translates:
- Inbound: FHIR resources → EREN domain entities
- Outbound: EREN domain entities → FHIR resources
