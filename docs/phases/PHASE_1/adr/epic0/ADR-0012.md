# ADR-0012: No Raw SQL Outside Repositories

**Status:** Accepted

**Date:** 2026-03-15

**Deciders:** Security Team

---

## Context

Raw SQL in application code creates SQL injection risks and bypasses the repository abstraction. Every query must go through the repository layer.

---

## Decision

**All database access goes through repository classes. No raw SQL in application code.**

Repository classes are the only code that knows about SQL. Application code uses domain objects.

---

## Reasons

1. **SQL injection prevention:** Parameterized queries via SQLAlchemy ORM
2. **Repository pattern:** Business logic doesn't know about database schema
3. **Testability:** Repositories can be mocked in unit tests
4. **Audit:** Centralized audit logging of all DB access

---

## Consequences

### Positive
- No SQL injection vulnerabilities
- Clean separation of concerns
- Easy to switch database backends

### Negative
- Complex queries may require SQLAlchemy expression language
- Raw SQL migration requires repository refactoring
