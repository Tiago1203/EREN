# ADR-0056: Cognitive Capability Contracts

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Architecture Board

---

## Context

AI capabilities must have well-defined interfaces so they can be tested, swapped, and evolved independently.

---

## Decision

**All cognitive capabilities are defined as formal contracts (Python protocols).**

Each capability has:
- A defined protocol (input/output contract)
- A set of supported LLM models
- A confidence threshold
- An explanation requirement
- A fallback behavior

---

## Capability Contracts

| Capability | Input | Output | Confidence | Fallback |
|-----------|-------|--------|-----------|---------|
| Reason | Evidence + query | Reasoning chain | 0.0-1.0 | Rule-based |
| Decide | Reasoning chain | Recommendation | 0.0-1.0 | No recommendation |
| Explain | Reasoning chain | Natural language | Always | Minimal explanation |
| Trust | Response + evidence | Trust score | 0.0-1.0 | LOW |
| AssessRisk | Patient data | Risk level + factors | 0.0-1.0 | MEDIUM |

---

## Frozen Status

The cognitive model is **FROZEN** for EPIC 4 implementation. No changes to capability contracts without Architecture Board approval.

See `EREN_COGNITIVE_MODEL.md` (Epic 0 v1.1).
