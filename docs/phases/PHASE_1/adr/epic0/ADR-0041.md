# ADR-0041: MQTT for Device Connectivity

**Status:** Accepted

**Date:** 2026-06-01

**Deciders:** Integration Team

---

## Context

Medical devices (monitors, ventilators, infusion pumps) communicate via MQTT. EREN needs to ingest real-time device telemetry and alarms.

---

## Decision

**MQTT is the protocol for device-to-cloud data ingestion.**

Devices publish telemetry and alarm topics. EREN subscribes and processes events.

---

## Topic Structure

```
devices/{device_id}/telemetry     → Real-time device data (JSON)
devices/{device_id}/alarm        → Alarm events (JSON)
devices/{device_id}/status       → Online/offline status (JSON)
devices/{device_id}/command       → Commands to device (JSON)
```

---

## QoS Levels

| Topic | QoS | Rationale |
|-------|-----|-----------|
| Telemetry | 0 (at-most-once) | High volume, some loss acceptable |
| Alarms | 1 (at-least-once) | Alarms must not be lost |
| Commands | 1 (at-least-once) | Commands must be delivered |
| Status | 0 (at-most-once) | Status updates are frequent |
