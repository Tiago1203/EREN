# ADR-0043: DICOM Integration

**Status:** Accepted

**Date:** 2026-07-16

**Deciders:** Integration Team

---

## Context

EREN must handle medical imaging data (CT scans, X-rays, MRIs) stored in DICOM format. Images must be stored, retrieved, and displayed.

---

## Decision

**DICOM files are stored in S3/MinIO object storage with metadata in PostgreSQL.**

DICOM images are NOT stored in PostgreSQL. They are stored as encrypted objects in S3. Metadata (patient ID, study ID, modality) is indexed in PostgreSQL for fast lookup.

---

## Storage Architecture

```
DICOM Storage
├── S3 (DICOM files)  → Encrypted, versioned, lifecycle to Glacier
└── PostgreSQL (metadata) → Patient ID, study ID, series ID, SOP UID, modality
```

---

## Why DICOM PACS is NOT Used as Primary Store

- Modern healthcare uses cloud-native DICOM storage (AWS HealthLake, Google Health API)
- DICOM files stored in S3 enable CDN distribution for image viewing
- PostgreSQL metadata provides fast queryability
