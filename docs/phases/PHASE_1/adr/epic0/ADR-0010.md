# ADR-0010: Authentication/Authorization/Session Split

**Status:** Accepted

**Date:** 2026-03-15

**Deciders:** Security Team

---

## Context

EREN needs to authenticate users and authorize their actions across multiple roles (clinician, biomedical engineer, hospital admin). The system must support JWT-based stateless authentication with role-based access control.

---

## Decision

**JWT tokens with RBAC for authentication and authorization.**

- **Authentication:** JWT access tokens (15 min) + refresh tokens (24h, rotated)
- **Authorization:** Role-based (RBAC) with fine-grained permissions
- **Sessions:** Stateless JWT, no server-side session storage
- **Token storage:** HttpOnly cookies for browser clients, headers for API clients

---

## Roles

| Role | Permissions |
|------|------------|
| `clinical_admin` | Full patient, diagnosis, treatment access |
| `biomedical_engineer` | Full device, alarm, maintenance access |
| `hospital_admin` | Full hospital management, user management |
| `clinician` | Read patient data, create incidents |
| `technician` | Read device data, create maintenance requests |
| `viewer` | Read-only access |

---

## Guardrail G2.2 (Epic 0.2)

> "tenant context is mandatory for every API request"

Every request must include a valid tenant_id. Requests without tenant context return 403 Forbidden.
