# ADR-9000: Auditoría FASE 1 - Corrección de Imports y Estructura

## Estado

**Aceptado** - 2026-07-22

## Contexto

Durante la auditoría de FASE 1 (Business Domain) se identificaron varios problemas:

### Problemas Críticos

1. **Imports inválidos en `core/PHASE_3/knowledge_assets/`**
   - El módulo `knowledge_assets` importaba desde `core.PHASE_1.domain.knowledge_assets.*`
   - Los archivos nunca existieron en esa ubicación
   - Causaba `ModuleNotFoundError`

2. **Tests con imports incorrectos**
   - `tests/unit/PHASE_1/workflows/composition/test_composition.py` importaba desde `core.composition`
   - `tests/unit/PHASE_1/workflows/workflows/test_engine.py` importaba desde `core.workflows`
   - Estos módulos existen en `core.PHASE_1.workflows.*`

### Problemas Documentados (sin cambios)

3. **Estructura `domain/domain` anidada**
   - 8 de 10 módulos en `core/PHASE_1/domain/` tienen carpetas `domain/` anidadas
   - Ejemplo: `core/PHASE_1/domain/device/domain/entities/`
   - Esta estructura funciona correctamente con los imports actuales
   - **Decisión**: Mantener la estructura para evitar romper dependencias existentes

## Decisiones

### 1. Corrección de imports en `knowledge_assets`

**Antes:**
```python
from core.PHASE_1.domain.knowledge_assets.catalog import AssetCatalog
from core.PHASE_1.domain.knowledge_assets.types import AssetType
```

**Después:**
```python
from core.PHASE_3.knowledge_assets.catalog import AssetCatalog
from core.PHASE_3.knowledge_assets.types import AssetType
```

**Archivos modificados:**
- `core/PHASE_3/knowledge_assets/__init__.py`
- `core/PHASE_3/knowledge_assets/catalog.py`
- `core/PHASE_3/knowledge_assets/collections.py`
- `core/PHASE_3/knowledge_assets/exceptions.py`
- `core/PHASE_3/knowledge_assets/permissions.py`
- `core/PHASE_3/knowledge_assets/registry.py`
- `core/PHASE_3/knowledge_assets/types.py`
- `core/PHASE_3/knowledge_assets/versions.py`
- `tests/unit/PHASE_3/knowledge_assets/test_registry.py`

### 2. Corrección de tests con imports incorrectos

**Antes:**
```python
from core.composition import CognitiveCompositionRoot
from core.workflows.types import WorkflowType
```

**Después:**
```python
from core.PHASE_1.workflows.composition import CognitiveCompositionRoot
from core.PHASE_1.workflows.workflows.types import WorkflowType
```

**Archivos modificados:**
- `tests/unit/PHASE_1/workflows/composition/test_composition.py`
- `tests/unit/PHASE_1/workflows/workflows/test_engine.py`

## Consecuencias

### Positivas

- ✅ Imports funcionan correctamente
- ✅ Los tests pueden ejecutarse
- ✅ El módulo `knowledge_assets` se puede importar sin errores
- ✅ Documentación actualizada para reflejar la estructura correcta

### Negativas

- ⚠️ La estructura `domain/domain` anidada permanece
  - **Justificación**: El riesgo de romper dependencias existentes es mayor que el beneficio de limpiar la estructura
