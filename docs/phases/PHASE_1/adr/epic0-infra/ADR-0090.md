# ADR-0090: GitOps with ArgoCD

**Status:** Proposed

**Date:** 2026-07-16

**Deciders:** Infrastructure Team

---

## Context

EREN needs declarative, Git-driven deployment to Kubernetes. Changes to infrastructure and applications should be applied automatically when merged to main.

---

## Decision

**ArgoCD is used for GitOps-based deployment to Kubernetes.**

All Kubernetes manifests and Helm values are stored in Git. ArgoCD syncs the cluster state with Git.

---

## GitOps Flow

```
Git (main branch)
    │
    ▼
GitHub Actions (CI pipeline)
    │
    ├── Build Docker images
    ├── Push to registry
    ├── Update Helm values in Git
    │
    ▼
ArgoCD (detects drift)
    │
    ▼
Kubernetes cluster
```

---

## What Goes in Git

| What | Location |
|------|---------|
| K8s manifests | `infra/kubernetes/` |
| Helm charts | `infra/helm/` |
| Helm values (per env) | `infra/helm/{app}/values-{env}.yaml` |
| ArgoCD app definitions | `infra/argocd/` |
