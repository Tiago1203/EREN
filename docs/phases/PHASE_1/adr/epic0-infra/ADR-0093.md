# ADR-0093: Celery as Task Queue

**Status:** Accepted

**Date:** 2026-07-16 (Accepted 2026-07-16)

**Deciders:** Infrastructure Team

**Implemented by:** Epic 1 — Infrastructure Platform (`apps/api/app/tasks/`)

---

## Context

EREN has long-running background tasks: backup scripts, data exports, CDS recommendation generation, outbox publishing, scheduled reports. These need an async task queue.

---

## Decision

**Celery is the async task queue for EREN.**

- **Broker:** Redis (development/staging) or Kafka (production)
- **Backend:** Redis for task results
- **Scheduler:** Celery Beat for periodic tasks

---

## Task Queue Architecture

```
EREN API (FastAPI)
    │
    ├── Submit task ──► Celery Broker (Redis/Kafka)
    │                         │
    │                         ▼
    │                    Celery Workers
    │                         │
    │                         ├── Outbox publisher (100ms)
    │                         ├── CDS recommendations (5s)
    │                         ├── Database backup (1h)
    │                         ├── Report generation (30s)
    │                         └── ML feedback processing (1h)
    │
    └── Read result ◄── Redis (results backend)
```

---

## Celery Queues

| Queue | Priority | Use Case |
|-------|---------|---------|
| `celery.high_priority` | High | Critical alerts, CDS P0 |
| `celery.default` | Medium | Standard tasks |
| `celery.low_priority` | Low | Analytics, batch jobs |
| `celery.maintenance` | Low | Scheduled maintenance |

---

## Scaling

- Workers scale based on queue depth (KEDA)
- 3 replicas minimum for high-priority queue
- 1-5 replicas for low-priority queues
