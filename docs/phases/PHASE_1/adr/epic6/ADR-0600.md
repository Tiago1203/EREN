# ADR-0600: Integration Architecture Overview

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EPIC 6 connects EREN with external hospital systems, medical devices, and enterprise software. This enables data exchange and workflow automation.

## Dependencies

- **EPIC 0** вҖ” Philosophy, Architecture
- **EPIC 1** вҖ” Infrastructure Platform
- **EPIC 4** вҖ” AI Core (Cognitive Runtime)
- **EPIC 5** вҖ” Clinical Intelligence (CDSS, Reasoning)

## Decision

We will implement an **Adapter-based Integration Layer** with:

1. **Hospital Systems** вҖ” HL7, FHIR, DICOM
2. **Medical Devices** вҖ” Philips, GE, DrГӨger, Mindray
3. **Enterprise** вҖ” SAP, ServiceNow, Maximo, Azure AD

| Pattern | Use Case |
|---------|----------|

| Batch | Bulk data sync |
| Streaming | Real-time telemetry |

## Consequences

### Positive
- Rich data integration
- Workflow automation
- Vendor flexibility

### Negative
- Complexity management
- Protocol overhead
