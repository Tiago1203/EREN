# ADR-0610: IBM Maximo Integration

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

IBM Maximo is used for asset management and maintenance. EREN needs to sync assets and work orders.

## Decision

We will implement **Maximo REST API adapter**:

### Supported Operations

| Object | Operations | Use Case |
|--------|------------|----------|
| ASSET | CRUD | Asset sync |
| WORKORDER | CRUD | Work orders |
| PM | Read | Preventive maintenance |
| METERS | Read | Usage tracking |

### Implementation

```python
class MaximoAdapter:
    async def connect(self, maximo_url: str):
        self.client = MaximoClient(maximo_url)
        await self.client.authenticate()
        
    async def sync_asset(self, asset: Asset) -> str:
        existing = await self.find_asset(asset.serial_number)
        if existing:
            return await self.update_asset(existing.id, asset)
        return await self.create_asset(asset)
```

## Consequences

### Positive
- Asset management
- Maintenance workflow

### Negative
- Complex object structure
- OSLC requirements
