# ADR-0804: HIPAA Compliance

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

HIPAA compliance is required for handling Protected Health Information (PHI) in US healthcare.

## Decision

We will implement **HIPAA Technical Safeguards**:

### Safeguards

| Safeguard | Implementation |
|-----------|---------------|
| Access Control | User authentication + RBAC |
| Audit Controls | Logging + Immutable logs |
| Integrity | Checksums + Versioning |
| Transmission Security | TLS 1.3 + VPN |
| Authentication | MFA + Session timeout |

### PHI Data Classification

```python
PHI_FIELDS = [
    "patient_id",
    "patient_name",
    "date_of_birth",
    "medical_record_number",
    "diagnosis",
    "treatment_plan",
]
```

### Compliance Checklist

- [ ] Encryption at rest (AES-256)
- [ ] Encryption in transit (TLS 1.3)
- [ ] Access logging
- [ ] Audit trail
- [ ] BAA with cloud providers
- [ ] Regular security assessments
