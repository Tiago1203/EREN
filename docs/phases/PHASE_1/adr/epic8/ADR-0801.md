# ADR-0801: Caching Strategy

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Caching improves response times and reduces database load for frequently accessed data.

## Decision

We will implement **Multi-Level Caching**:

### Cache Layers

| Layer | Technology | TTL | Scope |
|-------|------------|-----|-------|
| L1 | In-memory (Redis) | 5 min | Local |
| L2 | Redis Cluster | 30 min | Global |
| CDN | CloudFront | 1 hour | Static |

### Cached Data

```python
CACHE_STRATEGIES = {
    "device_list": {"ttl": 300, "invalidate": "on_update"},
    "device_detail": {"ttl": 600, "invalidate": "on_change"},
    "dashboard_metrics": {"ttl": 60, "invalidate": "on_event"},
    "user_session": {"ttl": 3600, "invalidate": "on_logout"},
}
```

### Cache Invalidation

- **Event-driven** — Invalidate on entity updates
- **TTL-based** — Automatic expiration
- **Manual** — Admin cache clear
