# ADR-0810: Multi-Region Deployment

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Multi-region deployment ensures high availability and low latency for global hospitals.

## Decision

We will implement **Active-Passive Multi-Region**:

### Regions

| Region | Role | Latency Target |
|--------|------|----------------|
| US-East | Primary | < 50ms |
| US-West | Secondary | Failover |
| EU-West | Secondary | Failover |
| AP-South | Secondary | Failover |

### Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                         DNS (Route 53)                         │
│                    Latency-based Routing                       │
└────────────────┬────────────────┬────────────────┬─────────────┘
                 │                │                │
                 ▼                ▼                ▼
          ┌──────────┐     ┌──────────┐     ┌──────────┐
          │  US-East │     │  EU-West │     │ AP-South │
          │ PRIMARY  │────▶│ PASSIVE  │     │ PASSIVE  │
          └──────────┘     └──────────┘     └──────────┘
                 │                │                │
                 └────────────────┴────────────────┘
                                │
                    ┌───────────┴───────────┐
                    │   Data Replication     │
                    │   (PostgreSQL + S3)   │
                    └───────────────────────┘
```

### Failover Process

1. Health check failure detected
2. DNS TTL updated (60 seconds)
3. Traffic routed to secondary
4. Data consistency verified
5. Operations resumed
