# ADR-0802: Load Testing Strategy

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Load testing ensures EREN can handle production traffic with acceptable performance.

## Decision

We will implement **Comprehensive Load Testing**:

### Test Types

| Type | Tool | Target |
|------|------|--------|
| Load | k6 | Normal traffic |
| Stress | k6 | Peak traffic |
| Spike | k6 | Sudden increases |
| Soak | k6 | Extended duration |

### Performance Targets

| Metric | Target | Threshold |
|--------|--------|-----------|
| Response Time (p95) | < 200ms | 500ms |
| Throughput | 1000 req/s | 500 req/s |
| Error Rate | < 0.1% | 1% |
| CPU Usage | < 70% | 85% |

### Test Scenarios

```yaml
scenarios:
  - name: "Device List"
    requests: 1000
    duration: "5m"
  - name: "Dashboard Load"
    concurrent: 100
    duration: "10m"
  - name: "AI Query"
    requests: 100
    duration: "2m"
```
