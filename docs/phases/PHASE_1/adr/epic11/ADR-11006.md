# ADR-11006: Repository Access Control

**Status:** Proposed  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

Los repositorios de dominio (DeviceRepository, IncidentRepository, etc.) contienen métodos tanto de lectura como de escritura. Actualmente, el acceso es directo:

```
AI Core → Repository → Database
```

Esto viola múltiples principios de DDD y crea acoplamiento.

---

## Decision

**Regla: AI Core nunca accede directamente a repositorios. Todo acceso es via Application Services.**

### Reglas de Acceso

| Acceso | Permitido desde AI Core? | Vía |
|--------|-------------------------|-----|
| Repository.get_by_id() | ❌ No | QueryService.get_by_id() |
| Repository.save() | ❌ No | CommandService.execute() |
| Repository.get_by_status() | ❌ No | QueryService.get_by_status() |
| Repository.delete() | ❌ No | CommandService.delete() |

### Implementación

```python
# ❌ INCORRECTO - No hacer esto
class AICoreController:
    def __init__(self, device_repo: DeviceRepository):
        self._repo = device_repo  # Violación!
    
    async def get_device(self, id):
        return await self._repo.get_by_id(id)  # No permitido

# ✅ CORRECTO - Usar Query Service
class AICoreController:
    def __init__(self, device_query: IDeviceQueryService):
        self._query = device_query  # Correcto!
    
    async def get_device(self, id):
        return await self._query.get_by_id(id)  # Permitido
```

### Marker Interface (Opcional)

```python
# Para marcar repositorios como "solo para interno"
class IDomainRepository(ABC):
    """Marker interface. No usar directamente desde AI Core."""
    pass

class DeviceRepository(IDomainRepository):
    """Repositorio para uso interno del dominio."""
    ...

class IDeviceQueryService(ABC):
    """Interfaz para AI Core. Implementa esta, no Repository."""
    ...
```

---

## Consequences

### Positive
- Dominio protegido de acceso directo
- Clear boundaries
- Fácil de auditar accesos

### Negative
- Más capas de indirección
- Posible overhead
