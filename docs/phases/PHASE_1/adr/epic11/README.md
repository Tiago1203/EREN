## Proceso de Decisión

1. **Proponer**: Crear ADR con contexto y decisión
2. **Discutir**: Revisión por equipo de arquitectura
3. **Aceptar**: Mergear ADR a main
4. **Implementar**: Código basado en ADR aceptado
5. **Actualizar**: Mantener ADR actualizado si hay cambios

---

## Formato de ADR

Cada ADR debe incluir:

- **Status**: Proposed, Accepted, Deprecated, Superseded
- **Context**: Situación que motiva la decisión
- **Decision**: Decisión tomada
- **Consequences**: Positivas, negativas, riesgos
- **References**: ADRs relacionados, documentación
