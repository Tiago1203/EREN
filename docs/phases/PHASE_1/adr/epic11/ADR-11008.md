# ADR-11008: Tool Service Mapping

**Status:** Proposed  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

AI Core (FASE 2) usa Tool Orchestrator para ejecutar herramientas. Cada tool debe mapear a un Application Service.

---

## Decision

Mapear cada tool a su servicio correspondiente.

### Mapeo Tool → Service

| Tool | Query Service | Command Service |
|------|---------------|-----------------|
| get_device | DeviceQueryService.get_device | - |
| list_devices | DeviceQueryService.list_devices | - |
| register_device | - | DeviceCommandService.register |
| search_knowledge | KnowledgeQueryService.search | - |
| create_incident | - | IncidentCommandService.create |
| get_recommendations | RecommendationQueryService.get_pending | - |
| get_cdss_recommendations | - | CDSSService.get_recommendations |
| analyze_root_cause | - | RCAAnalysisService.analyze |

### Estructura de DomainTools

```python
# core/ai/tools/domain/
class IDomainTool(ABC):
    """Base interface for domain tools."""
    
    @property
    def name(self) -> str:
        ...
    
    @property
    def description(self) -> str:
        ...
    
    @abstractmethod
    async def execute(self, params: dict) -> dict:
        ...

class DeviceQueryTool(IDomainTool):
    """Tool for querying devices."""
    
    def __init__(self, query_service: IDeviceQueryService):
        self._service = query_service
    
    @property
    def name(self) -> str:
        return "get_device"
    
    @property
    def description(self) -> str:
        return "Get device details by ID"
    
    async def execute(self, params: dict) -> dict:
        device = await self._service.get_device(
            device_id=params["device_id"],
            tenant_id=params["tenant_id"],
        )
        return asdict(device)
```

### Registro en ToolOrchestrator

```python
# En ToolOrchestrator registration
self.register(
    tool=ToolDefinition(
        name="get_device",
        description="Get device details",
        category=ToolCategory.DOMAIN,
        parameters=DeviceParameters,
    ),
    implementation=DeviceQueryTool(query_service),
)
```

---

## Consequences

### Positive
- Clear mapping entre AI y dominio
- Herramientas documentadas
- Fácil de agregar nuevas herramientas

### Negative
- Más código de glue
- Mantener sincronización
