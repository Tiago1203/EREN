# ADR-11001: Query Services Design

**Status:** Proposed  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

Necesitamos una forma consistente de exponer operaciones de lectura (queries) del dominio de negocio al AI Core. Cada bounded context tiene múltiples métodos de query en sus repositorios, pero:

- No hay un punto de entrada unificado
- Los queries están mezclados con commands en los repositorios
- Los resultados son entidades de dominio, no DTOs

---

## Decision

Implementar **Query Services** siguiendo el patrón CQRS (Command Query Responsibility Segregation).

### Estructura

```
core/application/services/query/
├── device_query_service.py
├── incident_query_service.py
├── knowledge_query_service.py
├── recommendation_query_service.py
├── work_order_query_service.py
├── hospital_query_service.py
├── staff_query_service.py
├── asset_query_service.py
└── inventory_query_service.py
```

### Interfaz Base

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Generic, TypeVar, Result

T = TypeVar("T")
DTO = TypeVar("DTO")

class IQueryService(ABC, Generic[DTO]):
    """Base interface for Query Services."""
    
    @abstractmethod
    async def get_by_id(self, id: str) -> Result[DTO | None, str]:
        """Get entity by ID."""
        ...
    
    @abstractmethod
    async def list_all(
        self,
        limit: int = 100,
        offset: int = 0,
    ) -> Result[list[DTO], str]:
        """List all entities."""
        ...
```

### Ejemplo: DeviceQueryService

```python
@dataclass(frozen=True)
class DeviceDTO:
    """DTO for Device entity."""
    id: str
    serial_number: str
    name: str
    device_type: str
    status: str
    manufacturer: str
    model: str
    is_critical: bool
    location: dict

class IDeviceQueryService(ABC):
    """Interface for Device query operations."""
    
    async def get_device(
        self,
        device_id: DeviceId,
        tenant_id: TenantId,
    ) -> Result[DeviceDTO | None, str]:
        ...
    
    async def list_devices(
        self,
        tenant_id: TenantId,
        limit: int = 100,
        offset: int = 0,
    ) -> Result[list[DeviceDTO], str]:
        ...
    
    async def get_critical_devices(
        self,
        tenant_id: TenantId,
    ) -> Result[list[DeviceDTO], str]:
        ...
    
    async def get_needing_maintenance(
        self,
        tenant_id: TenantId,
    ) -> Result[list[DeviceDTO], str]:
        ...
```

## Consequences

### Positive
- Separación clara de read vs write
- AI solo ve DTOs, no entidades
- Tests más fáciles (mock de servicios)
- Queries específicos para cada caso de uso de AI

### Negative
- Más clases/interfaces a mantener
- Posible duplicación de lógica de query
- Transformación DTO adds overhead

### Risks
- Mantener DTOs sincronizados con entidades
- Queries inconsistentes entre servicios
