# ADR-11004: Event Exposure Strategy

**Status:** Proposed  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

AI Core necesita reaccionar a eventos del dominio de negocio para:
- Notificar al usuario sobre cambios relevantes
- Actualizar contexto automáticamente
- Generar recomendaciones proactivas

Pero no todos los eventos deben ser expuestos a AI.

---

## Decision

Implementar una estrategia de exposición selectiva de eventos.

### Categorías de Eventos

| Categoría | Expuesto a AI | Ejemplos |
|-----------|---------------|----------|
| **Business** | ✅ Sí | DeviceRegistered, IncidentReported, WorkOrderCompleted |
| **Status Change** | ✅ Sí | DeviceStatusChanged, IncidentEscalated |
| **Recommendation** | ✅ Sí | RecommendationGenerated, RecommendationAccepted |
| **Capacity** | ✅ Sí | BedOccupied, BedAvailable |
| **System** | ❌ No | DatabaseError, CacheMiss, BackupCompleted |
| **Security** | ❌ No | SecurityBreach, LoginFailed, AccessDenied |
| **Audit** | ❌ No | EntityModified, ConfigChanged |

### Interfaz de Suscripción

```python
@dataclass(frozen=True)
class AIEventSubscription:
    event_type: str
    callback: Callable[[dict], Awaitable[None]]
    filter: dict | None = None
    tenant_id: str | None = None

class IEventSubscriptionService(ABC):
    """Interface for AI to subscribe to domain events."""
    
    async def subscribe(
        self,
        subscription: AIEventSubscription,
    ) -> str:
        """Subscribe to events. Returns subscription ID."""
        ...
    
    async def unsubscribe(self, subscription_id: str) -> bool:
        ...
    
    async def get_subscriptions(
        self,
        tenant_id: str,
    ) -> list[AIEventSubscription]:
        ...

# AI Core usage
subscription_id = await event_service.subscribe(
    AIEventSubscription(
        event_type="IncidentReported",
        callback=handle_incident,
        filter={"severity": "critical"},
        tenant_id="hospital-1",
    )
)
```

### Eventos Expuestos a AI

| Evento | Filtros Disponibles | Payload |
|--------|---------------------|---------|
| DeviceRegistered | device_type, is_critical | device_id, type |
| DeviceStatusChanged | status, device_type | device_id, old, new |
| IncidentReported | severity, device_type | incident_id, device_id, severity |
| IncidentResolved | - | incident_id, resolution_time |
| WorkOrderCreated | priority | work_order_id, device_id |
| WorkOrderCompleted | - | work_order_id, duration |
| RecommendationGenerated | confidence_min | recommendation_id, confidence |
| BedOccupied | campus_id, bed_type | bed_id, patient_id |

---

## Consequences

### Positive
- AI reacciona a cambios de negocio
- Contexto actualizado automáticamente
- Notificaciones proactivas

### Negative
- Más eventos a mantener
- Posible sobrecarga de eventos
- Filtrado complejo
