# ADR-11002: Command Services Design

**Status:** Proposed  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

AI Core necesita poder ejecutar operaciones de escritura (commands) en el dominio de negocio, pero:
- Repositorios mezclan read/write
- Entidades exponen lógica de negocio interna
- No hay validación centralizada de commands

---

## Decision

Implementar **Command Services** siguiendo el patrón CQRS.

### Estructura

```
core/application/services/command/
├── device_command_service.py
├── incident_command_service.py
├── knowledge_command_service.py
├── recommendation_command_service.py
├── work_order_command_service.py
└── clinical_command_service.py
```

### Patrón Command

```python
from dataclasses import dataclass
from abc import ABC, abstractmethod

# Commands (Input)
@dataclass(frozen=True)
class RegisterDeviceCommand:
    tenant_id: str
    serial_number: str
    name: str
    device_type: str
    manufacturer_name: str
    manufacturer_model: str
    is_critical: bool = False

@dataclass(frozen=True)
class TransferDeviceCommand:
    device_id: str
    tenant_id: str
    new_building: str
    new_floor: str
    new_department: str

# Service Interface
class IDeviceCommandService(ABC):
    """Interface for Device command operations."""
    
    @abstractmethod
    async def register_device(
        self,
        command: RegisterDeviceCommand,
    ) -> Result[DeviceDTO, str]:
        """Register a new device."""
        ...
    
    @abstractmethod
    async def transfer_device(
        self,
        command: TransferDeviceCommand,
    ) -> Result[DeviceDTO, str]:
        """Transfer device to new location."""
        ...
```

### Principios

1. **Command es inmutable**: dataclass(frozen=True)
2. **Un command, una operación**: No combinar operaciones
3. **Validación en Command**: Validar inputs antes de ejecutar
4. **返回值 es DTO**: Nunca exponer entidad directamente

---

## Consequences

### Positive
- Operaciones de escritura centralizadas
- Validación de inputs en un lugar
- AI no puede modificar entidades directamente
- Auditable (log de commands)

### Negative
- Más código (commands + handlers)
- Posible latencia adicional
- Duplicación de validación
