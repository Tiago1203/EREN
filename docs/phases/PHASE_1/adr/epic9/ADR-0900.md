# ADR-0900: Machine Learning Architecture

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EREN necesita una arquitectura de Machine Learning que soporte Feedback Learning, Model Evaluation, Fine Tuning, Prompt Optimization, Analytics y AI Metrics de manera integrada.

---

## Decision

Implementar una arquitectura de ML en 3 capas:

```
┌─────────────────────────────────────────────────────────────┐
│                    ML Application Layer                      │
│         (Feedback, Evaluation, Analytics, Metrics)         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    ML Training Layer                         │
│         (Fine Tuning, RLHF, Prompt Optimization)           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    ML Infrastructure                         │
│         (Model Registry, Feature Store, Tracking)          │
└─────────────────────────────────────────────────────────────┘
```

---

## Consequences

**Pros:**
- Separación clara de responsabilidades
- Escalabilidad independiente por capa
- Testing más fácil

**Cons:**
- Mayor complejidad operacional
- Más servicios a mantener

---

## Dependencies

- EPIC 0: Foundation
- EPIC 4: AI Core
- EPIC 5: Clinical Intelligence
- EPIC 7: User Experience
