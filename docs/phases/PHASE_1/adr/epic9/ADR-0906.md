# ADR-0906: AI Metrics Definition

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Definir métricas específicas para medir el rendimiento del AI.

---

## Decision

Framework de AI Metrics:

### Core Metrics

| Metric | Description | Target |
|--------|-------------|--------|
| Task Accuracy | % tasks completed correctly | > 85% |
| Response Time P95 | 95th percentile latency | < 2s |
| Safety Score | Harmful content rate | < 0.1% |
| User Satisfaction | Average rating | > 4.0/5 |

### Domain Metrics

| Metric | Description | Target |
|--------|-------------|--------|
| Diagnosis Accuracy | Correct diagnosis rate | > 90% |
| Recommendation Quality | Clinical acceptance rate | > 70% |
| Alert Precision | True positive rate | > 80% |

### Business Metrics

| Metric | Description | Target |
|--------|-------------|--------|
| Cost per Query | Inference cost | < $0.01 |
| Uptime | System availability | > 99.9% |
| User Retention | 30-day retention | > 60% |

---

## Consequences

**Pros:**
- Clear success criteria
- Measurable progress
- Stakeholder alignment

**Cons:**
- Metric gaming risk
- Complex measurement
