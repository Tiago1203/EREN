# ADR-0006: Analytics & Reports Architecture

*Status: Proposed*
*Date: 2026-07-24*
*Deciders: Architecture Board*

---

## Context

Necesitamos implementar un sistema de analytics y reportes que:
- Proporcione dashboards analíticos interactivos
- Genere reportes configurables
- Exporte datos en múltiples formatos
- Muestre métricas KPIs avanzados
- Consuma datos de PHASE 1 y genere insights con PHASE 3

---

## Decision

### Arquitectura del Módulo Analytics

1. **Módulo unificado**: Analytics + Reports en módulos separados pero relacionados
2. **Stores dedicados**: Zustand para estado de analytics y reportes
3. **Servicios**: AnalyticsService, ReportService, ExportService
4. **Componentes modulares**: Charts, Metrics, Reports, Export

### Estructura

```
modules/
├── analytics/          # Dashboard y métricas
│   ├── components/     # Charts, Metrics
│   ├── services/       # AnalyticsService
│   ├── stores/        # analytics.store.ts
│   └── types/         # analytics.types.ts
│
└── reports/           # Generación de reportes
    ├── components/    # ReportBuilder, Export
    ├── services/      # ReportService, ExportService
    ├── stores/        # reports.store.ts
    └── types/         # report.types.ts
```

### Flujo de Analytics

```
PHASE 1 (Data) → AnalyticsService → ChartBuilder → Dashboard
                                    ↓
                              PHASE 3 (Insights)
```

---

## Consequences

### Positive
- Visualización clara de métricas
- Reportes configurables
- Exportación flexible
- Integración con PHASE 3

### Negative
- Dependiente de PHASE 1
- Muchos componentes de charts

### Neutral
- Considerar librería de charts (Recharts, Chart.js)
- Cache de métricas frecuentes

---

## Alternatives Considered

### Alternative 1: Librería de charts externa (Recharts)
- **Aceptado**: Rápido desarrollo, buena documentación

### Alternative 2: Charts custom
- **Rechazado**: Más trabajo, inconsistencia visual
