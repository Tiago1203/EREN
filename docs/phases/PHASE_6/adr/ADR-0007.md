# ADR-0007: Notifications & Workspace Architecture

*Status: Proposed*
*Date: 2026-07-24*
*Deciders: Architecture Board*

---

## Context

Necesitamos implementar un sistema de notificaciones y workspace colaborativo que:
- Proporcione notificaciones en tiempo real
- Gestione tareas con tableros Kanban
- Muestre feed de actividad
- Integre con todos los módulos

---

## Decision

### Arquitectura de Módulos

1. **Notifications Module**: Gestión de notificaciones
2. **Workspace Module**: Tareas y colaboración
3. **Stores dedicados**: Zustand para estado
4. **Event Bus**: Comunicación entre módulos

### Flujo de Notificaciones

```
Event (desde cualquier módulo)
        │
        ▼
EventBus.emit('notification', {...})
        │
        ▼
NotificationService
        │
        ├── NotificationStore
        └── NotificationBell (actualiza badge)
```

### Tablero de Tareas

```
TaskBoard
├── Column: Backlog
├── Column: To Do
├── Column: In Progress
├── Column: Review
└── Column: Done
```

---

## Consequences

### Positive
- Colaboración en tiempo real
- Notificaciones centralizadas
- Tareas organizadas

### Negative
- Complex WebSocket handling
- Persistencia de estado

### Neutral
- Polling como fallback
- Optimistic updates

---

## Implementation Notes

### Event Bus

```typescript
// Shared event bus
export const eventBus = {
  emit: (event: string, data: any) => {...},
  subscribe: (event: string, callback: Function) => {...},
};
```

### Real-time Options

1. **WebSockets**: Tiempo real
2. **Polling**: Fallback simple
3. **Server-Sent Events**: Una dirección
