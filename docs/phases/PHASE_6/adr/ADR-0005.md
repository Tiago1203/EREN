# ADR-0005: Knowledge Center Architecture

*Status: Proposed*
*Date: 2026-07-24*
*Deciders: Architecture Board*

---

## Context

Necesitamos implementar el Centro de Conocimiento que proporciona:
- Búsqueda semántica de artículos
- Navegación por categorías y tags
- Visualización de citas y fuentes
- Integración con PHASE 4 (Knowledge Platform)

Este módulo es crítico porque conecta el frontend con toda la base de conocimiento del sistema.

---

## Decision

### Arquitectura del Módulo Knowledge

1. **Servicios dedicados**: KnowledgeService, SearchService, CitationService
2. **Store local**: Zustand para estado del módulo
3. **Componentes modulares**: SearchBar, ArticleList, CategoryNav, CitationViewer
4. **Integración PHASE 4**: RAG Pipeline, Embeddings, Vector DB

### Flujo de Búsqueda

```
User Input → SearchBar → SearchService
                              │
                              ├── Embedding (PHASE_4/embeddings)
                              ├── Vector Search (PHASE_4/qdrant)
                              ├── Article Retrieval
                              └── Reranking (PHASE_4/rag)
                                        │
                                        ▼
                              SearchResults
```

### Estructura de Datos

```typescript
// Article hierarchy
KnowledgeArticle
  ├── Category (jerárquico)
  ├── Tags (multi)
  ├── Citations (multi)
  └── RelatedArticles (multi)
```

---

## Consequences

### Positive
- Búsqueda semántica potente
- Navegación intuitiva por categorías
- Citas con formato estándar
- Integración con PHASE 4

### Negative
- Dependiente de PHASE 4
- Embeddings pueden ser lentos

### Neutral
- Cache de búsquedas frecuentes
- Lazy loading de artículos

---

## Alternatives Considered

### Alternative 1: Búsqueda full-text simple
- **Rechazado**: NoAprovecha semántica

### Alternative 2: Búsqueda semántica (Elegido)
- **Aceptado**: Mejor relevancia, integrado con PHASE 4
