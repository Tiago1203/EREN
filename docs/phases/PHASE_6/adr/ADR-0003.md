# ADR-0003: Operations Center Architecture

*Status: Proposed*
*Date: 2026-07-24*
*Deciders: Architecture Board*

---

## Context

Necesitamos implementar un Centro de Operaciones que centralice:
- Work Orders (órdenes de trabajo)
- Incidents (incidentes)
- Alerts (alertas)
- Device Status (estado de dispositivos)
- Timeline (línea de tiempo)

Este centro consume datos de PHASE 1 (Business Domain) y debe integrarse con el módulo Dashboard de EPIC 1.

---

## Decision

### Arquitectura del Módulo Operations

1. **Módulo unificado**: Un solo módulo `operations/` con sub-componentes
2. **Estado centralizado**: Zustand store para estado global del módulo
3. **Servicios dedicados**: OperationsService para lógica de negocio
4. **Integración PHASE 1**: Consumir APIs existentes de mantenimientos e incidentes

### Estructura

```typescript
// modules/operations/
// ├── components/     # UI components
// ├── pages/         # Page component
// ├── hooks/         # React hooks
// ├── services/      # Business logic
// ├── stores/        # Zustand stores
// ├── types/         # TypeScript types
// └── utils/         # Utilities
```

### Flujo de Datos

```
PHASE 1 (Business Domain)
        │
        ▼
OperationsService
        │
        ├── WorkOrderService
        ├── IncidentService
        ├── AlertService
        └── DeviceStatusService
                │
                ▼
        OperationsStore (Zustand)
                │
                ▼
        Operations Dashboard
```

---

## Consequences

### Positive
- Centralización de operaciones
- Estado unificado
- Fácil acceso a métricas
- Integración con Dashboard

### Negative
- Store puede crecer con muchos datos
- Requiere paginación para grandes volúmenes

### Neutral
- Dependiente de PHASE 1
- Necesita validación de datos

---

## Alternatives Considered

### Alternative 1: Módulos separados por tipo
- work-orders/, incidents/, alerts/
- **Rechazado**: Demasiada fragmentación para una funcionalidad integrada

### Alternative 2: Todo en Dashboard
- **Rechazado**: Mezcla responsabilidades

### Alternative 3: Módulo unificado (Elegido)
- **Aceptado**: Mejor organización, estado compartido
