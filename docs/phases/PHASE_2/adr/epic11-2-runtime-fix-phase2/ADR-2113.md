# ADR-2113: SessionEvent Naming Conflict Resolution

**Título:** Resolución de Conflicto de Nombres SessionEvent
**Estado:** ✅ ACEPTADO
**Fecha:** 2026-07-21
**Decisor:** Architecture Team

---

## Contexto

En el módulo `core/ai/sessions/models.py`, existían dos definiciones con el nombre `SessionEvent`:

1. **Enum `SessionEvent`** - Para tipos de eventos de sesión:
```python
class SessionEvent(str, Enum):
    CREATED = "created"
    UPDATED = "updated"
    COMPLETED = "completed"
    FAILED = "failed"
```

2. **Dataclass `SessionEvent`** - Para registros de eventos:
```python
@dataclass
class SessionEvent:
    id: str
    session_id: str
    event_type: SessionEvent  # Referencia a sí mismo
    timestamp: datetime
```

El dataclass sobrescribía la clase Enum, causando que `SessionEvent.CREATED` fallara con `AttributeError`.

---

## Decisiones

### Decisión 1: Renombrar Dataclass a SessionEventRecord

**Opción A:** Renombrar el dataclass a `SessionEventRecord` (✅ ELEGIDA)
**Opción B:** Usar nombres diferentes para Enum (`SessionEventType`)
**Opción C:** Separar en módulos diferentes

**Razón de la elección:**
- Claridad semántica: `SessionEvent` = tipo, `SessionEventRecord` = instancia
- Impacto mínimo en código existente
- Fácil de entender para nuevos desarrolladores

### Decisión 2: Estructura de Exports

**Elegida:** Exportar ambos desde `__init__.py`

```python
# core/ai/sessions/__init__.py
from core.ai.sessions.models import (
    SessionEvent,              # Enum
    SessionEventRecord,        # Dataclass
    SessionManager,
    # ...
)
```

---

## Consecuencias

### Positivas
- ✅ Elimina el conflicto de nombres
- ✅ Código más legible y mantenible
- ✅ Separa claramente tipos de instancias

### Negativas
- ⚠️ Requiere actualizar imports en `manager.py`
- ⚠️ Puede afectar código que use `SessionEvent` directamente

---

## Cambios Realizados

### 1. models.py
```python
# SessionEvent sigue siendo Enum
class SessionEvent(str, Enum):
    CREATED = "created"
    UPDATED = "updated"
    COMPLETED = "completed"
    FAILED = "failed"

# SessionEventRecord es el nuevo nombre del Dataclass
@dataclass
class SessionEventRecord:
    id: str
    session_id: str
    event_type: SessionEvent  # Ahora usa el Enum correctamente
    timestamp: datetime = field(default_factory=datetime.now)
    message_id: str | None = None
    tokens: int = 0
```

### 2. manager.py
```python
# Importar ambas clases explícitamente
from core.ai.sessions.models import (
    SessionEventRecord,              # Dataclass (nuevo nombre)
    SessionEvent as SessionEventEnum,  # Enum (alias para claridad)
)
```

### 3. __init__.py
```python
from core.ai.sessions.models import (
    SessionEvent,
    SessionEventRecord,
    # ...
)

__all__ = [
    "SessionEvent",
    "SessionEventRecord",
    # ...
]
```
