# ADR-2006: Provider Abstraction

## Estado
**Accepted** ✅

## Contexto
Necesitamos abstraer la comunicación con múltiples proveedores LLM.

## Decisión
Implementar:

- `BaseProvider`: Clase base para proveedores
- `OpenAIProvider`, `AnthropicProvider`, `AzureOpenAIProvider`
- `ProviderRouter`: Enrutador de requests
- `ProviderFactory`: Factory para crear instancias

## Consecuencias
- Positivas: Fácil agregar nuevos proveedores
- Positivas: Swapping entre proveedores
- Neutrales: Proveedores template requieren implementación real
