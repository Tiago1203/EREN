# ADR-2121: RAG Pipeline

## Estado

**Aceptado** - 2026-07-20

## Contexto

Epic 12 implementa el RAG Pipeline — el sistema que recupera conocimiento relevante y lo integra con generación LLM.

El RAG Pipeline es responsable de:
- Recuperar documentos relevantes de múltiples fuentes
- Reorganizar resultados para máxima relevancia
- Construir prompts con contexto recuperado
- Citar fuentes en las respuestas

## Decisiones

### Arquitectura del RAG Pipeline

```
┌─────────────────────────────────────────────────────────────┐
│                       RAG PIPELINE                                  │
│                                                               │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │ Query        │  │ Retrieval    │  │ Re-ranker        │  │
│  │ Processing   │→ │ Engine       │→ │                  │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
│                                                │             │
│                                                ▼             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │ Prompt       │← │ Context      │← │ Document         │  │
│  │ Builder      │  │ Fusion       │  │ Aggregator       │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### Componentes Principales

| Componente | Descripción |
|------------|-------------|
| `CitationBuilder` | Construye citas para las fuentes |
| `ContextBuilder` | Construye contexto para el prompt |
| `QueryProcessor` | Procesa la consulta del usuario |

## Consecuencias

### Positivas
- Recuperación de conocimiento preciso
- Respuestas citeables con fuentes
- Contexto enriquecido para LLM

### Negativas
- Latencia adicional por retrieval
- Dependencia de quality de embeddings
