# ADR-2300: Prompt Architecture

## Estado
**Accepted** ✅

## Contexto
Necesitamos un sistema flexible de construcción de prompts que soporte múltiples modelos y estrategias.

## Decisión
Implementar PromptBuilder como API unificada:

```
core/ai/prompt/
├── models.py           # PromptTemplate, RenderedPrompt
├── renderer.py         # PromptRenderer, ModelAdapter
├── optimizer.py        # PromptOptimizer, PromptAnalyzer
├── versioning.py       # PromptVersioningManager
└── strategy.py        # Estrategias de prompt
```

## Modelos Soportados
- GPT (OpenAI): gpt-4, gpt-3.5-turbo
- Claude (Anthropic): claude-3-opus, claude-3-sonnet
- Gemini (Google): gemini-pro
- Locales: llama-2, mistral

## Consecuencias
- Positivas: Soporte multi-modelo
- Positivas: Estrategias intercambiables
- Neutrales: Curva de aprendizaje
