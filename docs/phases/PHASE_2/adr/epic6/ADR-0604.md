# ADR-0604: Philips Intellivue Integration

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

Philips IntelliVue is a leading patient monitoring system. EREN needs to integrate for telemetry and alerts.

## Decision

We will implement **Philips IntelliVue adapter**:

### Data Points

| Metric | Description | Frequency |
|--------|-------------|-----------|
| Heart Rate | BPM | Real-time |
| SpO2 | Oxygen saturation | Real-time |
| NIBP | Blood pressure | Every 5 min |
| Temp | Temperature | Every 5 min |
| Alarms | Alert notifications | Event-driven |

### Implementation

```python
class PhilipsAdapter:
    async def connect(self, host: str, port: int):
        self.client = PhilipsClient(host, port)
        await self.client.authenticate()
        
    async def subscribe_telemetry(self, bed_id: str):
        await self.client.subscribe(f"/beds/{bed_id}/telemetry")
        
    async def on_telemetry(self, data: TelemetryData):
        await self.process_vitals(data)
        await self.check_alarms(data)
```

## Consequences

### Positive
- Real-time monitoring
- Alarm integration

### Negative
- Proprietary protocol
- Vendor lock-in risk
