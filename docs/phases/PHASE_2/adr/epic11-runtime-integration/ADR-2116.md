# ADR-2116: AICoreController Integration Pattern

**Status:** Proposed  
**Date:** 2026-07-21  
**Deciders:** Architecture Team  

---

## Context

AICoreController actualmente no usa la Integration Layer de EPIC 10. Los Context Providers se crean sin gateways, resultando en datos mock.

Necesitamos integrar setup_integration() en AICoreController.

---

## Decision

Modificar AICoreController para usar setup_integration() durante initialize().

## Implementation

```python
# core/ai/integration/controller.py
class AICoreController:
    def __init__(self, config: AICoreConfig | None = None) -> None:
        self._config = config or AICoreConfig()
        
        # Integration components (lazy initialization)
        self._gateways: dict[str, Any] = {}
        self._memory_bridge: MemoryBridge | None = None
        self._event_bridge: EventBridge | None = None
        
        # Existing components
        self._conversation: ConversationController | None = None
        self._context: ContextBuilder | None = None
        self._providers: ProviderManager | None = None
        # ... otros componentes
    
    async def initialize(self) -> None:
        """Inicializa todos los componentes."""
        # Setup integration FIRST
        integration = setup_integration()
        self._gateways = integration["gateways"]
        self._memory_bridge = integration["memory_bridge"]
        self._event_bridge = integration["event_bridge"]
        
        # Initialize Context Providers with gateways
        self._context = ContextBuilder(
            providers=self._create_providers_with_gateways()
        )
        
        # Initialize other components
        self._conversation = ConversationController(...)
        self._providers = ProviderManager(...)
    
    def _create_providers_with_gateways(self) -> list[BaseContextProvider]:
        """Crea providers con gateways inyectados."""
        return [
            DeviceContextProvider(self._gateways["device"]),
            IncidentContextProvider(self._gateways["incident"]),
            KnowledgeContextProvider(self._gateways["knowledge"]),
            RecommendationContextProvider(self._gateways["recommendation"]),
            HospitalContextProvider(self._gateways["hospital"]),
            WorkOrderContextProvider(self._gateways["workorder"]),
        ]
```

---

## Consequences

### Positive
- AICoreController es el punto de integración
- Todos los providers reciben gateways reales
- Flujo completo sin mocks

### Negative
- AICoreController sabe de la Integration Layer
- Más responsabilidad en el controller
