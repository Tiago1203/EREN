# ADR-2115: Repository Factory Pattern for AI Integration

**Status:** Proposed  
**Date:** 2026-07-21  
**Deciders:** Architecture Team  

---

## Context

El análisis de integración reveló que algunos repositorios (Incident, Knowledge, Recommendation) no existen en `apps/api/app/domain/` donde los espera el AIUnitOfWork. Existen en `core/` pero son interfaces abstractas.

Necesitamos decidir cómo resolver esta discrepancia.

---

## Decision

Crear **Repository Factory** que permita usar implementaciones de `core/` directamente, sin duplicar código en `apps/api/app/domain/`.

### Opción 1: Crear repositorios en apps/api/ (NO RECOMENDADO)
- Duplicación de código
- Mantenimiento dual
- Violación DRY

### Opción 2: Usar core/ directamente (RECOMENDADO)
- Una sola implementación
- DRY compliant
- Requiere verificar que core/ tiene SQLAlchemy impl

### Opción 3: Repository Factory (ELEGIDO)
- Factory que detecta dónde está la implementación
- Fallback a core/ si no existe en apps/api/
- Máximo flexibility

## Implementation

```python
# core/ai/integration/repository_factory.py
class RepositoryFactory:
    """
    Factory que resuelve imports de repositorios.
    
    Busca implementaciones en:
    1. apps.api.app.domain.<context>.repository
    2. core.<context>.domain.repositories
    """
    
    REPOSITORY_PATHS = {
        "device": [
            "apps.api.app.domain.device.repository:SQLAlchemyDeviceRepository",
            "core.device.domain.repositories:DeviceRepository",
        ],
        "incident": [
            "core.incident.domain.repositories:IncidentRepository",
        ],
        "knowledge": [
            "core.knowledge.domain.repositories:KnowledgeRepository",
        ],
        "recommendation": [
            "core.recommendation.domain.repositories:RecommendationRepository",
        ],
        "capacity": [
            "apps.api.app.domain.capacity.repository:SQLAlchemyCapacityRepository",
        ],
        "work_order": [
            "apps.api.app.domain.work_order.repository:SQLAlchemyWorkOrderRepository",
        ],
    }
    
    def get_repository(self, context: str) -> Any:
        """Obtiene repositorio para el contexto dado."""
        paths = self.REPOSITORY_PATHS.get(context, [])
        for module_path, class_name in paths:
            try:
                module = importlib.import_module(module_path)
                cls = getattr(module, class_name)
                return cls
            except (ImportError, AttributeError):
                continue
        raise RepositoryNotFoundError(f"No repository found for {context}")
```

---

## Consequences

### Positive
- DRY compliant
- Single source of truth
- Flexible fallback

### Negative
- Más complejo que import directo
- Requires runtime discovery
