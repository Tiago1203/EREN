# ADR-0500: Clinical Intelligence Architecture

**Status:** Accepted

**Date:** 2026-07-20

**Deciders:** Architecture Board

---

## Context

EPIC 5 builds on EPIC 4's AI Core to provide **Clinical Intelligence** capabilities. This layer transforms the general-purpose AI into a specialized medical device management assistant.

## Dependencies

- **EPIC 0** — Philosophy, Architecture
- **EPIC 2** — Domain Models (Device, Incident, Recommendation, Knowledge)
- **EPIC 3** — Hospital Contexts (Organization, Department, Staffing)
- **EPIC 4** — AI Core (Cognitive Runtime, RAG, Reasoning, Memory)

## Prerequisite for

- **EPIC 6** — Integrations (enriches data from external systems)
- **EPIC 7** — User Experience (provides AI-powered interfaces)
- **EPIC 9** — Machine Learning (feedback loop for improvements)

## Decision

We will implement a **Clinical Intelligence Layer** that:

1. **Leverages EPIC 4 Cognitive Runtime** — Uses the Conversation, RAG, Reasoning, and Memory components
2. **Specializes in Medical Devices** — Adds domain-specific knowledge for biomedical engineering
3. **Provides Clinical Decision Support** — Guides engineers through troubleshooting and maintenance
4. **Integrates with EPIC 2 Domains** — Device, Incident, Recommendation, Knowledge

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    Clinical Intelligence Layer                │
│  CDSS | Root Cause Analysis | Diagnosis | Troubleshooting  │
└────────────────────────┬────────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
┌─────────────────┐ ┌──────────┐ ┌──────────────┐
│    EPIC 4      │ │  EPIC 2  │ │   EPIC 3     │
│ Cognitive      │ │  Domain  │ │  Hospital     │
│ Runtime        │ │  Models  │ │  Contexts     │
└─────────────────┘ └──────────┘ └──────────────┘
```

## Components

### 1. Clinical Decision Support System (CDSS)
- Evidence-based recommendations
- Clinical guideline adherence
- Alert prioritization

### 2. Root Cause Analysis Engine
- Ishikawa diagrams
- 5 Whys analysis
- Fault tree analysis

### 3. Differential Diagnosis
- Symptom matching
- Device-specific error codes
- Historical patterns

### 4. Troubleshooting Engine
- Step-by-step guides
- Decision trees
- Safety checks

### 5. Failure Prediction
- Pattern recognition
- Maintenance scheduling
- Risk scoring

## Consequences

### Positive
- Specialized AI for biomedical engineering
- Improved incident resolution time
- Proactive maintenance
- Evidence-based decisions

### Negative
- Higher complexity
- Requires specialized training data
