# ADR-2500: Tool Architecture

## Estado
**Accepted** ✅

## Contexto
Necesitamos un sistema de herramientas extensible y seguro.

## Decisión
Implementar ToolOrchestrator con:

```python
core/ai/tools/
├── models.py         # ToolDefinition, ToolExecution
├── registry.py      # ToolRegistry
├── executor.py      # ToolExecutor
├── router.py        # ToolRouter
└── sandbox.py      # ToolSandbox
```

Arquitectura:
```
Registry → Router → Executor → Sandbox
```

## Categorías Soportadas
- DATABASE: PostgreSQL, MySQL
- VECTOR_STORE: Qdrant, Pinecone
- MESSAGE_QUEUE: RabbitMQ, Kafka
- API, CODE, WEB, FILE, SYSTEM, MCP

## Consecuencias
- Positivas: Sistema extensible
- Positivas: Seguridad con sandbox
- Neutrales: Complejidad adicional
