# ADR-2901: Processing Pipeline

## Estado
**Accepted** ✅

## Contexto
El procesamiento de solicitudes necesita un pipeline definido.

## Decisión

### Pipeline de Procesamiento
```python
async def process(user_input, session_id, ...):
    # 1. Session
    session = get_or_create_session(...)
    
    # 2. Context Building
    context = build_context(...)
    
    # 3. Prompt Rendering
    prompt = render_prompt(...)
    
    # 4. Provider Call
    result = call_provider(...)
    
    # 5. Response Composition
    response = compose_response(...)
    
    # 6. Memory Update
    update_memory(...)
    
    return response
```

### Estados de Procesamiento
```python
class ProcessingState(Enum):
    IDLE = "idle"
    BUILDING_CONTEXT = "building_context"
    RENDERING_PROMPT = "rendering_prompt"
    CALLING_PROVIDER = "calling_provider"
    EXECUTING_TOOLS = "executing_tools"
    COMPOSING_RESPONSE = "composing_response"
    COMPLETE = "complete"
    FAILED = "failed"
```

## Consecuencias
- Positivas: Pipeline claro
- Positivas: Debugging fácil
- Neutrales: Latencia por paso
