# ADR-2700: Provider Architecture

## Estado
**Accepted** ✅

## Contexto
Necesitamos una capa de abstracción para múltiples proveedores de IA.

## Decisión
Implementar ProviderManager con proveedores:

```python
core/ai/providers/
├── models.py           # Modelos base
├── manager.py         # ProviderManager
├── openai_provider.py # OpenAI
└── anthropic_provider.py # Anthropic
```

### ProviderManager
- Registro de proveedores
- Fallback automático
- Rate limiting
- Retry con backoff
- Uso y costos

## Proveedores Soportados
| Proveedor | Estado |
|-----------|--------|
| OpenAI | ✅ GPT-4, GPT-4-Turbo, GPT-3.5-Turbo |
| Anthropic | ✅ Claude-3-Opus, Sonnet, Haiku |
| Google | PENDIENTE |
| Ollama | PENDIENTE |
| Azure OpenAI | PENDIENTE |

## Consecuencias
- Positivas: Flexibilidad multi-proveedor
- Positivas: Alta disponibilidad
- Neutrales: Complejidad adicional
