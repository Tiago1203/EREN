# ADR-2702: Rate Limiting & Cost Tracking

## Estado
**Accepted** ✅

## Contexto
Se necesita control de uso y costos.

## Decisión

### Rate Limiter
```python
class RateLimiter:
    requests_per_minute: int = 60
    requests_per_hour: int = 1000
    requests_per_day: int = 10000
```

### Token Usage
```python
@dataclass
class TokenUsage:
    prompt_tokens: int
    completion_tokens: int
    total_tokens: int
```

### Cost Tracking
```python
@dataclass
class ModelInfo:
    input_cost: float  # por 1K tokens
    output_cost: float  # por 1K tokens
```

### Estadísticas
```python
@dataclass
class ProviderStats:
    total_requests: int
    successful_requests: int
    failed_requests: int
    total_tokens: int
    total_cost: float
    avg_latency_ms: float
```

## Consecuencias
- Positivas: Control de costos
- Positivas: Prevención de rate limits
- Neutrales: Overhead de tracking
