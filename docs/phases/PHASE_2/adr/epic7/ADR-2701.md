# ADR-2701: Fallback & Retry

## Estado
**Accepted** ✅

## Contexto
Se necesita resiliencia ante fallos de proveedores.

## Decisión

### Fallback Strategy
```python
result = await manager.chat_complete(
    messages,
    fallback_order=[
        ProviderType.OPENAI,
        ProviderType.ANTHROPIC,
        ProviderType.GOOGLE,
    ],
)
```

### Retry Logic
```python
@dataclass
class ProviderConfig:
    max_retries: int = 3
    retry_delay_seconds: float = 1.0
```

Características:
- Reintentos automáticos con backoff exponencial
- Cambio automático al siguiente proveedor en fallback
- Registro de intentos fallidos

## Consecuencias
- Positivas: Resiliencia
- Positivas: Alta disponibilidad
- Neutrales: Latencia adicional en fallos
