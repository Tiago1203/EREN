# ADR-2102: Context Provider Pattern

**Status:** Accepted  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

El Context Builder necesita construir contexto desde múltiples fuentes (Conversation, Memory, Device, Incident, Knowledge, etc.) de forma extensible y ordenada.

---

## Decision

Implementar **Context Providers** como componentes que proporcionan contexto temático.

### Estructura


### Contrato Base

```python
# base.py
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Any
import asyncio

@dataclass
class ContextItem:
    """A piece of context for the AI."""
    source: str
    content: str
    relevance_score: float
    metadata: dict | None = None

@dataclass
class ContextQuery:
    """Query for context building."""
    conversation_id: str
    user_id: str
    tenant_id: str
    max_items: int = 50
    max_tokens: int = 4096
    sources: list[str] | None = None  # Filter by sources

class BaseContextProvider(ABC):
    """
    Base class for all context providers.
    
    Each provider is responsible for gathering context
    from a specific domain or source.
    """
    
    @property
    @abstractmethod
    def name(self) -> str:
        """Provider name for identification."""
        raise NotImplementedError
    
    @property
    @abstractmethod
    def priority(self) -> int:
        """
        Provider priority (lower = higher priority).
        Used for ordering when building context.
        """
        raise NotImplementedError
    
    @abstractmethod
    async def get_context(
        self,
        query: ContextQuery,
    ) -> list[ContextItem]:
        """
        Get context items for the given query.
        
        Returns list of ContextItem ordered by relevance.
        """
        raise NotImplementedError
    
    async def _fetch_with_timeout(
        self,
        coroutine,
        timeout: float = 5.0,
    ) -> Any:
        """Fetch with timeout to prevent hanging."""
        try:
            return await asyncio.wait_for(coroutine, timeout=timeout)
        except asyncio.TimeoutError:
            return []
```

```python
        self,
        query: ContextQuery,

        # Get device status if mentioned
```

### Orquestación de Providers

```python
# context_builder/builder.py
class ContextBuilder:
    """Builds context from multiple providers."""
    
    def __init__(self):
        self._providers: list[BaseContextProvider] = []
    
    def register_provider(self, provider: BaseContextProvider) -> None:
        """Register a context provider."""
        self._providers.append(provider)
        self._providers.sort(key=lambda p: p.priority)
    
    async def build_context(
        self,
        query: ContextQuery,
    ) -> list[ContextItem]:
        """Build context from all providers."""
        all_items = []
        
        # Execute all providers concurrently
        tasks = [
            provider.get_context(query)
            for provider in self._providers
            if self._should_use_provider(provider, query)
        ]
        
        results = await asyncio.gather(*tasks, return_exceptions=True)
        
        for result in results:
            if isinstance(result, list):
                all_items.extend(result)
        
        # Sort by relevance and limit
        all_items.sort(key=lambda x: x.relevance_score, reverse=True)
        return all_items[:query.max_items]
    
    def _should_use_provider(
        self,
        provider: BaseContextProvider,
        query: ContextQuery,
    ) -> bool:
        """Check if provider should be used for this query."""
        if query.sources is None:
            return True
        return provider.name in query.sources
```

## Consequences

### Positive
- Extensible: agregar nuevos providers sin modificar existentes
- Priorizable: providers con prioridad configurable
- Testable: cada provider puede probarse aisladamente
- Lazy loading: providers solo ejecutan si son necesarios

### Negative
- Más componentes a mantener
- Posible inconsistencia entre providers
