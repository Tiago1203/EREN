# ADR-2105: AI-to-Domain Dependency Inversion

**Status:** Accepted  
**Date:** 2026-07-20  
**Deciders:** Architecture Board  

---

## Context

AI Core y Business Domain tienen una relación bidireccional natural:
- AI Core necesita datos del dominio
- Dominio no debe conocer a AI Core

Necesitamos aplicar Dependency Inversion para romper esta dependencia.

---

## Decision

Aplicar DIP de SOLID: ambos módulos dependen de abstracciones (interfaces).

### Diagrama de Dependencias

```
┌─────────────────────────────────────────────────────────────────┐
│  ❌ ANTES - Acoplamiento directo                                  │
│                                                                 │
│  AI Core ──────────────▶ DeviceRepositoryImpl                   │
│        │                      ▲                                   │
│        │                      │                                   │
│        └──────────────────────┘                                   │
│        (AI conoce implementación)                                │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  ✅ DESPUÉS - Dependency Inversion                               │
│                                                                 │
│  AI Core ────▶ IDeviceGateway ◀─── DeviceGateway                │
│        │              ▲           │                              │
│        │              │           │                              │
│        └──────────────┘           │                              │
│        (AI solo conoce interfaz)  │                              │
│                                    │                              │
│                      DeviceGateway usa DeviceRepository         │
│                      (interfaz, no implementación)               │
└─────────────────────────────────────────────────────────────────┘
```

### Inversión de Control

```python
# core/ai/domain/contracts.py
from abc import ABC, abstractmethod
from typing import Generic, TypeVar

T = TypeVar("T")

class IDeviceGateway(ABC, Generic[T]):
    """Contrato que AI Core usa."""
    
    @abstractmethod
    async def get_by_id(self, id: str, tenant_id: str) -> T | None:
        ...
    
    @abstractmethod
    async def search(
        self,
        query: str,
        tenant_id: str,
        filters: dict | None = None,
    ) -> list[T]:
        ...

# core/ai/domain/device_gateway.py
class DeviceGateway(IDeviceGateway["DeviceDTO"]):
    """
    Implementación del contrato.
    
    AI Core no conoce esta clase directamente,
    solo conoce el contrato IDeviceGateway.
    """
    
    def __init__(
        self,
        repository: "DeviceRepository",  # Interfaz, no impl
        memory: "MemoryManager",
    ):
        self._repository = repository
        self._memory = memory
    
    # ... implementación ...

# core/ai/tools/device_tools.py
class SearchDeviceTool:
    """
    Tool que usa el contrato, no la implementación.
    
    Esto permite fácilmente mockear el gateway para tests.
    """
    
    def __init__(self, gateway: IDeviceGateway):  # Tipo es interfaz
        self._gateway = gateway
    
    async def execute(self, params: dict, tenant_id: str):
        # AI usa IDeviceGateway, no DeviceGateway
        return await self._gateway.search(
            query=params["query"],
            tenant_id=tenant_id,
        )
```

### Dependency Injection

```python
# core/ai/integration/controller.py
class AICoreController:
    """
    Controller que inyecta dependencias.
    
    Esto permite fácilmente cambiar implementaciones
    sin modificar el código de AI Core.
    """
    
    def __init__(
        self,
        device_gateway: IDeviceGateway,           # ✅ Interfaz
        incident_gateway: IIncidentGateway,       # ✅ Interfaz
        knowledge_gateway: IKnowledgeGateway,      # ✅ Interfaz
        memory_manager: MemoryManager,
        provider_manager: ProviderManager,
    ):
        self._device = device_gateway
        self._incident = incident_gateway
        self._knowledge = knowledge_gateway
        self._memory = memory_manager
        self._providers = provider_manager
    
    async def process(self, user_input: str, tenant_id: str):
        # Usa interfaces, no implementaciones
        devices = await self._device.search(
            query=user_input,
            tenant_id=tenant_id,
        )
        ...
```

## Consequences

### Positive
- AI Core y dominio completamente desacoplados
- Easy to test (mock gateways)
- Easy to swap implementations
- Clean boundaries

### Negative
- Más interfaces a mantener
- Indirección adicional
