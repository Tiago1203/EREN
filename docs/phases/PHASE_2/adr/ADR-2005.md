# ADR-2005: Model Registry

## Estado
**Accepted** ✅

## Contexto
Necesitamos un registro centralizado de modelos de IA disponibles.

## Decisión
Implementar `ModelRegistry` con:

- Registro/eliminación de modelos
- Búsqueda por capacidades
- Modelo por defecto
- Información de costos

## Consecuencias
- Positivas: Selección dinámica de modelos
- Positivas: Información centralizada
- Neutrales: Requiere mantener actualizado
