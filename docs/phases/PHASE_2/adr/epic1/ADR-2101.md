# ADR-2101: Conversation State Machine

## Estado
**Accepted** ✅

## Contexto
Las conversaciones tienen diferentes estados y transiciones.

## Decisión
Implementar ConversationState como enum:

```python
class ConversationState(str, Enum):
    ACTIVE = "active"      # Puede recibir mensajes
    PAUSED = "paused"     # Temporalmente detenida
    CLOSED = "closed"      # Finalizada
    ARCHIVED = "archived"  # Almacenada
```

Transiciones:
- ACTIVE → PAUSED → ACTIVE
- ACTIVE → CLOSED
- CLOSED → ARCHIVED
- CLOSED → ACTIVE (reopening)

## Consecuencias
- Positivas: Estados claramente definidos
- Positivas: Validación de transiciones
- Neutrales: Complejidad adicional en lifecycle
