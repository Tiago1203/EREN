# ADR-2100: Conversation Architecture

## Estado
**Accepted** ✅

## Contexto
Necesitamos un sistema de gestión de conversaciones que soporte múltiples usuarios y tenants.

## Decisión
Implementar ConversationController como orquestador principal:

```
core/ai/conversation/
├── models.py          # Conversation, Session, Message, Participant
├── events.py         # ConversationEvent, EventDispatcher
├── lifecycle.py      # ConversationLifecycle, SessionLifecycle
├── repository.py     # Repository interface + InMemory impl
└── controller.py    # ConversationController
```

## Consecuencias
- Positivas: API clara y unificada
- Positivas: Soporte multi-tenant nativo
- Positivas: Separación de concerns
- Neutrales: Requiere configuración de repository
