# ADR-2004: Configuration Model

## Estado
**Accepted** ✅

## Contexto
Necesitamos un sistema de configuración flexible para el AI Core.

## Decisión
Implementar dos tipos de configuración:

- `FileAIConfiguration`: Configuración desde archivos YAML
- `DictAIConfiguration`: Configuración programática

Ambos implementan `AIConfiguration` con get/set/validate.

## Consecuencias
- Positivas: Flexible para diferentes entornos
- Positivas: Validación de configuración
- Neutrales: Puede haber inconsistencias entre tipos
