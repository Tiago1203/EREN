# ADR-2801: Token Budget

## Estado
**Accepted** ✅

## Contexto
Se necesita control de uso de tokens por sesión.

## Decisión

### TokenBudget
```python
@dataclass
class TokenBudget:
    total_limit: int = 100000
    used_tokens: int = 0
    
    daily_limit: int = 50000
    daily_used: int = 0
    
    warning_threshold: float = 0.8
    critical_threshold: float = 0.95
```

### Características
- Límite total por sesión
- Límite diario con reset automático
- Alertas de advertencia (80%) y crítico (95%)
- Tracking de uso

### API
```python
budget.remaining  # Tokens restantes
budget.consume(tokens)  # Consumir tokens
budget.check_daily_reset()  # Verificar reset
```

## Consecuencias
- Positivas: Control de costos
- Positivas: Prevención de abusos
- Neutrales: Overhead de tracking
