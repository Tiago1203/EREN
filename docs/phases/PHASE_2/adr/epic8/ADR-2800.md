# ADR-2800: Session Architecture

## Estado
**Accepted** ✅

## Contexto
Necesitamos gestionar sesiones de IA con contexto y límites.

## Decisión
Implementar SessionManager:

```python
core/ai/sessions/
├── __init__.py     # Exports
├── models.py       # Session, TokenBudget, etc.
└── manager.py     # SessionManager
```

### Componentes
| Componente | Descripción |
|-------------|-------------|
| Session | Sesión completa con estado y contexto |
| SessionContext | Contexto de la sesión |
| TokenBudget | Presupuesto de tokens |
| ConversationLimit | Límites de conversación |
| Message | Mensaje individual |

## Estados de Sesión
| Estado | Descripción |
|--------|-------------|
| ACTIVE | Sesión activa |
| IDLE | Sesión pausada |
| EXPIRED | Sesión expirada |
| CLOSED | Sesión cerrada |

## Consecuencias
- Positivas: Gestión centralizada de sesiones
- Positivas: Control de costos con token budget
- Neutrales: Complejidad adicional
