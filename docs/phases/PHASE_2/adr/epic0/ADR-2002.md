# ADR-2002: DTO Schema

## Estado
**Accepted** ✅

## Contexto
Necesitamos definir los objetos de transferencia de datos para comunicación entre componentes.

## Decisión
Implementar DTOs usando dataclasses:

- `Message`, `AIRequest`, `AIResponse`: Mensajes y requests
- `ModelInfo`, `ProviderInfo`, `UsageInfo`: Información de modelos
- `ToolDefinition`, `ToolCall`, `ToolResult`: Herramientas
- `ContextMetadata`, `AIContext`: Contexto
- `HealthReport`, `HealthStatus`: Estado de salud

## Consecuencias
- Positivas: Tipado fuerte con Pydantic/dataclasses
- Positivas: Serialización fácil
- Neutrales: Puede haber duplicación con Domain objects
