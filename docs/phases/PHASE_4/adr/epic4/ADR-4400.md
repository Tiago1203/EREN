# ADR-4400: Clinical RAG Pipeline Architecture

## Estado
**Proposed** 📝

## Contexto
El pipeline RAG clínico debe:
- Procesar queries médicas específicas
- Integrar con PHASE_2 context builder y RAG
- Conectar con PHASE_3 reasoning y decision engines
- Generar respuestas con evidencia y citas
- Aplicar validaciones de seguridad

## Decisión

### Tipos de Query Clínica

| Tipo | Descripción | Ejemplo |
|------|-------------|---------|
| DIAGNOSIS | Diagnóstico diferencial | "Patient with X symptoms" |
| TREATMENT | Recomendaciones de tratamiento | "Best treatment for Y" |
| DEVICE_USAGE | Uso de dispositivos | "How to calibrate pump" |
| TROUBLESHOOTING | Resolución de problemas | "Infusion pump error 5" |
| SAFETY_ALERT | Alertas de seguridad | "Recall for device X" |
| REGULATORY | Cumplimiento regulatorio | "FDA requirements for Y" |

### Pipeline de Clinical RAG

```
Clinical Query
    │
    ├──→ Query Processor
    │        ├── Classify Query Type
    │        └── Extract Clinical Context
    │
    ├──→ Knowledge Retrieval
    │        ├── Vector Search
    │        ├── Evidence Search
    │        └── Knowledge Search
    │
    ├──→ Context Builder
    │        ├── Medical Context
    │        ├── Device Context
    │        └── Safety Context
    │
    ├──→ Clinical Intelligence (PHASE_3)
    │        ├── Reasoning Engine
    │        ├── Evidence Evaluation
    │        └── Safety Validation
    │
    ├──→ Response Generator
    │        ├── LLM Generation
    │        ├── Citation Injection
    │        └── Safety Alerts
    │
    └──→ Clinical RAG Response
             ├── Answer
             ├── Citations
             ├── Confidence
             └── Safety Alerts
```

## Consecuencias

### Positivas
- Query types especializados para clínica
- Integración profunda con PHASE_3
- Safety alerts integrados
- Trazabilidad de evidencia

### Negativas
- Más complejo que RAG genérico
- Requiere coordinación de múltiples componentes
