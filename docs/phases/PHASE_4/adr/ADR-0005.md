# ADR-0005: EPIC 5 - Hybrid Retrieval Architecture

## Status

**Accepted** - 2026-07-23

## Context

Necesitamos recuperar conocimiento de manera efectiva combinando:
1. Búsqueda semántica (vectores)
2. Búsqueda por keywords (BM25)
3. Métodos de fusión para combinar resultados

## Decision

### 1. Arquitectura de Búsqueda

Tres niveles de búsqueda:

```
Search/
├── VectorSearcher ───► Semantic similarity
├── KeywordSearcher ───► Exact match
└── SemanticSearcher ──► Combined (parallel)
```

**Ventajas:**
- Flexibilidad según query
- Parallel execution
- Combina strengths de ambos métodos

### 2. Variantes BM25

Implementamos tres variantes:

```python
BM25/
├── BM25Classic ──── Standard (k1=1.5, b=0.75)
├── BM25L ────────── Short documents
└── BM25Plus ─────── Avoid zero scores
```

**Selección:**
- `BM25Classic`: Default para documentos largos
- `BM25L`: Para títulos, abstracts
- `BM25Plus`: Para robustez

### 3. Métodos de Fusión

```
Fusion/
├── ReciprocalRankFusion (RRF)
├── WeightedAverageFusion
├── CombSUMFusion
└── CombMNZFusion
```

**Estrategia:**
- `RRF`: Robusto, no requiere normalización
- `WeightedAverage`: Cuando tenemos scores comparables
- `CombSUM/MNZ`: Cuando queremos penalizar resultados de una sola fuente

### 4. Arquitectura Modular

```
search/
├── VectorSearcher
├── KeywordSearcher
└── SemanticSearcher

bm25/
├── BM25Classic
├── BM25L
└── BM25Plus

fusion/
├── RRF
├── WeightedAverage
├── CombSUM
└── CombMNZ
```

**Ventajas:**
- Composable
- Testeable
- Intercambiable

## Consequences

### Positive

1. **Flexibilidad**: Múltiples estrategias de búsqueda
2. **Robustez**: RRF no depende de scores normalizados
3. **Performance**: SemanticSearcher ejecuta en paralelo

### Negative

1. **Complejidad**: Más componentes
2. **Selección**: Puede ser difícil elegir estrategia

### Neutral

1. **Rendimiento**: BM25 es rápido pero requiere index
2. **Memoria**: In-memory index puede crecer

## Alternatives Considered

### 1. Solo vector search
- **Rejected**: Falta recall para queries con términos específicos

### 2. Solo keyword search
- **Rejected**: Falta semantic understanding

### 3. Solo un método de fusión
- **Rejected**: RRF es bueno pero no siempre óptimo

## Implementation Notes

### RRF Formula

```python
class ReciprocalRankFusion:
    def __init__(self, k: int = 60):
        self.k = k
    
    def fuse(self, result_lists):
        scores = {}
        for results in result_lists:
            for rank, result in enumerate(results, 1):
                scores[result.doc_id] += 1 / (self.k + rank)
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```
