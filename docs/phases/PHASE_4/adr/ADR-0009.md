# ADR-0009: EPIC 9 - Medical Knowledge Repository Architecture

## Status

**Accepted** - 2026-07-23

## Context

Necesitamos gestionar todas las fuentes biomédicas:
1. Administrar repositorios
2. Control de versiones
3. Gestión de colecciones
4. Registro de fuentes

## Decision

### 1. Arquitectura de Repository

```
Repository/
├── Repository ──────────────────► Modelo de repositorio
├── RepositoryStats ──────────────► Estadísticas
├── InMemoryRepositoryManager ───► Gestor de repos
└── SourceRegistryEntry ──────────► Registro de fuentes
```

**Tipos:**
- PRIMARY: Repo principal
- ARCHIVE: Histórico
- STAGING: Pre-publicación
- VALIDATION: Validación

### 2. Arquitectura de Versioning

```
Versioning/
├── KnowledgeVersion ─────────────► Modelo de versión
├── VersionHistory ───────────────► Historial
├── InMemoryVersionManager ──────► Gestor de versiones
└── VersionComparator ───────────► Comparador
```

**Tipos de versión:**
- MAJOR: Breaking changes
- MINOR: New features
- PATCH: Bug fixes

**Estados:**
- DRAFT → REVIEW → PUBLISHED → ARCHIVED/DEPRECATED

### 3. Arquitectura de Collections

```
Collections/
├── Collection ──────────────────► Modelo de colección
├── CollectionMetadata ────────────► Metadatos
├── InMemoryCollectionManager ────► Gestor de colecciones
└── CollectionSearcher ──────────► Buscador
```

**Tipos:**
- GUIDELINE: Guías clínicas
- PROTOCOL: Protocolos
- TRAINING: Entrenamiento
- REFERENCE: Referencias
- ARCHIVE: Archivo
- CUSTOM: Personalizada

## Consequences

### Positive

1. **Organization**: Fuentes organizadas
2. **Versioning**: Control de cambios
3. **Collections**: Agrupación lógica
4. **Searchability**: Fácil búsqueda

### Negative

1. **Storage**: Más metadata
2. **Complexity**: Más componentes

### Neutral

1. **In-memory**: Para demo; production requiere DB
2. **Scalability**: Diseñado para escalar

## Alternatives Considered

### 1. Sin repositorio centralizado
- **Rejected**: No hay organización

### 2. Sin control de versiones
- **Rejected**: No hay trazabilidad de cambios

### 3. Sin colecciones
- **Rejected**: No hay agrupamiento lógico
