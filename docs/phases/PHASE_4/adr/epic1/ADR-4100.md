# ADR-4100: Medical Embedding Providers

## Estado
**Proposed** 📝

## Contexto
Necesitamos generar embeddings especializados para conocimiento biomédico que capturen:
- Terminología médica específica
- Relaciones entre conceptos clínicos
- Contexto de especialidad médica

Los embeddings genéricos no son suficientes para retrieval efectivo en el dominio clínico.

## Decisión

Implementar proveedores de embeddings médicos que extienden PHASE_2:

### Modelos Soportados

| Modelo | Dimensiones | Uso |
|--------|-------------|-----|
| PubMedBERT | 768 | Literature biomédica |
| BioBERT | 768 | General biomedical |
| ClinicalBERT | 768 | Notas clínicas |
| SciBERT | 768 | Publicaciones científicas |

### Chunking Strategy

| Estrategia | Descripción | Caso de uso |
|------------|-------------|-------------|
| SEMANTIC | Chunking basado en significado | Documentos largos |
| MEDICAL_SECTION | Respeta secciones médicas | Guías clínicas |
| DOCUMENT_STRUCTURE | Respeta estructura del documento | Manuales |
| HYBRID | Combinación de estrategias | General |

## Consecuencias

### Positivas
- Embeddings optimizados para biomedicina
- Chunking que preserva contexto médico
- Mejora en retrieval de conocimiento clínico

### Negativas
- Requiere modelos especializados
- Mayor costo computacional
