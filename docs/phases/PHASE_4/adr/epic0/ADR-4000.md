# ADR-4000: Knowledge Infrastructure Architecture

## Estado
**Proposed** 📝

## Contexto
PHASE 4 debe proporcionar infraestructura de conocimiento que conecte las fases anteriores:
- PHASE_1: Conocimiento del negocio (10 Bounded Contexts)
- PHASE_2: AI Core (embeddings, retrieval, RAG)
- PHASE_3: Clinical Intelligence (reasoning, evidence, decision)

Necesitamos una arquitectura que permita:
1. Generar embeddings especializados para biomedicina
2. Almacenar y buscar vectores en Qdrant
3. Recuperar conocimiento clínico relevante
4. Orquestar RAG específico para clínica
5. Generar citas con trazabilidad de fuentes

## Decisión

Implementar PHASE_4 con 5 módulos principales:

```
core/PHASE_4/
├── embeddings/         # Clinical embeddings
├── qdrant/            # Vector DB integration
├── knowledge/         # Knowledge retrieval
├── rag/               # Clinical RAG pipeline
└── citations/         # Citation engine
```

### Integración con Fases Anteriores

```
PHASE_1 ─────────────────────────────────────────────→ PHASE_4
(Knowledge Context, KnowledgeArticles, Standards)           │
                                                         │
                                                         ▼
PHASE_2 ─────────────────────────────────────────────→ PHASE_4
(Embeddings, Retrieval, RAG, Context Builder)              │
                                                         │
                                                         ▼
PHASE_3 ─────────────────────────────────────────────→ PHASE_4
(Evidence, Reasoning, Decision, Safety, Explainability)    │
                                                         │
                                                         ▼
                                                    User Query
```

### Responsabilidades de Módulo

| Módulo | Responsabilidad | Dependencias |
|---------|----------------|--------------|
| embeddings | Generar embeddings médicos | PHASE_2 embeddings |
| qdrant | Vector storage y search | PHASE_2 cognitive/rag |
| knowledge | Retrieve clinical knowledge | PHASE_1, PHASE_3 |
| rag | Clinical RAG pipeline | PHASE_2 rag, knowledge |
| citations | Source attribution | PHASE_3 evidence |

## Consecuencias

### Positivas
- Arquitectura modular y extensible
- Integración clara con fases anteriores
- Especialización para conocimiento clínico
- Trazabilidad de fuentes mediante citas

### Negativas
- Dependencias adicionales
- Mayor complejidad en pipelines

### Neutrales
- Requiere Qdrant como dependencia externa
