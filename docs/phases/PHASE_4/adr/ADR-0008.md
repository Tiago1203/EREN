# ADR-0008: EPIC 8 - Knowledge Quality Architecture

## Status

**Accepted** - 2026-07-23

## Context

Necesitamos validar la calidad del conocimiento recuperado:
1. Analizar múltiples dimensiones de calidad
2. Detectar sesgos en fuentes
3. Rankear evidencia por confiabilidad
4. Filtrar evidencia de baja calidad

## Decision

### 1. Arquitectura de Quality

```
Quality/
├── QualityScore ──────────────► Score multidimensional
├── ClinicalQualityAnalyzer ───► Analizador clínico
└── QualityReport ─────────────► Reporte de calidad
```

**Dimensiones:**
- ACCURACY: 30% peso
- COMPLETENESS: 20% peso
- CONSISTENCY: 20% peso
- CURRENCY: 10% peso
- RELEVANCE: 20% peso

### 2. Arquitectura de Bias

```
Bias/
├── BiasReport ─────────────────► Reporte de sesgo
├── ClinicalBiasDetector ───────► Detector clínico
└── BiasIndicator ──────────────► Indicadores individuales
```

**Tipos de sesgo:**
- INDUSTRY: Financiamiento industrial
- PUBLICATION: Solo resultados positivos
- LANGUAGE: Solo inglés
- TEMPORAL: Fuentes desactualizadas

### 3. Arquitectura de Ranking

```
Ranking/
├── ClinicalRankingEngine ───────► Ranking clínico
├── QualityRankingEngine ────────► Ranking por calidad
└── DuplicateDetector ───────────► Detección de duplicados
```

**Pesos del ranking clínico:**
- Quality Score: 35%
- Evidence Level: 25%
- Relevance: 25%
- Bias Score: 15%

### 4. Evidence Levels

| Level | Source Type | Score |
|-------|-------------|-------|
| 1 | Systematic Review, RCT | 1.0 |
| 2 | RCT, Cohort | 0.75 |
| 3 | Case-Control | 0.5 |
| 4 | Expert Opinion | 0.25 |
| 5 | Animal/Bench | 0.0 |

## Consequences

### Positive

1. **Quality assurance**: Solo evidencia confiable pasa
2. **Bias detection**: Sesgos identificados
3. **Transparency**: Reportes detallados
4. **Clinical focus**: Optimizado para clínica

### Negative

1. **Complexity**: Múltiples componentes
2. **Processing time**: Análisis addicional
3. **Subjectivity**: Algunos umbrales son arbitrarios

### Neutral

1. **Configurable weights**: Ajustable por contexto
2. **Extensible**: Fácil añadir nuevos tipos de sesgo

## Alternatives Considered

### 1. Solo score simple
- **Rejected**: No captura complejidad de calidad

### 2. Sin detección de sesgos
- **Rejected**: Sesgos pueden afectar resultados

### 3. Sin ranking
- **Rejected**: No hay priorización
