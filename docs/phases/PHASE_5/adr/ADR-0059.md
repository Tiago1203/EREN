# ADR-0059: Collaboration Engine Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 1 (Agent Orchestrator) necesita un mecanismo para que los agentes colaboren entre sí. Los EPICs 2-6 crean agentes especializados que necesitan:
- Compartir contexto
- Intercambiar mensajes
- Sincronizar estado
- Coordinar workspaces

EPIC 7 debe crear el **Collaboration Engine** - un motor que permita que los agentes trabajen juntos de manera efectiva.

---

## Decision

Se adopta una **arquitectura de motor de colaboración con 4 componentes principales**:

### 1. CollaborationEngine como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                    CollaborationEngine                                │
│  (Fachada - recibe tareas, delega a motores)                   │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│   Context   │    │    Agent     │    │   Collaboration │
│   Sharing  │    │  Messaging   │    │      Bus        │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │    Shared       │
                  │   Workspace     │
                  └──────────────────┘
```

### 2. Motores de Colaboración

| Motor | Responsabilidad | API |
|-------|----------------|-----|
| `ContextSharing` | Compartir contexto | `share_context()`, `get_context()` |
| `AgentMessaging` | Mensajería directa | `send_message()`, `receive_messages()` |
| `CollaborationBus` | Pub/Sub | `publish()`, `subscribe()` |
| `SharedWorkspace` | Artefactos compartidos | `create_workspace()`, `add_artifact()` |

### 3. Domain Objects

```python
# Sesión de colaboración
class CollaborationSession:
    session_id: str
    participants: list[Participant]
    shared_context: SharedContext
    status: SessionStatus

# Contexto compartido
class SharedContext:
    context_id: str
    entries: list[ContextEntry]
    
    def add_entry(self, entry: ContextEntry): ...
    def get_entry(self, key: str): ...

# Mensaje
class Message:
    sender_id: str
    recipient_id: str
    content: str
    message_type: MessageType
```

### 4. Tipos de Contexto

```python
class ContextType(Enum):
    TASK = "task"           # Contexto de tarea
    RESULT = "result"       # Contexto de resultado
    STATE = "state"         # Estado del agente
    KNOWLEDGE = "knowledge" # Conocimiento
    PLAN = "plan"           # Plan
    DECISION = "decision"   # Decisión
```

### 5. Tipos de Mensaje

```python
class MessageType(Enum):
    REQUEST = "request"       # Solicitud
    RESPONSE = "response"     # Respuesta
    NOTIFICATION = "notif"   # Notificación
    QUERY = "query"          # Consulta
    COMMAND = "command"      # Comando
    EVENT = "event"         # Evento
```

---

## Consequences

### Positive

1. **Colaboración real**: Agentes pueden trabajar juntos
2. **Modularidad**: Cada motor es independiente
3. **Escalabilidad**: Bus permite muchos-to-muchos
4. **Flexibilidad**: Múltiples tipos de contexto y mensaje

### Negative

1. **Complejidad**: Más componentes que un agente simple
2. **Consistencia**: Requiere sincronización de estado
3. **Latencia**: Mensajería adds latencia

### Neutral

1. **TTL**: Contexto tiene tiempo de vida
2. **Topics**: Bus usa patrón pub/sub

---

## Alternatives Considered

### Alternative 1: Direct Messaging Only

```python
class Agent:
    async def send_message(self, to, content): ...
    async def receive_message(self): ...
```

**Pros**: Simple
**Cons**: No hay contexto compartido, no hay pub/sub

### Alternative 2: Shared Memory

```python
shared_memory = {}
# All agents read/write to shared memory
```

**Pros**: Muy simple
**Cons**: Sin estructura, sin TTL, sin tipos
