# ADR-0058: Planning Agent Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 1 (Agent Orchestrator) necesita agentes especializados para convertir decisiones clínicas en planes ejecutables. EPIC 5 (Research Agent) genera investigación, pero no tiene la capacidad de:
- Convertir decisiones en acciones concretas
- Generar horarios de ejecución
- Descomponer planes en tareas ejecutables
- Evaluar riesgos de planes

EPIC 6 debe crear el **Planning Agent** - un agente dedicado a generar planes de acción que pueda:
- Crear planes de acción a partir de objetivos
- Generar horarios de ejecución
- Descomponer acciones en tareas
- Evaluar y mitigar riesgos
- Crear planes clínicos estructurados

El diseño debe permitir que el PlanningAgent:
- Use el Decision Engine de FASE 3 para decisiones
- Integre con el Orchestrator para coordinación
- Soporte múltiples tipos de plan
- Genere tareas ejecutables con dependencias

---

## Decision

Se adopta una **arquitectura de agente con motores de planificación especializados**:

### 1. PlanningAgent como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                      PlanningAgent                                   │
│  (Fachada - recibe tareas, delega a motores)                   │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│   Action    │    │  Schedule   │    │     Risk       │
│  Planner   │    │  Generator  │    │     Planner    │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │     Task        │
                  │   Generator     │
                  └──────────────────┘
```

**Ventajas:**
- Agente único visible hacia el Orchestrator
- Motores son detalles de implementación
- Fácil agregar/quitar motores de planificación
- Testing simplificado

### 2. Motores de Planificación

| Motor | Responsabilidad | Integración |
|-------|-----------------|-------------|
| `ActionPlanner` | Planificación de acciones | Crea planes de acción |
| `ScheduleGenerator` | Generación de horarios | Programa tareas |
| `TaskGenerator` | Descomposición de tareas | Descompone acciones |
| `RiskPlanner` | Evaluación de riesgos | Evalúa riesgos |

### 3. Tipos de Plan

```python
# Plan de acción
class ActionPlan:
    actions: list[ActionItem]
    critical_path: list[ActionItem]
    progress_percentage: float

# Plan clínico
class ClinicalPlan:
    phases: list[ClinicalPhaseItem]
    current_phase_index: int
    progress_percentage: float
    timeline: list[dict]

# Tarea ejecutable
class ExecutionTask:
    task_type: TaskType
    status: TaskStatus
    dependencies: list[str]
    is_overdue: bool
```

### 4. Prioridades

```python
class ActionPriority(Enum):
    CRITICAL = "critical"    # Crítica - acción inmediata
    HIGH = "high"            # Alta - ASAP
    MEDIUM = "medium"        # Media - esta semana
    LOW = "low"             # Baja - planificar
    ROUTINE = "routine"     # Rutina - cuando sea posible
```

### 5. Tipos de Tarea

```python
class TaskType(Enum):
    CLINICAL = "clinical"         # Clínica
    TECHNICAL = "technical"       # Técnica
    MAINTENANCE = "maintenance"   # Mantenimiento
    CALIBRATION = "calibration"   # Calibración
    INSPECTION = "inspection"     # Inspección
    TRAINING = "training"        # Capacitación
    DOCUMENTATION = "documentation"  # Documentación
    COORDINATION = "coordination"  # Coordinación
```

## Consequences

### Positive

1. **Especialización**: Agente enfocado en planificación
2. **Modularidad**: Motores son independientes
3. **Integración FASE 3**: Usa Decision Engine
4. **Multiple Plan Types**: Soporta planes de acción y clínicos
5. **Extensibilidad**: Fácil agregar nuevos tipos de plan

### Negative

1. **Dependencia FASE 3**: Requiere Decision Engine
2. **Complejidad**: Más clases que un agente simple
3. **Scheduling**: Necesita considerar disponibilidad de recursos

### Neutral

1. **Dependencies**: Maneja dependencias entre tareas
2. **Risk assessment**: Reusa Risk Engine de FASE 3

---

## Alternatives Considered

### Alternative 1: Agente Genérico con Templates

```python
class GenericPlanningAgent:
    async def create_plan(self, objectives: list) -> Plan:
        return self.template_engine.apply("default", objectives)
```

**Pros**: Simple
**Cons**: No adaptable, sin soporte para riesgos

### Alternative 2: Multi-Agent Planning

```
PlanningAgent → ActionPlanner + ScheduleAgent + RiskAgent
```

**Pros**: Máximo desacoplamiento
**Cons**: Overhead de mensajería, más complejidad
