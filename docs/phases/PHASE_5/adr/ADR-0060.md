# ADR-0060: Consensus Engine Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 7 (Collaboration Engine) permite que los agentes colaboren. Cuando múltiples agentes generan respuestas diferentes, necesitamos:
- Un mecanismo para votar
- Resolución de conflictos
- Ranking de respuestas
- Construcción de consenso

EPIC 8 debe crear el **Consensus Engine** - un motor que resuelva conflictos y genere una única respuesta consensuada.

---

## Decision

Se adopta una **arquitectura de motor de consenso con 4 componentes especializados**:

### 1. ConsensusEngine como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                       ConsensusEngine                                  │
│  (Fachada - recibe tareas, delega a motores)                   │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│    Voting   │    │   Conflict   │    │     Ranking    │
│    Engine  │    │   Resolver   │    │     Engine     │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │    Consensus    │
                  │     Builder     │
                  └──────────────────┘
```

### 2. Motores de Consenso

| Motor | Responsabilidad | API |
|-------|----------------|-----|
| `VotingEngine` | Gestión de votación | `vote()`, `tally_votes()` |
| `ConflictResolver` | Resolución de conflictos | `resolve()` |
| `RankingEngine` | Ranking de respuestas | `rank()` |
| `ConsensusBuilder` | Construcción de consenso | `build()` |

### 3. Domain Objects

```python
# Decisión de consenso
class ConsensusDecision:
    decision_id: str
    votes: list[AgentVote]
    consensus_level: ConsensusLevel
    final_decision: dict

# Voto de agente
class AgentVote:
    agent_id: str
    option: VoteOption
    confidence: float
    expertise_level: float
    
    def get_weight(self): ...

# Caso de conflicto
class ConflictCase:
    case_id: str
    conflict_type: ConflictType
    conflicting_responses: list[dict]
    
    def resolve(self, strategy, response): ...
```

### 4. Niveles de Consenso

```python
class ConsensusLevel(Enum):
    NONE = "none"        # < 33%
    MINIMAL = "minimal"  # >= 33%
    MODERATE = "moderate"  # >= 50%
    STRONG = "strong"  # >= 66%
    UNANIMOUS = "unanimous"  # 100%
```

### 5. Estrategias de Resolución

```python
class ResolutionStrategy(Enum):
    MAJORITY = "majority"    # Mayoría simple
    WEIGHTED = "weighted"    # Ponderado por confianza
    EXPERT = "expert"        # Decide el experto
    DELAYED = "delayed"      # Aplazar decisión
    MERGED = "merged"        # Fusionar respuestas
```

---

## Consequences

### Positive

1. **Consenso estructurado**: Votación formal
2. **Resolución de conflictos**: Estrategias definidas
3. **Ranking**: Ordenamiento de respuestas
4. **Ponderación**: Confianza y expertise considerados

### Negative

1. **Complejidad**: Más componentes
2. **Latencia**: Votación toma tiempo
3. **Fallback**: Qué hacer si no hay consenso

### Neutral

1. **Dependencia EPIC 7**: Requiere colaboración
2. **Múltiples estrategias**: Puede ser confuso

---

## Alternatives Considered

### Alternative 1: Simple Majority

```python
def resolve(responses):
    return max(responses, key=lambda r: r["votes"])
```

**Pros**: Simple
**Cons**: No considera confianza, no hay ranking

### Alternative 2: Single Decision Maker

```python
def resolve(responses):
    return expert_agent.decide(responses)
```

**Pros**: Rápido
**Cons**: Centralizado, sin colaboración
