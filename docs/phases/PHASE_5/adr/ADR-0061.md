# ADR-0061: Agent Memory Engine Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 8 (Consensus Engine) necesita que los agentes recuerden. Cuando múltiples agentes colaboran, necesitamos:
- Persistir experiencias
- Gestionar contexto de conversaciones
- Compartir memorias entre agentes
- Aprender colectivamente

EPIC 9 debe crear el **Agent Memory Engine** - un motor que dé memoria individual y colectiva a los agentes.

---

## Decision

Se adopta una **arquitectura de motor de memoria con 5 memorias especializadas**:

### 1. AgentMemory como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                          AgentMemory                                    │
│  (Fachada - recibe tareas, delega a memorias)                 │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│   Episodic  │    │    Shared     │    │    LongTerm     │
│   Memory   │    │    Memory     │    │    Memory       │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│ Conversation │    │   Memory     │    │    Memory      │
│   Memory    │    │ Synchronizer │    │   Analytics    │
└──────────────┘    └───────────────┘    └──────────────────┘
```

### 2. Motores de Memoria

| Motor | Responsabilidad | API |
|-------|----------------|-----|
| `EpisodicMemory` | Eventos específicos | `store()`, `retrieve()` |
| `SharedMemory` | Compartida entre agentes | `share()`, `get_shared()` |
| `LongTermMemory` | Experiencias | `store_experience()`, `retrieve_experiences()` |
| `ConversationMemory` | Conversaciones | `create_context()`, `add_message()` |
| `MemorySynchronizer` | Sincronización | `sync_agent()` |

### 3. Tipos de Memoria

```python
class MemoryType(Enum):
    EPISODIC = "episodic"     # Eventos específicos
    SEMANTIC = "semantic"     # Conocimiento general
    PROCEDURAL = "procedural" # Habilidades
    WORKING = "working"       # Corto plazo
    SHARED = "shared"         # Entre agentes
```

### 4. Importancia de Memoria

```python
class MemoryImportance(Enum):
    CRITICAL = "critical"  # No olvidar
    HIGH = "high"          # Alta
    MEDIUM = "medium"      # Media
    LOW = "low"           # Baja
```

### 5. Resultados de Experiencia

```python
class ExperienceOutcome(Enum):
    SUCCESS = "success"   # Exitoso
    FAILURE = "failure"   # Fallido
    PARTIAL = "partial"   # Parcial
    UNKNOWN = "unknown"   # Desconocido
```

---

## Consequences

### Positive

1. **Memoria estructurada**: Diferentes tipos de memoria
2. **Compartición**: Agentes pueden compartir memorias
3. **Aprendizaje**: Experiencias se almacenan y recuperan
4. **Sincronización**: Consistencia entre agentes

### Negative

1. **Complejidad**: 5 motores de memoria
2. **Almacenamiento**: Necesita persistencia
3. **TTL**: Gestión de expiración

### Neutral

1. **Dependencia FASE 2**: Reusa componentes de memoria
2. **Integración FASE 4**: Comparte conocimiento

---

## Alternatives Considered

### Alternative 1: Single Memory Store

```python
memory_store = {}
# Todo en un solo store
```

**Pros**: Simple
**Cons**: No hay separación de tipos, no hay especialización

### Alternative 2: External Database Only

```python
db.store(agent_id, memory)
# Solo base de datos externa
```

**Pros**: Persistencia robusta
**Cons**: No hay motores especializados, no hay sincronización
