# ADR-0062: Agent Learning & Optimization Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 9 (Agent Memory) almacena experiencias. Ahora necesitamos:
- Analizar el rendimiento de agentes
- Detectar áreas de mejora
- Optimizar estrategias de colaboración
- Aprender de forma continua

EPIC 10 debe crear el **Agent Learning Engine** - un motor que optimice continuamente el comportamiento de los agentes.

---

## Decision

Se adopta una **arquitectura de motor de aprendizaje con 4 componentes especializados**:

### 1. AgentLearningEngine como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                    AgentLearningEngine                                   │
│  (Fachada - recibe tareas, delega a motores)                   │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│  Performance │    │   Strategy    │    │      Agent       │
│  Analyzer   │    │   Optimizer   │    │    Evaluator    │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │  Collaboration   │
                  │   Optimizer     │
                  └──────────────────┘
```

### 2. Motores de Aprendizaje

| Motor | Responsabilidad | API |
|-------|----------------|-----|
| `PerformanceAnalyzer` | Análisis de rendimiento | `analyze()`, `detect_anomalies()` |
| `StrategyOptimizer` | Optimización de estrategias | `optimize()`, `apply_recommendation()` |
| `AgentEvaluator` | Evaluación de agentes | `evaluate()` |
| `CollaborationOptimizer` | Optimización de colaboración | `optimize()`, `suggest_role_adjustments()` |

### 3. Domain Objects

```python
# Métrica de agente
class AgentMetric:
    metric_id: str
    agent_id: str
    metric_type: MetricType
    current_value: float
    trend: str
    
    def calculate_trend(self): ...
    def get_change_percentage(self): ...

# Sesión de aprendizaje
class LearningSession:
    session_id: str
    metrics: list[AgentMetric]
    status: SessionStatus
    
    def start(self): ...
    def complete(self): ...

# Recomendación
class Recommendation:
    title: str
    expected_impact: float
    priority: int
    implemented: bool
```

### 4. Tipos de Métrica

```python
class MetricType(Enum):
    PERFORMANCE = "performance"     # Rendimiento general
    ACCURACY = "accuracy"           # Precisión
    RESPONSE_TIME = "response_time" # Tiempo de respuesta
    COLLABORATION = "collaboration" # Colaboración
    LEARNING = "learning"           # Aprendizaje
```

---

## Consequences

### Positive

1. **Mejora continua**: Agentes aprenden y mejoran
2. **Análisis estructurado**: Métricas y tendencias
3. **Optimización**: Estrategias y colaboración
4. **Evaluación**: Scores comparativos

### Negative

1. **Complejidad**: 4 motores especializados
2. **Datos necesarios**: Requiere métricas
3. **Overhead**: Análisis toma tiempo

### Neutral

1. **Dependencia FASE 3**: Usa componentes de Learning/Improvement
2. **Integración EPIC 9**: Recibe datos de memoria

---

## Alternatives Considered

### Alternative 1: Simple Metric Tracking

```python
metrics_store = {}
# Solo almacenar métricas sin análisis
```

**Pros**: Simple
**Cons**: No hay análisis ni optimización

### Alternative 2: External ML Service

```python
# Usar servicio externo de ML
result = ml_service.optimize(agent_id, data)
```

**Pros**: ML poderoso
**Cons**: Dependencia externa, costo
