# ADR-0057: Research Agent Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 1 (Agent Orchestrator) necesita agentes especializados para realizar investigaciones biomédicas. EPIC 4 (Knowledge Agent) proporciona acceso a literatura y normas, pero no tiene la capacidad de:
- Planificar investigaciones sistemáticas
- Comparar evidencia entre estudios
- Generar revisiones estructuradas
- Producir meta-análisis

EPIC 5 debe crear el **Research Agent** - un agente dedicado a investigación biomédica que pueda:
- Realizar revisiones sistemáticas
- Comparar artículos científicos
- Analizar calidad metodológica
- Generar revisiones de literatura
- Producir resúmenes técnicos

El diseño debe permitir que el ResearchAgent:
- Use EPIC 4 (KnowledgeAgent) como fuente de literatura
- Integre con bases de datos científicas (PubMed, IEEE, etc.)
- Genere revisiones en múltiples formatos
- Soporte diferentes tipos de investigación

---

## Decision

Se adopta una **arquitectura de agente con motores de investigación especializados**:

### 1. ResearchAgent como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                      ResearchAgent                                  │
│  (Fachada - recibe tareas, delega a motores)                   │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│  Research   │    │   Evidence   │    │    Paper        │
│  Planner   │    │  Comparator  │    │    Analyzer     │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │  Literature     │
                  │  Reviewer      │
                  └──────────────────┘
```

**Ventajas:**
- Agente único visible hacia el Orchestrator
- Motores son detalles de implementación
- Fácil agregar/quitar motores de investigación
- Testing simplificado

### 2. Motores de Investigación

| Motor | Responsabilidad | Integración |
|-------|-----------------|-------------|
| `ResearchPlanner` | Planificación sistemática | Crea planes de investigación |
| `EvidenceComparator` | Comparación de estudios | Compara resultados |
| `PaperAnalyzer` | Análisis de artículos | Evalúa calidad |
| `LiteratureReviewer` | Generación de revisiones | Produce reportes |

### 3. Tipos de Investigación

```python
class ResearchType(Enum):
```

## Consequences

### Positive

1. **Especialización**: Agente enfocado en investigación
2. **Modularidad**: Motores son independientes
3. **Integración EPIC 4**: Acceso a literatura de KnowledgeAgent
4. **Multiple Review Types**: Soporta revisiones sistemáticas y meta-análisis
5. **Extensibilidad**: Fácil agregar nuevos tipos de investigación

### Negative

1. **Dependencia EPIC 4**: Requiere que EPIC 4 esté implementado
2. **Complejidad**: Más clases que un agente simple
3. **Base de datos externa**: Necesita acceso a PubMed/IEEE

### Neutral

1. **NLP**: Usa NLP para análisis de artículos
2. **Citation tracking**: Reusa Citation Engine de FASE 4

---

## Alternatives Considered

### Alternative 1: Agente Genérico con Búsqueda

```python
class GenericResearchAgent:
    async def search_and_summarize(self, question: str) -> str:
        return await self.llm.summarize(
            await self.search_database(question)
        )
```

**Pros**: Simple
**Cons**: No sistemático, sin soporte para meta-análisis

### Alternative 2: Multi-Agent Research

```
ResearchAgent → SystematicReviewAgent + MetaAnalysisAgent + ...
```

**Pros**: Máximo desacoplamiento
**Cons**: Overhead de mensajería, más complejidad para Orchestrator
