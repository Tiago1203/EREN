# ADR-0063: Multi-Agent Governance Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 10 (Agent Learning) optimiza agentes. Ahora necesitamos:
- Gobernar todo el ecosistema
- Gestionar permisos y accesos
- Auditar acciones
- Aplicar políticas de seguridad
- Validar cumplimiento regulatorio

EPIC 11 debe crear el **Multi-Agent Governance** - el gobierno completo del sistema multiagente.

---

## Decision

Se adopta una **arquitectura de gobierno centralizada con 4 servicios especializados**:

### 1. GovernanceService como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                         GovernanceService                                  │
│  (Fachada - coordina todos los servicios)                      │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│  Permission  │    │    Audit     │    │      Policy     │
│   Service    │    │   Service    │    │     Engine      │
└──────────────┘    └───────────────┘    └──────────────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │    Compliance   │
                  │    Validator    │
                  └──────────────────┘
```

### 2. Servicios de Gobierno

| Servicio | Responsabilidad | API |
|----------|----------------|-----|
| `PermissionService` | Gestión de permisos | `grant_permission()`, `check_permission()` |
| `AuditService` | Auditoría de acciones | `create_audit()`, `generate_report()` |
| `PolicyEngine` | Gestión de políticas | `create_policy()`, `evaluate_policy()` |
| `ComplianceValidator` | Validación de cumplimiento | `validate_agent()`, `validate_system()` |

### 3. Domain Objects

```python
# Política de agente
class AgentPolicy:
    policy_id: str
    name: str
    policy_type: PolicyType
    status: PolicyStatus
    rules: list[str]

# Permiso de agente
class AgentPermission:
    permission_id: str
    agent_id: str
    resource: str
    permission_type: PermissionType
    granted: bool

# Auditoría de agente
class AgentAudit:
    audit_id: str
    agent_id: str
    audit_type: AuditType
    action: str
    status: AuditStatus

# Regla de gobierno
class GovernanceRule:
    rule_id: str
    rule_type: RuleType
    condition: str
    action: str
    enabled: bool

# Versión de agente
class AgentVersion:
    version_id: str
    agent_id: str
    version: str
    status: VersionStatus
```

### 4. Tipos de Política

```python
class PolicyType(Enum):
    SECURITY = "security"       # Seguridad
    ACCESS = "access"          # Acceso
    BEHAVIOR = "behavior"      # Comportamiento
    DATA = "data"              # Datos
    COMPLIANCE = "compliance"  # Cumplimiento
```

### 5. Tipos de Permiso

```python
class PermissionType(Enum):
    READ = "read"       # Lectura
    WRITE = "write"     # Escritura
    EXECUTE = "execute" # Ejecución
    ADMIN = "admin"     # Administración
    DELEGATE = "delegate" # Delegación
```

---

## Consequences

### Positive

1. **Gobierno completo**: Todo bajo control
2. **Auditoría**: Trazabilidad completa
3. **Seguridad**: Permisos y políticas
4. **Cumplimiento**: Validación regulatoria

### Negative

1. **Complejidad**: 4+ servicios
2. **Overhead**: Cada acción requiere validación
3. **Administración**: Gestión de políticas

### Neutral

1. **Dependencia FASE 3**: Usa Learning/Improvement
2. **Dependencia FASE 4**: Usa Knowledge
3. **Integración EPIC 10**: Recibe métricas

---

## Alternatives Considered

### Alternative 1: Distributed Governance

```python
# Cada agente governance su propia política
agent.governance.apply()
```

**Pros**: Descentralizado
**Cons**: Inconsistencia, difícil auditoría global

### Alternative 2: External Governance Service

```python
# Servicio externo de gobierno
external.govern(agent)
```

**Pros**: Separación clara
**Cons**: Dependencia externa, latencia
