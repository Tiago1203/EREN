# ADR-0056: Knowledge Agent Architecture

*Status: Proposed*
*Date: 2026-07-23*
*Deciders: Architecture Board*

---

## Context

EPIC 1 (Agent Orchestrator) necesita agentes especializados para consultar conocimiento biomédico. EPIC 2 (Biomedical Agent) y EPIC 3 (Diagnostic Agent) generan contexto que necesita ser enrichecido con conocimiento de la plataforma FASE 4. EPIC 4 debe crear el **Knowledge Agent** - un agente especializado en:

- Consultar toda la plataforma de conocimiento construida en FASE 4
- Búsqueda de literatura científica
- Búsqueda de normas y estándares (IEC, ISO, AAMI)
- Generación de citas y referencias
- Construcción de paquetes de conocimiento

El diseño debe permitir que el KnowledgeAgent:
- Use RAG pipeline de FASE 4
- Acceda a vector store (Qdrant)
- Genere citas en múltiples formatos (APA, Vancouver)
- Se integre con EPIC 2 y EPIC 3 para enriquecimiento de contexto

---

## Decision

Se adopta una **arquitectura de agente con motores de búsqueda especializados**:

### 1. KnowledgeAgent como Fachada

```
┌────────────────────────────────────────────────────────────────┐
│                      KnowledgeAgent                                │
│  (Fachada - recibe tareas, delega a motores)                   │
└──────────────────────────┬─────────────────────────────────────┘
                           │
     ┌─────────────────────┼─────────────────────┐
     │                     │                     │
     ▼                     ▼                     ▼
┌──────────────┐    ┌───────────────┐    ┌──────────────────┐
│  Knowledge   │    │  Citation    │    │    Search       │
│  Retriever  │    │  Collector   │    │    Engines      │
└──────────────┘    └───────────────┘    └──────────────────┘
```

**Ventajas:**
- Agente único visible hacia el Orchestrator
- Motores son detalles de implementación
- Fácil agregar/quitar motores de búsqueda
- Testing simplificado

### 2. Motores de Búsqueda

| Motor | Responsabilidad | Integración FASE 4 |
|-------|-----------------|-------------------|
| `KnowledgeRetriever` | Retrieval de conocimiento | RAG Pipeline, Qdrant |
| `CitationCollector` | Colección de citas | Citation Engine |
| `KnowledgeSearchEngine` | Búsqueda general | Vector Store |
| `LiteratureSearchEngine` | Literatura científica | PubMed API |
| `StandardsSearchEngine` | Normas y estándares | Standards DB |

### 3. Query Types

```python
class QueryType(Enum):
    FACTUAL = "factual"           # Consulta factual
    PROCEDURAL = "procedural"     # Procedimientos
    REGULATORY = "regulatory"    # Normativa/regulación
    TECHNICAL = "technical"       # Técnica
    CLINICAL = "clinical"        # Clínica
    EQUIPMENT = "equipment"      # Equipos
    SAFETY = "safety"           # Seguridad
    COMPLIANCE = "compliance"    # Cumplimiento
```

### 4. Domain Objects

```python
# Consulta de conocimiento
class KnowledgeQuery:
    query_id: str
    query_type: QueryType
    question: str
    sources: list[KnowledgeSource]
    max_results: int
    min_relevance: float

# Paquete de conocimiento
class KnowledgePackage:
    package_id: str
    items: list[KnowledgeItem]
    total_items: int
    avg_relevance: float

# Bundle de citas
class CitationBundle:
    bundle_id: str
    citations: list[Citation]
    
    def format_references(self, style: str) -> str:
        """APA, Vancouver, etc."""
```

### 5. Citation Formats

```python
# APA
citation.format_apa()  # "Authors (Year). Title. Journal."

# Vancouver  
citation.format_vancouver()  # "Authors. Title. Journal. Year;Vol(Issue):Pages."
```

## Consequences

### Positive

1. **Especialización**: Agente enfocado en conocimiento
2. **Modularidad**: Motores son independientes
3. **Integración FASE 4**: Acceso completo a la plataforma de conocimiento
4. **Citation Support**: Generación de citas en múltiples formatos
5. **Extensibilidad**: Fácil agregar nuevos motores de búsqueda

### Negative

1. **Dependencia FASE 4**: Requiere que FASE 4 esté implementado
2. **Complejidad**: Más clases que un agente simple

### Neutral

1. **Vector Store**: Usa Qdrant para similarity search
2. **Citation Engine**: Reusa Citation Engine de FASE 4

---

## Alternatives Considered

### Alternative 1: Agente Genérico con RAG

```python
class GenericAgent:
    async def query_knowledge(self, question: str) -> str:
        return await self.rag_pipeline.query(question)
```

**Pros**: Simple
**Cons**: No especializado, sin citation support

### Alternative 2: Multi-Agent Knowledge

```
KnowledgeAgent → ManualAgent + StandardsAgent + LiteratureAgent
```

**Pros**: Máximo desacoplamiento
**Cons**: Overhead de mensajería, más complejidad para Orchestrator
